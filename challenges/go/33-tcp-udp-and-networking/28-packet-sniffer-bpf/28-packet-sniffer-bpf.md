## Prerequisites

- Go 1.22+ installed
- Completed TCP/UDP and Custom Wire Protocol exercises
- Understanding of network layers: Ethernet, IP, TCP, UDP header formats
- Root/admin access for raw socket or packet capture permissions
- Familiarity with `tcpdump` and packet capture concepts

## Learning Objectives

- **Create** a packet sniffer that captures raw network packets and dissects protocol headers
- **Implement** BPF (Berkeley Packet Filter) programs to filter captured packets efficiently in the kernel
- **Design** a protocol dissector that parses Ethernet, IP, TCP, and UDP headers from raw bytes

## The Challenge

Packet sniffers capture raw network traffic for debugging, monitoring, and security analysis. Tools like `tcpdump` and Wireshark capture packets at the data link layer, parsing protocol headers to display human-readable information. BPF (Berkeley Packet Filter) programs run in the kernel to filter packets before they are copied to userspace, dramatically improving performance by discarding uninteresting packets early.

Your task is to build a packet sniffer in Go that captures raw packets from a network interface, parses Ethernet, IPv4, IPv6, TCP, UDP, and DNS headers, applies BPF filter expressions to select specific traffic, and outputs results in a human-readable format. You will also implement pcap file writing to save captured packets for later analysis in Wireshark.

## Requirements

1. Open a raw packet capture handle using `gopacket/pcap` or raw sockets (`syscall.Socket` with `AF_PACKET`/`SOCK_RAW`)
2. Parse Ethernet frames: extract source MAC, destination MAC, and EtherType
3. Parse IPv4 headers: extract version, IHL, total length, TTL, protocol, source IP, destination IP, and header checksum
4. Parse IPv6 headers: extract version, traffic class, flow label, payload length, next header, hop limit, source/destination addresses
5. Parse TCP headers: extract source port, destination port, sequence number, acknowledgment number, flags (SYN, ACK, FIN, RST, PSH), window size, and checksum
6. Parse UDP headers: extract source port, destination port, length, and checksum
7. Implement BPF filter compilation and attachment: accept filter expressions like `tcp port 80`, `host 192.168.1.1`, or `udp and port 53` using `pcap.CompileBPFFilter` or manual BPF bytecode
8. Implement packet output format: timestamp, source address:port, destination address:port, protocol, flags, payload length (similar to `tcpdump` output)
9. Write captured packets to pcap format files that can be opened in Wireshark
10. Implement packet statistics: packets captured, packets dropped (kernel buffer overflow), bytes captured, protocol distribution (TCP/UDP/ICMP/other)
11. Parse DNS payloads from UDP port 53 traffic: extract query name, type, and response records
12. Write tests that capture packets from a loopback interface during a known TCP exchange and verify correct header parsing

## Hints

- `gopacket` and `gopacket/pcap` provide high-level packet capture and dissection; for the learning exercise, implement your own header parsing first, then compare with `gopacket` output
- Raw packet capture requires elevated privileges (root or `CAP_NET_RAW` capability)
- Ethernet frames on the loopback interface use a different format (no MAC addresses on Linux; BSD uses a 4-byte header)
- BPF filter expressions compile to bytecode instructions that run in a virtual machine in the kernel; `pcap.CompileBPFFilter` generates this bytecode from tcpdump-style expressions
- For pcap file format: 24-byte global header (magic, version, snaplen, link type) followed by per-packet headers (timestamp, captured length, original length) and packet data
- TCP header length is variable due to options; use the Data Offset field (4 bits, in 32-bit words) to determine the actual header length
- Use `binary.BigEndian` for network byte order parsing; IP and TCP headers are always big-endian

## Success Criteria

1. The sniffer captures packets from a specified network interface
2. Ethernet, IPv4, TCP, and UDP headers are parsed correctly with all major fields extracted
3. BPF filters correctly select only matching traffic (e.g., `tcp port 80` captures only HTTP traffic)
4. DNS queries and responses on port 53 are parsed and displayed
5. Captured packets are written to pcap files that Wireshark opens without errors
6. Packet statistics accurately report counts and protocol distribution
7. The sniffer captures packets generated by a test TCP exchange on the loopback interface
8. All header checksum validation produces correct results
9. Tests pass with the `-race` flag enabled

## Summary

- Packet sniffers capture raw network frames at the data link layer for analysis and debugging
- BPF filters run in the kernel to select specific traffic before copying to userspace, improving performance
- Protocol dissection parses layered headers: Ethernet wraps IP, which wraps TCP/UDP, which wraps application data
- Network byte order (big-endian) is used for all protocol header fields
- pcap files store captured packets with timestamps for offline analysis in tools like Wireshark
- Raw packet capture requires elevated privileges; use `gopacket/pcap` for a portable capture interface
