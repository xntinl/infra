# Exercise 29.3: Writing a Custom Code Generator

## Prerequisites

Before starting this exercise, you should be comfortable with:

- `//go:generate` directives and the stringer tool (Exercises 29.1-29.2)
- Go `text/template` package
- Interfaces and type assertions

## Learning Objectives

By the end of this exercise, you will be able to:

1. Design and implement a custom code generator in Go
2. Parse command-line flags in a generator to accept type names and output files
3. Use `text/template` to emit well-formatted generated code
4. Integrate the generator with `//go:generate` directives

## Why This Matters

While tools like `stringer` cover common cases, production Go projects often need domain-specific generators -- for creating mock implementations, building type-safe API clients, or generating validation code. Writing your own generator gives you full control over the output and lets you eliminate entire categories of boilerplate.

---

## Problem

Build a custom code generator called `jsonenumgen` that, given a type name, generates `MarshalJSON` and `UnmarshalJSON` methods for integer enum types. The generator should:

1. Accept `-type` and `-output` flags
2. Read the `GOPACKAGE` environment variable to determine the target package
3. Use `text/template` to produce clean, formatted output
4. Include the standard `DO NOT EDIT` marker

### Hints

- Use `flag` package for argument parsing in your generator
- The generator does not need to parse Go source to find const values for this exercise -- accept them as a comma-separated `-values` flag
- Use `go/format` to ensure the generated code is `gofmt`-compliant
- Structure your generator under `cmd/jsonenumgen/main.go` with the `//go:build ignore` tag

### Step 1: Create the project structure

```bash
mkdir -p custom-gen/cmd/jsonenumgen
cd custom-gen
go mod init custom-gen
```

### Step 2: Write the generator

Create `cmd/jsonenumgen/main.go`:

```go
//go:build ignore

package main

import (
	"bytes"
	"flag"
	"fmt"
	"go/format"
	"os"
	"strings"
	"text/template"
)

var tmpl = template.Must(template.New("enum").Parse(`// Code generated by jsonenumgen; DO NOT EDIT.

package {{ .Package }}

import (
	"encoding/json"
	"fmt"
)

var _{{ .Type }}NameMap = map[{{ .Type }}]string{
{{- range .Values }}
	{{ $.Type }}{{ . }}: "{{ . }}",
{{- end }}
}

var _{{ .Type }}ValueMap = map[string]{{ .Type }}{
{{- range .Values }}
	"{{ . }}": {{ $.Type }}{{ . }},
{{- end }}
}

func (t {{ .Type }}) MarshalJSON() ([]byte, error) {
	if name, ok := _{{ .Type }}NameMap[t]; ok {
		return json.Marshal(name)
	}
	return nil, fmt.Errorf("invalid {{ .Type }} value: %d", int(t))
}

func (t *{{ .Type }}) UnmarshalJSON(data []byte) error {
	var s string
	if err := json.Unmarshal(data, &s); err != nil {
		return err
	}
	if val, ok := _{{ .Type }}ValueMap[s]; ok {
		*t = val
		return nil
	}
	return fmt.Errorf("unknown {{ .Type }}: %q", s)
}
`))

type data struct {
	Package string
	Type    string
	Values  []string
}

func main() {
	typeName := flag.String("type", "", "type name")
	values := flag.String("values", "", "comma-separated value names (without type prefix)")
	output := flag.String("output", "", "output file name")
	flag.Parse()

	if *typeName == "" || *values == "" {
		fmt.Fprintln(os.Stderr, "usage: jsonenumgen -type=T -values=A,B,C [-output=file.go]")
		os.Exit(1)
	}

	pkg := os.Getenv("GOPACKAGE")
	if pkg == "" {
		pkg = "main"
	}

	outFile := *output
	if outFile == "" {
		outFile = strings.ToLower(*typeName) + "_json_gen.go"
	}

	vals := strings.Split(*values, ",")
	for i := range vals {
		vals[i] = strings.TrimSpace(vals[i])
	}

	d := data{
		Package: pkg,
		Type:    *typeName,
		Values:  vals,
	}

	var buf bytes.Buffer
	if err := tmpl.Execute(&buf, d); err != nil {
		fmt.Fprintf(os.Stderr, "template error: %v\n", err)
		os.Exit(1)
	}

	formatted, err := format.Source(buf.Bytes())
	if err != nil {
		fmt.Fprintf(os.Stderr, "format error: %v\n", err)
		fmt.Fprintln(os.Stderr, buf.String())
		os.Exit(1)
	}

	if err := os.WriteFile(outFile, formatted, 0644); err != nil {
		fmt.Fprintf(os.Stderr, "write error: %v\n", err)
		os.Exit(1)
	}

	fmt.Printf("Generated %s for type %s\n", outFile, *typeName)
}
```

### Step 3: Create the source file that uses the generator

Create `color.go`:

```go
package main

import (
	"encoding/json"
	"fmt"
)

//go:generate go run cmd/jsonenumgen/main.go -type=Color -values=Red,Green,Blue,Yellow

type Color int

const (
	ColorRed Color = iota
	ColorGreen
	ColorBlue
	ColorYellow
)

func main() {
	c := ColorGreen

	data, err := json.Marshal(c)
	if err != nil {
		fmt.Println("Marshal error:", err)
		return
	}
	fmt.Println("Marshaled:", string(data))

	var decoded Color
	if err := json.Unmarshal(data, &decoded); err != nil {
		fmt.Println("Unmarshal error:", err)
		return
	}
	fmt.Printf("Unmarshaled: %d (Green=%d)\n", decoded, ColorGreen)

	// Test round-trip with a struct
	type Palette struct {
		Primary   Color `json:"primary"`
		Secondary Color `json:"secondary"`
	}

	p := Palette{Primary: ColorBlue, Secondary: ColorYellow}
	pJSON, _ := json.MarshalIndent(p, "", "  ")
	fmt.Println("Palette JSON:")
	fmt.Println(string(pJSON))
}
```

### Step 4: Generate and run

```bash
go generate ./...
go run .
```

Expected output:

```
Marshaled: "Green"
Unmarshaled: 1 (Green=1)
Palette JSON:
{
  "primary": "Blue",
  "secondary": "Yellow"
}
```

## Verify

```bash
rm -f *_gen.go
go generate ./...
cat color_json_gen.go
go run .
```

Confirm that:
- The generated file contains the `DO NOT EDIT` marker
- `MarshalJSON` produces string values like `"Green"`
- `UnmarshalJSON` converts strings back to the correct integer constants
- Invalid values produce clear error messages

## Summary

- Custom generators are regular Go programs invoked by `//go:generate`
- Use `flag` for configuration, `GOPACKAGE` env var for package context
- `text/template` produces the code structure; `go/format` ensures it is valid Go
- Always include the `DO NOT EDIT` marker in generated output
- Generators can produce any Go code: JSON methods, validators, builders, mocks
