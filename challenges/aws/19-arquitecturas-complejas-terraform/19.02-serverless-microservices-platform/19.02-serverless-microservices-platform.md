# 19.02 Serverless Microservices Platform

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform |
| **Servicios AWS** | API Gateway, Lambda, DynamoDB, SQS, SNS, EventBridge, Cognito |

## Descripcion

Construye una plataforma de microservicios serverless: API Gateway -> Lambda functions -> DynamoDB/SQS/SNS -> EventBridge para comunicacion entre servicios. Incluye Cognito para autenticacion, X-Ray para tracing, dominios personalizados y roles IAM por servicio.

## Objetivos de aprendizaje

- Disenar arquitectura serverless a escala con multiples microservicios
- Implementar aislamiento de servicios con roles IAM dedicados
- Aplicar patrones event-driven para comunicacion entre servicios
- Configurar autenticacion con Cognito y observabilidad con X-Ray

## Requisitos previos

- Conocimientos: Terraform intermedio-avanzado, arquitectura serverless, patrones de microservicios, IAM policies
- Herramientas: Terraform >= 1.6, AWS CLI, Node.js o Python para funciones Lambda
- Servicios: API Gateway, Lambda, DynamoDB, SQS, SNS, EventBridge, Cognito, X-Ray

## Arquitectura

Plataforma de microservicios con comunicacion asincrona:

1. **API Gateway**: Punto de entrada unificado con autorizadores Cognito y dominios personalizados
2. **Microservicios Lambda**: Cada servicio con su propio conjunto de funciones, rol IAM y tabla DynamoDB
3. **Mensajeria**: SQS para colas de trabajo, SNS para notificaciones fan-out
4. **Eventos**: EventBridge como bus central para comunicacion desacoplada entre servicios
5. **Observabilidad**: X-Ray para tracing distribuido a traves de todos los servicios

## Pasos sugeridos

1. Disenar la estructura de microservicios (minimo 3 servicios con responsabilidades claras)
2. Crear el modulo base de microservicio (Lambda + DynamoDB + rol IAM dedicado)
3. Configurar API Gateway con rutas, stages y modelo de autorizacion Cognito
4. Implementar el Cognito User Pool con app clients y flujos de autenticacion
5. Configurar EventBridge con bus personalizado y reglas de enrutamiento entre servicios
6. Desplegar colas SQS con DLQ para procesamiento asincrono
7. Configurar SNS topics para notificaciones y suscripciones
8. Habilitar X-Ray tracing en Lambda y API Gateway
9. Configurar dominios personalizados con ACM y Route 53
10. Validar la comunicacion end-to-end entre todos los microservicios

## Validacion

- [ ] API Gateway responde correctamente en el dominio personalizado
- [ ] Cognito autentica usuarios y API Gateway valida los tokens
- [ ] Cada microservicio tiene su propio rol IAM con permisos minimos
- [ ] Las funciones Lambda procesan requests y escriben en DynamoDB
- [ ] EventBridge enruta eventos correctamente entre servicios
- [ ] SQS procesa mensajes con reintentos y DLQ funcional
- [ ] SNS entrega notificaciones a los suscriptores
- [ ] X-Ray muestra trazas completas a traves de los servicios
- [ ] Los servicios estan desacoplados (pueden desplegarse independientemente)
