# 19.08 CI/CD Platform Infrastructure

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform |
| **Servicios AWS** | CodePipeline, CodeBuild, ECR, S3, IAM, Secrets Manager |

## Descripcion

Construye la infraestructura para una plataforma CI/CD completa: CodePipeline, CodeBuild con imagenes personalizadas (ECR), almacenamiento de artefactos (S3), roles de despliegue cross-account, Secrets Manager para credenciales y CloudWatch dashboards para metricas de pipelines.

## Objetivos de aprendizaje

- Implementar infraestructura CI/CD completa como codigo con Terraform
- Configurar roles cross-account para despliegues seguros en multiples cuentas
- Gestionar imagenes de build personalizadas con ECR
- Configurar observabilidad de pipelines con CloudWatch dashboards

## Requisitos previos

- Conocimientos: Terraform intermedio-avanzado, CI/CD conceptos, IAM cross-account, contenedores Docker
- Herramientas: Terraform >= 1.6, AWS CLI, Docker
- Servicios: CodePipeline, CodeBuild, ECR, S3, IAM, Secrets Manager, CloudWatch

## Arquitectura

Plataforma CI/CD con despliegues cross-account:

1. **Pipelines**: CodePipeline para orquestar las etapas de CI/CD (source, build, test, deploy)
2. **Build**: CodeBuild con imagenes personalizadas almacenadas en ECR para entornos de build consistentes
3. **Artefactos**: S3 con encriptacion KMS para almacenar artefactos entre etapas del pipeline
4. **Secretos**: Secrets Manager para credenciales de servicios externos y tokens de API
5. **Cross-account**: Roles IAM assume-role para desplegar en cuentas de staging y produccion
6. **Observabilidad**: CloudWatch dashboards con metricas de ejecucion, duracion y tasa de exito de pipelines
7. **Notificaciones**: SNS para alertas de fallos en pipelines

## Pasos sugeridos

1. Crear el repositorio ECR y construir la imagen personalizada de build con las herramientas necesarias
2. Configurar el bucket S3 para artefactos con encriptacion KMS y lifecycle policies
3. Crear los roles IAM para CodeBuild y CodePipeline con permisos minimos
4. Configurar Secrets Manager con las credenciales necesarias para los pipelines
5. Crear los proyectos CodeBuild con buildspec para las fases de build y test
6. Implementar los roles cross-account en las cuentas destino (staging, produccion)
7. Crear los pipelines CodePipeline con todas las etapas y aprobaciones manuales
8. Configurar CloudWatch dashboards con metricas de ejecucion de pipelines
9. Crear alarmas SNS para fallos en pipelines y builds
10. Validar el flujo completo: commit -> build -> test -> deploy a staging -> aprobacion -> produccion

## Validacion

- [ ] ECR contiene la imagen de build personalizada y es accesible por CodeBuild
- [ ] S3 almacena artefactos encriptados entre etapas del pipeline
- [ ] Secrets Manager provee credenciales a CodeBuild sin exponerlas en logs
- [ ] CodeBuild ejecuta builds exitosos usando la imagen personalizada
- [ ] CodePipeline orquesta todas las etapas en el orden correcto
- [ ] Los roles cross-account permiten desplegar en cuentas de staging y produccion
- [ ] Las aprobaciones manuales funcionan correctamente antes de produccion
- [ ] CloudWatch dashboard muestra metricas de exito, duracion y frecuencia de pipelines
- [ ] Las alertas SNS se disparan cuando un pipeline falla
