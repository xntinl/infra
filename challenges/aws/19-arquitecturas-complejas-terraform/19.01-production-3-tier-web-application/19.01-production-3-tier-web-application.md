# 19.01 Production 3-Tier Web Application

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform |
| **Servicios AWS** | CloudFront, ALB, ECS, Aurora, ElastiCache, WAF, ACM, Route 53 |

## Descripcion

Despliega una arquitectura de produccion completa: CloudFront -> ALB -> ECS Fargate (auto-scaling) -> Aurora PostgreSQL (Multi-AZ) + ElastiCache Redis. Incluye VPC con 3 AZs, WAF, ACM, Route 53, CloudWatch dashboards y alertas SNS. Todo gestionado mediante modulos de Terraform.

## Objetivos de aprendizaje

- Disenar y desplegar una arquitectura de produccion completa con multiples capas
- Componer modulos de Terraform para construir sistemas complejos
- Implementar capas de seguridad (WAF, security groups, ACM)
- Configurar monitoreo y alertas con CloudWatch y SNS

## Requisitos previos

- Conocimientos: Terraform avanzado, redes en AWS (VPC, subnets, routing), contenedores (ECS Fargate), bases de datos relacionales (Aurora)
- Herramientas: Terraform >= 1.6, AWS CLI, Docker
- Servicios: CloudFront, ALB, ECS Fargate, Aurora PostgreSQL, ElastiCache Redis, WAF, ACM, Route 53, CloudWatch, SNS

## Arquitectura

Arquitectura de tres capas con alta disponibilidad:

1. **Capa de presentacion**: CloudFront como CDN frontal con WAF para proteccion, certificados ACM y Route 53 para DNS
2. **Capa de aplicacion**: ALB distribuyendo trafico a servicios ECS Fargate con auto-scaling en 3 AZs
3. **Capa de datos**: Aurora PostgreSQL Multi-AZ para persistencia y ElastiCache Redis para cache
4. **Observabilidad**: CloudWatch dashboards centralizados y alertas SNS

La VPC abarca 3 Availability Zones con subnets publicas, privadas y de base de datos.

## Pasos sugeridos

1. Crear el modulo de VPC con 3 AZs, subnets publicas, privadas y de base de datos, NAT Gateways y tablas de ruteo
2. Configurar el cluster Aurora PostgreSQL Multi-AZ con parameter groups y subnet groups
3. Desplegar ElastiCache Redis con replicacion en multiples AZs
4. Crear el cluster ECS Fargate con task definitions, servicios y auto-scaling policies
5. Configurar el ALB con target groups, health checks y listeners HTTPS
6. Desplegar CloudFront con origen en el ALB, cache policies y certificado ACM
7. Configurar WAF con reglas de proteccion (rate limiting, SQL injection, XSS)
8. Establecer Route 53 con registros DNS apuntando a CloudFront
9. Crear CloudWatch dashboards con metricas de cada capa y alarmas SNS
10. Validar la arquitectura completa con pruebas de conectividad y rendimiento

## Validacion

- [ ] La VPC tiene 3 AZs con subnets publicas, privadas y de datos correctamente configuradas
- [ ] Aurora PostgreSQL esta en modo Multi-AZ con failover funcional
- [ ] ElastiCache Redis responde y replica entre AZs
- [ ] ECS Fargate escala automaticamente segun la carga
- [ ] El ALB distribuye trafico correctamente con health checks activos
- [ ] CloudFront sirve contenido con cache y certificado SSL valido
- [ ] WAF bloquea solicitudes maliciosas (probar con patrones SQL injection)
- [ ] Route 53 resuelve el dominio correctamente
- [ ] CloudWatch dashboards muestran metricas de todas las capas
- [ ] Las alertas SNS se disparan ante condiciones criticas
