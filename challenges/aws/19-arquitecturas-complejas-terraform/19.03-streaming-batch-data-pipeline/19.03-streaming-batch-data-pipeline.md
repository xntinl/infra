# 19.03 Streaming & Batch Data Pipeline

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform |
| **Servicios AWS** | Kinesis, Lambda, Glue, Athena, S3, DynamoDB |

## Descripcion

Despliega un pipeline de datos de modo dual: tiempo real (Kinesis -> Lambda -> DynamoDB) y batch (S3 -> Glue ETL -> Athena). Incluye validacion de calidad de datos, schema registry, monitoreo y una capa de consulta unificada. Toda la infraestructura gestionada por Terraform.

## Objetivos de aprendizaje

- Disenar arquitectura de pipeline de datos con procesamiento stream y batch
- Implementar procesamiento en tiempo real con Kinesis y Lambda
- Configurar ETL batch con Glue y consultas ad-hoc con Athena
- Aplicar controles de calidad de datos y gestion de esquemas

## Requisitos previos

- Conocimientos: Terraform intermedio-avanzado, conceptos de data engineering, procesamiento de streams, ETL
- Herramientas: Terraform >= 1.6, AWS CLI, Python (para scripts Glue y Lambda)
- Servicios: Kinesis Data Streams, Lambda, Glue (ETL + Catalog), Athena, S3, DynamoDB

## Arquitectura

Pipeline de datos dual con capa de consulta unificada:

1. **Ingesta en tiempo real**: Kinesis Data Streams recibe eventos, Lambda los procesa y escribe en DynamoDB para acceso de baja latencia
2. **Ingesta batch**: Datos aterrizan en S3 (raw zone), Glue ETL los transforma y deposita en S3 (curated zone) con formato Parquet
3. **Catalogo**: Glue Data Catalog como metastore centralizado con schema registry
4. **Consultas**: Athena consulta datos batch en S3; DynamoDB sirve datos en tiempo real
5. **Calidad**: Validaciones de esquema en Lambda y Glue Data Quality rules
6. **Monitoreo**: CloudWatch metricas para throughput, errores y latencia del pipeline

## Pasos sugeridos

1. Crear la estructura de buckets S3 (raw, staging, curated) con lifecycle policies
2. Desplegar Kinesis Data Stream con la capacidad adecuada (on-demand o provisioned)
3. Implementar la funcion Lambda consumidora de Kinesis con logica de transformacion
4. Configurar DynamoDB como destino del pipeline en tiempo real
5. Crear el Glue Data Catalog con databases y tablas para los datos
6. Implementar Glue ETL jobs para transformacion batch (PySpark)
7. Configurar Glue Crawlers para descubrimiento automatico de esquemas
8. Desplegar Athena workgroups con configuracion de resultados en S3
9. Implementar validaciones de calidad de datos en ambos pipelines
10. Configurar monitoreo con CloudWatch dashboards y alarmas

## Validacion

- [ ] Kinesis Data Stream ingesta eventos correctamente
- [ ] Lambda procesa records de Kinesis y escribe en DynamoDB sin errores
- [ ] Los datos llegan a S3 raw zone con la estructura esperada
- [ ] Glue ETL transforma y deposita datos en formato Parquet en curated zone
- [ ] Glue Crawlers detectan y actualizan esquemas automaticamente
- [ ] Athena ejecuta consultas exitosas sobre los datos curados
- [ ] Las validaciones de calidad detectan datos malformados
- [ ] CloudWatch muestra metricas de throughput y latencia del pipeline
- [ ] El pipeline batch se ejecuta segun el schedule configurado
