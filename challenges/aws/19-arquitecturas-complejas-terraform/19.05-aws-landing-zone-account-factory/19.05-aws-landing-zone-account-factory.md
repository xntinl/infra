# 19.05 AWS Landing Zone & Account Factory

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Experto |
| **Modalidad** | Terraform |
| **Servicios AWS** | Organizations, Control Tower, Transit Gateway, GuardDuty, Security Hub |

## Descripcion

Construye una landing zone empresarial: AWS Organizations, estructura de OUs, Account Factory for Terraform (AFT), SCPs de baseline, logging centralizado, servicios de seguridad (GuardDuty, Security Hub, Config) y networking compartido con Transit Gateway.

## Objetivos de aprendizaje

- Disenar e implementar una landing zone empresarial con Terraform
- Gestionar multiples cuentas a escala con AWS Organizations y AFT
- Aplicar gobernanza organizacional con SCPs y servicios de seguridad
- Implementar networking compartido con Transit Gateway

## Requisitos previos

- Conocimientos: Terraform experto, AWS Organizations, multi-account strategy, networking avanzado, seguridad en AWS
- Herramientas: Terraform >= 1.6, AWS CLI, acceso a una cuenta de gestion de Organizations
- Servicios: AWS Organizations, Control Tower, AFT, Transit Gateway, GuardDuty, Security Hub, AWS Config, CloudTrail

## Arquitectura

Landing zone empresarial multi-cuenta:

1. **Organizacion**: AWS Organizations con estructura de OUs (Security, Infrastructure, Workloads, Sandbox)
2. **Account Factory**: AFT para provisionamiento automatico de cuentas con baseline de Terraform
3. **Gobernanza**: SCPs por OU para restricciones (regiones permitidas, servicios bloqueados, proteccion de guardrails)
4. **Seguridad**: GuardDuty, Security Hub y Config delegados a una cuenta Security centralizada
5. **Logging**: CloudTrail organizacional + S3 centralizado en cuenta Log Archive
6. **Networking**: Transit Gateway en cuenta Network para conectividad compartida entre cuentas
7. **Baseline**: Cada cuenta nueva recibe automaticamente: IAM roles, Config rules, CloudTrail, VPC baseline

## Pasos sugeridos

1. Configurar AWS Organizations con la estructura de OUs definida
2. Crear las cuentas fundacionales (Security, Log Archive, Network, Shared Services)
3. Implementar SCPs de baseline por OU (restriccion de regiones, proteccion de roles criticos)
4. Configurar Account Factory for Terraform (AFT) con los pipelines de provisionamiento
5. Desplegar Transit Gateway en la cuenta Network con route tables y attachments
6. Configurar CloudTrail organizacional con destino al bucket centralizado en Log Archive
7. Habilitar y delegar GuardDuty a la cuenta Security con auto-enable para nuevas cuentas
8. Configurar Security Hub con estandares de cumplimiento (CIS, AWS Foundational)
9. Desplegar AWS Config aggregator con reglas de conformance packs
10. Crear el template de baseline para nuevas cuentas (IAM, networking, logging, seguridad)

## Validacion

- [ ] La estructura de OUs refleja el diseno organizacional
- [ ] SCPs estan activas y bloquean acciones no permitidas (probar con acceso a region restringida)
- [ ] AFT provisiona nuevas cuentas con el baseline completo automaticamente
- [ ] Transit Gateway permite conectividad entre cuentas autorizadas
- [ ] CloudTrail organizacional registra eventos de todas las cuentas en el bucket centralizado
- [ ] GuardDuty esta habilitado en todas las cuentas con findings centralizados
- [ ] Security Hub muestra el compliance score de todas las cuentas
- [ ] Config rules evaluan recursos en todas las cuentas miembro
- [ ] Nuevas cuentas reciben automaticamente la configuracion baseline
