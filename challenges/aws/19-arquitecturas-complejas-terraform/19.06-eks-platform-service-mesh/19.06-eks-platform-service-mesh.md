# 19.06 EKS Platform & Service Mesh

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Experto |
| **Modalidad** | Terraform (EKS Blueprints) |
| **Servicios AWS** | EKS, App Mesh, ADOT, Route 53, EBS, EFS, Karpenter |

## Descripcion

Despliega un cluster EKS de produccion: Karpenter para nodos, IRSA para IAM de pods, App Mesh para service mesh, ADOT para observabilidad, External DNS + cert-manager para ingress, EBS CSI + EFS CSI para storage. Todo mediante Terraform EKS Blueprints.

## Objetivos de aprendizaje

- Desplegar y gestionar Kubernetes de produccion en AWS con EKS
- Configurar add-ons de EKS mediante Terraform EKS Blueprints
- Implementar service mesh con App Mesh para comunicacion entre servicios
- Configurar observabilidad avanzada con ADOT (OpenTelemetry)

## Requisitos previos

- Conocimientos: Terraform experto, Kubernetes avanzado, networking de contenedores, service mesh, IAM para workloads
- Herramientas: Terraform >= 1.6, AWS CLI, kubectl, helm
- Servicios: EKS, App Mesh, ADOT, Route 53, EBS CSI, EFS CSI, Karpenter, ACM

## Arquitectura

Plataforma EKS de produccion con service mesh y observabilidad:

1. **Cluster EKS**: Control plane gestionado con acceso privado y publico configurado
2. **Nodos**: Karpenter para aprovisionamiento automatico y eficiente de nodos (spot + on-demand)
3. **Identidad**: IRSA (IAM Roles for Service Accounts) para acceso seguro a servicios AWS desde pods
4. **Service Mesh**: App Mesh para observabilidad, traffic management y seguridad entre servicios
5. **Ingress**: ALB Ingress Controller + External DNS + cert-manager para exposicion de servicios
6. **Storage**: EBS CSI Driver para volumenes de bloque y EFS CSI Driver para almacenamiento compartido
7. **Observabilidad**: ADOT Collector para metricas, trazas y logs hacia CloudWatch y X-Ray

## Pasos sugeridos

1. Crear la VPC con subnets etiquetadas para EKS (tags de kubernetes.io/role)
2. Desplegar el cluster EKS usando el modulo EKS Blueprints de Terraform
3. Configurar Karpenter con Provisioners para distintos tipos de workloads
4. Implementar IRSA creando roles IAM y asociandolos a service accounts de Kubernetes
5. Desplegar App Mesh controller y configurar virtual nodes, services y routers
6. Instalar EBS CSI Driver y EFS CSI Driver como EKS add-ons
7. Configurar ALB Ingress Controller con External DNS y cert-manager
8. Desplegar ADOT Collector configurado para enviar telemetria a CloudWatch y X-Ray
9. Desplegar una aplicacion de ejemplo con multiples microservicios en el mesh
10. Validar observabilidad end-to-end: metricas, trazas y logs desde los servicios

## Validacion

- [ ] El cluster EKS esta funcional con acceso via kubectl
- [ ] Karpenter aprovisiona y desaprovisiona nodos automaticamente segun la demanda
- [ ] IRSA permite a los pods acceder a servicios AWS sin access keys
- [ ] App Mesh rutea trafico entre servicios con virtual nodes y routers
- [ ] EBS CSI crea PersistentVolumes dinamicamente para pods stateful
- [ ] EFS CSI permite almacenamiento compartido entre multiples pods
- [ ] External DNS crea registros DNS automaticamente para Ingress resources
- [ ] cert-manager provisiona certificados TLS automaticamente
- [ ] ADOT envia metricas y trazas a CloudWatch y X-Ray
- [ ] La aplicacion de ejemplo funciona correctamente dentro del service mesh
