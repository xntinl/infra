# 19.09 Compliance & Governance Platform

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Experto |
| **Modalidad** | Terraform |
| **Servicios AWS** | Config, Security Hub, GuardDuty, CloudTrail, Lambda, SSM |

## Descripcion

Despliega una plataforma de compliance organizacional: AWS Config (aggregator + reglas), Security Hub, GuardDuty, CloudTrail (org trail) y remediacion automatizada (Lambda + SSM). Todo gestionado centralmente via Terraform con provisionamiento por cuenta.

## Objetivos de aprendizaje

- Implementar gobernanza a escala en multiples cuentas con Terraform
- Automatizar la deteccion y remediacion de problemas de compliance
- Configurar servicios de seguridad multi-cuenta de forma centralizada
- Disenar estrategias de compliance como codigo

## Requisitos previos

- Conocimientos: Terraform experto, seguridad en AWS, AWS Organizations, compliance frameworks (CIS, PCI)
- Herramientas: Terraform >= 1.6, AWS CLI, Python (para funciones Lambda de remediacion)
- Servicios: AWS Config, Security Hub, GuardDuty, CloudTrail, Lambda, SSM, Organizations

## Arquitectura

Plataforma de compliance centralizada multi-cuenta:

1. **AWS Config**: Aggregator en cuenta central que recopila evaluaciones de todas las cuentas. Config Rules como conformance packs desplegados organizacionalmente
2. **Security Hub**: Habilitado en todas las cuentas con estandares CIS y AWS Foundational. Findings centralizados en cuenta Security
3. **GuardDuty**: Detector en cada cuenta con administrador delegado. Findings de amenazas centralizados
4. **CloudTrail**: Trail organizacional que registra todas las API calls en todas las cuentas
5. **Remediacion automatica**: Lambda functions activadas por Config Rule violations que ejecutan documentos SSM Automation para corregir configuraciones
6. **Reportes**: Dashboards de compliance con metricas por cuenta y por regla

## Pasos sugeridos

1. Configurar AWS Config en la cuenta central con aggregator organizacional
2. Crear conformance packs con Config Rules (S3 public access, EBS encryption, security groups)
3. Desplegar Config Rules organizacionales para aplicar a todas las cuentas automaticamente
4. Habilitar Security Hub con estandares CIS Benchmark y AWS Foundational Security
5. Configurar GuardDuty con administrador delegado y auto-enable para nuevas cuentas
6. Crear CloudTrail organizacional con integracion a CloudWatch Logs y S3
7. Implementar funciones Lambda de remediacion para las violaciones mas criticas
8. Crear documentos SSM Automation para remediaciones complejas (multi-paso)
9. Configurar EventBridge rules que conecten Config non-compliant findings con las Lambdas de remediacion
10. Crear CloudWatch dashboards con metricas de compliance por cuenta, servicio y regla

## Validacion

- [ ] Config aggregator muestra evaluaciones de todas las cuentas miembro
- [ ] Config Rules evaluan correctamente los recursos (probar con recurso non-compliant)
- [ ] Security Hub muestra compliance score con estandares CIS y Foundational
- [ ] GuardDuty esta activo en todas las cuentas con findings centralizados
- [ ] CloudTrail organizacional registra eventos de todas las cuentas
- [ ] La remediacion automatica corrige violaciones (probar: S3 bucket publico se privatiza automaticamente)
- [ ] SSM Automation documents ejecutan remediaciones complejas exitosamente
- [ ] EventBridge conecta deteccion con remediacion sin intervencion manual
- [ ] Los dashboards muestran el estado de compliance actualizado de la organizacion
