# 3.09 S3 Intelligent-Tiering y Storage Lens

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Intermedio |
| **Modalidad** | Ambos |
| **Servicios AWS** | S3, S3 Intelligent-Tiering, S3 Storage Lens, S3 Storage Class Analysis |

## Descripcion

Configurar S3 Intelligent-Tiering con archive access tiers sobre un dataset grande. Habilitar S3 Storage Lens a nivel de organizacion, crear dashboards personalizados, analizar patrones de almacenamiento, y usar Storage Class Analysis para validar las decisiones de tiering.

## Objetivos de aprendizaje

- Optimizar costos de almacenamiento mediante tiering automatizado
- Comprender como Intelligent-Tiering mueve objetos entre tiers automaticamente
- Usar Storage Lens para obtener analiticas de almacenamiento a escala
- Analizar patrones de acceso a datos con Storage Class Analysis para tomar decisiones informadas

## Requisitos previos

- **Conocimientos**: Fundamentos de S3 (storage classes, lifecycle), conceptos de costos de almacenamiento en AWS
- **Herramientas**: AWS CLI v2, Terraform >= 1.5, AWS Management Console (para dashboards de Storage Lens)
- **Servicios AWS**: Permisos para S3, S3 Storage Lens; idealmente multiples buckets con datos existentes

## Arquitectura

Un bucket S3 con Intelligent-Tiering como storage class por defecto, configurado con los archive access tiers opcionales (Archive Access a 90 dias, Deep Archive Access a 180 dias). El bucket contiene un dataset grande con patrones de acceso variados (algunos objetos accedidos frecuentemente, otros raramente). S3 Storage Lens esta configurado a nivel de organizacion con un dashboard personalizado que muestra metricas de almacenamiento, actividad y costos. Storage Class Analysis esta habilitado en buckets clave para generar reportes de patrones de acceso y validar que Intelligent-Tiering esta tomando las decisiones correctas.

## Pasos sugeridos

1. Crear un bucket S3 con Intelligent-Tiering como storage class por defecto
2. Configurar Intelligent-Tiering con los archive access tiers (Archive Access y Deep Archive Access)
3. Subir un dataset grande con objetos de diferentes tamanos y simular patrones de acceso variados
4. Habilitar S3 Storage Lens con el dashboard por defecto a nivel de cuenta
5. Crear un dashboard personalizado de Storage Lens con metricas avanzadas (cost optimization, data protection)
6. Configurar Storage Lens para exportar metricas a un bucket S3 en formato CSV o Parquet
7. Habilitar Storage Class Analysis en el bucket principal con filtros por prefijo
8. Esperar al menos 24-48 horas para que las metricas y analisis empiecen a generar datos
9. Analizar los reportes de Storage Class Analysis para validar las transiciones de Intelligent-Tiering
10. Crear un resumen de costos comparando Intelligent-Tiering vs Standard vs lifecycle manual

## Validacion

- [ ] El bucket tiene Intelligent-Tiering como storage class con archive tiers habilitados
- [ ] Los objetos se mueven automaticamente entre tiers segun patrones de acceso
- [ ] S3 Storage Lens muestra metricas a nivel de cuenta/organizacion
- [ ] El dashboard personalizado de Storage Lens esta creado con metricas avanzadas
- [ ] Las metricas de Storage Lens se exportan a un bucket S3
- [ ] Storage Class Analysis genera reportes de patrones de acceso
- [ ] Se puede demostrar el ahorro de costos comparado con Standard storage class
