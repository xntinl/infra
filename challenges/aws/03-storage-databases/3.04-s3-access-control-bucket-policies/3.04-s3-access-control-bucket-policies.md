## Why Access Control Matters

S3 buckets are one of the most commonly misconfigured AWS resources. Public buckets have led to some of the largest data breaches in cloud history -- from customer databases to classified government documents. AWS has added multiple layers of protection (Block Public Access, bucket ownership controls, access points), but the fundamental responsibility for configuring access correctly remains with you.

S3 access control has two models: **resource-based policies** (bucket policies, attached to the bucket itself) and **identity-based policies** (IAM policies, attached to users/roles). When a request arrives, AWS evaluates both. If either policy explicitly denies the request, access is denied. If neither grants access, the request is implicitly denied. Only when at least one policy allows the request -- and no policy denies it -- is access granted.

Bucket policies are JSON documents that define who can do what on the bucket and its objects. They are the primary access control mechanism for S3 because they travel with the bucket (not with the user) and can grant access to external accounts, AWS services, and anonymous users. ACLs (Access Control Lists) are a legacy mechanism from before bucket policies existed. AWS recommends disabling ACLs entirely on new buckets using the `BucketOwnerEnforced` object ownership setting.

> **Best Practice:** Always use bucket policies over ACLs. ACLs are legacy, hard to audit, and limited to a small set of predefined grants. Bucket policies are expressive, version-controlled, and auditable. For new buckets, set object ownership to `BucketOwnerEnforced` to disable ACLs entirely.

### `s3.tf`

```hcl
resource "random_id" "suffix" {
  byte_length = 4
}

# ----------------------------------------------------------
# Bucket with strict access control (no public access)
# ----------------------------------------------------------
resource "aws_s3_bucket" "private" {
  bucket        = "${var.project_name}-private-${random_id.suffix.hex}"
  force_destroy = true

  tags = {
    Name        = "${var.project_name}-private"
    Environment = "lab"
  }
}

resource "aws_s3_bucket_public_access_block" "private" {
  bucket = aws_s3_bucket.private.id

  block_public_acls       = true
  block_public_policy     = true
  ignore_public_acls      = true
  restrict_public_buckets = true
}

# Disable ACLs entirely. BucketOwnerEnforced means the bucket
# owner automatically owns every object, regardless of who
# uploaded it. This eliminates the need for ACLs.
resource "aws_s3_bucket_ownership_controls" "private" {
  bucket = aws_s3_bucket.private.id

  rule {
    object_ownership = "BucketOwnerEnforced"
  }
}

# Bucket policy: restrict access to specific IAM principals.
# This policy allows read-only access from the current account
# and denies any request not using HTTPS (TLS).
resource "aws_s3_bucket_policy" "private" {
  bucket = aws_s3_bucket.private.id
  policy = data.aws_iam_policy_document.private_bucket.json

  depends_on = [aws_s3_bucket_public_access_block.private]
}

data "aws_iam_policy_document" "private_bucket" {
  # Enforce HTTPS-only access. Any request over plain HTTP
  # is denied. This protects data in transit.
  statement {
    sid    = "DenyInsecureTransport"
    effect = "Deny"

    principals {
      type        = "*"
      identifiers = ["*"]
    }

    actions = ["s3:*"]

    resources = [
      aws_s3_bucket.private.arn,
      "${aws_s3_bucket.private.arn}/*",
    ]

    condition {
      test     = "Bool"
      variable = "aws:SecureTransport"
      values   = ["false"]
    }
  }

  # Allow the current account to read objects
  statement {
    sid    = "AllowAccountReadAccess"
    effect = "Allow"

    principals {
      type        = "AWS"
      identifiers = ["arn:aws:iam::${data.aws_caller_identity.current.account_id}:root"]
    }

    actions = [
      "s3:GetObject",
      "s3:ListBucket",
    ]

    resources = [
      aws_s3_bucket.private.arn,
      "${aws_s3_bucket.private.arn}/*",
    ]
  }
}

# ----------------------------------------------------------
# Bucket with write-only access (log delivery pattern)
# ----------------------------------------------------------
resource "aws_s3_bucket" "logs" {
  bucket        = "${var.project_name}-logs-${random_id.suffix.hex}"
  force_destroy = true

  tags = {
    Name        = "${var.project_name}-logs"
    Environment = "lab"
  }
}

resource "aws_s3_bucket_public_access_block" "logs" {
  bucket = aws_s3_bucket.logs.id

  block_public_acls       = true
  block_public_policy     = true
  ignore_public_acls      = true
  restrict_public_buckets = true
}

resource "aws_s3_bucket_ownership_controls" "logs" {
  bucket = aws_s3_bucket.logs.id

  rule {
    object_ownership = "BucketOwnerEnforced"
  }
}

# Log delivery bucket policy: allow AWS services to write logs
# but deny deletion. This is a common pattern for audit logs,
# CloudTrail, and VPC Flow Logs.
resource "aws_s3_bucket_policy" "logs" {
  bucket = aws_s3_bucket.logs.id
  policy = data.aws_iam_policy_document.logs_bucket.json

  depends_on = [aws_s3_bucket_public_access_block.logs]
}

data "aws_iam_policy_document" "logs_bucket" {
  # Enforce HTTPS
  statement {
    sid    = "DenyInsecureTransport"
    effect = "Deny"

    principals {
      type        = "*"
      identifiers = ["*"]
    }

    actions = ["s3:*"]

    resources = [
      aws_s3_bucket.logs.arn,
      "${aws_s3_bucket.logs.arn}/*",
    ]

    condition {
      test     = "Bool"
      variable = "aws:SecureTransport"
      values   = ["false"]
    }
  }

  # Allow the logging service to deliver log files
  statement {
    sid    = "AllowLogDelivery"
    effect = "Allow"

    principals {
      type        = "Service"
      identifiers = ["logging.s3.amazonaws.com"]
    }

    actions = [
      "s3:PutObject",
    ]

    resources = [
      "${aws_s3_bucket.logs.arn}/logs/*",
    ]

    condition {
      test     = "StringEquals"
      variable = "aws:SourceAccount"
      values   = [data.aws_caller_identity.current.account_id]
    }
  }

  # Deny all delete operations -- logs must be immutable
  statement {
    sid    = "DenyDeletion"
    effect = "Deny"

    principals {
      type        = "*"
      identifiers = ["*"]
    }

    actions = [
      "s3:DeleteObject",
      "s3:DeleteObjectVersion",
    ]

    resources = [
      "${aws_s3_bucket.logs.arn}/*",
    ]
  }
}

# ----------------------------------------------------------
# Bucket with IP-restricted access
# ----------------------------------------------------------
resource "aws_s3_bucket" "restricted" {
  bucket        = "${var.project_name}-restricted-${random_id.suffix.hex}"
  force_destroy = true

  tags = {
    Name        = "${var.project_name}-restricted"
    Environment = "lab"
  }
}

resource "aws_s3_bucket_public_access_block" "restricted" {
  bucket = aws_s3_bucket.restricted.id

  block_public_acls       = true
  block_public_policy     = true
  ignore_public_acls      = true
  restrict_public_buckets = true
}

resource "aws_s3_bucket_ownership_controls" "restricted" {
  bucket = aws_s3_bucket.restricted.id

  rule {
    object_ownership = "BucketOwnerEnforced"
  }
}

# Policy that restricts access to specific IP ranges.
# This is useful for allowing access only from your office
# network or VPN. For lab purposes, we use the current
# account as the principal.
resource "aws_s3_bucket_policy" "restricted" {
  bucket = aws_s3_bucket.restricted.id
  policy = data.aws_iam_policy_document.restricted_bucket.json

  depends_on = [aws_s3_bucket_public_access_block.restricted]
}

data "aws_iam_policy_document" "restricted_bucket" {
  # Deny access from outside specified IP ranges.
  # Replace 203.0.113.0/24 with your actual office CIDR.
  statement {
    sid    = "DenyAccessOutsideAllowedIPs"
    effect = "Deny"

    principals {
      type        = "*"
      identifiers = ["*"]
    }

    actions = ["s3:*"]

    resources = [
      aws_s3_bucket.restricted.arn,
      "${aws_s3_bucket.restricted.arn}/*",
    ]

    condition {
      test     = "NotIpAddress"
      variable = "aws:SourceIp"
      values   = ["203.0.113.0/24", "198.51.100.0/24"]
    }

    # Exclude requests from AWS services (they use internal IPs)
    condition {
      test     = "Bool"
      variable = "aws:ViaAWSService"
      values   = ["false"]
    }
  }

  # Enforce HTTPS
  statement {
    sid    = "DenyInsecureTransport"
    effect = "Deny"

    principals {
      type        = "*"
      identifiers = ["*"]
    }

    actions = ["s3:*"]

    resources = [
      aws_s3_bucket.restricted.arn,
      "${aws_s3_bucket.restricted.arn}/*",
    ]

    condition {
      test     = "Bool"
      variable = "aws:SecureTransport"
      values   = ["false"]
    }
  }
}

# Upload test objects to the private bucket
resource "aws_s3_object" "private_test" {
  bucket       = aws_s3_bucket.private.id
  key          = "documents/policy.txt"
  content      = "This document is protected by a bucket policy."
  content_type = "text/plain"

  tags = {
    Name = "${var.project_name}-policy-test"
  }
}

resource "aws_s3_object" "private_config" {
  bucket       = aws_s3_bucket.private.id
  key          = "config/settings.json"
  content      = jsonencode({
    access_level = "restricted"
    encryption   = true
  })
  content_type = "application/json"

  tags = {
    Name = "${var.project_name}-config"
  }
}
```

> **Best Practice:** Every bucket policy should include a `DenyInsecureTransport` statement that denies requests where `aws:SecureTransport` is `false`. This ensures all data in transit is encrypted via TLS, which is a common compliance requirement (PCI-DSS, HIPAA, SOC 2).

## Common Mistakes

### 1. Using `jsonencode()` instead of `data "aws_iam_policy_document"`

Terraform's `aws_iam_policy_document` data source provides type safety, interpolation, and better readability than hand-writing JSON. It also catches errors at plan time instead of apply time.

**Wrong -- hardcoded JSON:**

```hcl
resource "aws_s3_bucket_policy" "this" {
  bucket = aws_s3_bucket.this.id
  policy = jsonencode({
    Version = "2012-10-17"
    Statement = [{
      Effect    = "Deny"
      Principal = "*"
      Action    = "s3:*"
      Resource  = "${aws_s3_bucket.this.arn}/*"
      Condition = {
        Bool = { "aws:SecureTransport" = "false" }
      }
    }]
  })
}
```

**Fix -- use the data source:**

```hcl
data "aws_iam_policy_document" "this" {
  statement {
    effect = "Deny"
    principals {
      type        = "*"
      identifiers = ["*"]
    }
    actions   = ["s3:*"]
    resources = ["${aws_s3_bucket.this.arn}/*"]
    condition {
      test     = "Bool"
      variable = "aws:SecureTransport"
      values   = ["false"]
    }
  }
}

resource "aws_s3_bucket_policy" "this" {
  bucket = aws_s3_bucket.this.id
  policy = data.aws_iam_policy_document.this.json
}
```

### 2. Forgetting the bucket ARN in resource list

Bucket-level operations (`s3:ListBucket`, `s3:GetBucketLocation`) apply to the bucket ARN. Object-level operations (`s3:GetObject`, `s3:PutObject`) apply to the bucket ARN with `/*`. Mixing these up causes `AccessDenied` errors.

**Wrong -- only object ARN:**

```hcl
resources = ["${aws_s3_bucket.this.arn}/*"]  # Missing bucket-level ARN
```

**Fix -- include both:**

```hcl
resources = [
  aws_s3_bucket.this.arn,        # For bucket-level operations
  "${aws_s3_bucket.this.arn}/*", # For object-level operations
]
```

### 3. Applying the bucket policy before the public access block

If you apply a public bucket policy before Block Public Access is in place, the bucket is briefly public. Always ensure the public access block is created first.

**Fix -- add explicit dependency:**

```hcl
resource "aws_s3_bucket_policy" "this" {
  bucket     = aws_s3_bucket.this.id
  policy     = data.aws_iam_policy_document.this.json
  depends_on = [aws_s3_bucket_public_access_block.this]
}
```

## Summary

- **Bucket policies** are resource-based JSON documents attached to the bucket itself
- **IAM policies** are identity-based and attached to users, groups, or roles
- AWS evaluates both -- an explicit `Deny` in either always wins
- **Block Public Access** overrides any policy or ACL that would grant public access
- **Object ownership** set to `BucketOwnerEnforced` disables ACLs entirely
- Use `data "aws_iam_policy_document"` in Terraform for type-safe, readable policies
- Always include a `DenyInsecureTransport` statement to enforce HTTPS
- Include both the bucket ARN and `/*` pattern in policy resources
