# 14.05 Patron Resource Factory con Configuracion YAML

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform |
| **Features TF** | `yamldecode()`, `for_each`, dynamic blocks, `fileset()` |

## Descripcion

Construir un modulo "resource factory" que lea archivos de configuracion YAML y cree recursos via `for_each`. Ejemplo: una Lambda factory que lea `functions.yaml` para crear funciones Lambda, roles IAM y alarmas CloudWatch para cada entrada.

## Objetivos de aprendizaje

- Implementar infraestructura dirigida por configuracion (config-driven)
- Integrar YAML como fuente de verdad para recursos de Terraform
- Aplicar el patron factory para creacion masiva de recursos
- Lograr configuraciones DRY eliminando repeticion

## Requisitos previos

- Conocimientos: Terraform modules, YAML, `for_each`, dynamic blocks, Lambda
- Herramientas: Terraform >= 1.0, AWS CLI
- Servicios/Features: Lambda, IAM, CloudWatch Alarms, `yamldecode()`, `fileset()`

## Arquitectura

El patron factory desacopla la definicion de recursos de su implementacion:

- **Archivos YAML**: Fuente de verdad que define cada recurso con sus propiedades (nombre, runtime, memoria, timeout, alarmas)
- **Factory Module**: Modulo de Terraform que lee los YAML con `yamldecode()` y crea recursos via `for_each`
- **Recursos Generados por Entrada**:
  - Lambda Function con la configuracion especificada
  - IAM Role y Policy con permisos minimos
  - CloudWatch Log Group con retencion configurable
  - CloudWatch Alarms (errores, duracion) por funcion
- **Dynamic Blocks**: Para generar configuraciones variables como environment variables o VPC config
- **Extensibilidad**: Agregar una nueva Lambda requiere solo agregar una entrada al YAML

## Pasos sugeridos

1. Disenar la estructura del archivo YAML de configuracion de funciones Lambda
2. Crear el modulo factory con una variable que acepte la ruta al directorio de archivos YAML
3. Usar `fileset()` para descubrir todos los archivos YAML y `yamldecode()` para parsearlos
4. Implementar la creacion de funciones Lambda con `for_each` sobre las entradas del YAML
5. Crear IAM roles y policies dinamicamente para cada funcion usando `for_each`
6. Usar dynamic blocks para configurar environment variables opcionales por funcion
7. Implementar alarmas CloudWatch para cada funcion (errores, throttles, duracion)
8. Agregar CloudWatch Log Groups con retencion configurable por funcion
9. Probar agregando una nueva funcion al YAML y verificando que Terraform la detecta
10. Validar que eliminar una entrada del YAML resulta en la destruccion del recurso correspondiente

## Validacion

- [ ] El modulo lee correctamente archivos YAML con `yamldecode()`
- [ ] `for_each` crea los recursos correspondientes a cada entrada del YAML
- [ ] Los dynamic blocks generan configuraciones variables correctamente
- [ ] Agregar una entrada al YAML crea todos los recursos asociados
- [ ] Eliminar una entrada del YAML destruye los recursos correspondientes
- [ ] Cada funcion Lambda tiene su IAM role, log group y alarmas asociadas
- [ ] La configuracion es DRY: no hay repeticion de bloques de recursos
