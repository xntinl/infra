# 14.03 Testing de Modulos con Terratest

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform + Go |
| **Features TF** | Module outputs, Terratest helpers |

## Descripcion

Escribir tests de integracion para un modulo de Terraform usando Terratest (Go). Los tests deben: desplegar el modulo, validar outputs, verificar atributos de recursos via AWS SDK, comprobar que endpoints HTTP responden, y limpiar los recursos. Ejecutar los tests en CI.

## Objetivos de aprendizaje

- Implementar testing de infraestructura con Terratest
- Aplicar patrones de test para modulos de Terraform
- Garantizar aislamiento de tests y limpieza de recursos
- Integrar tests de infraestructura en pipelines CI

## Requisitos previos

- Conocimientos: Terraform modules, Go basico, testing conceptos, AWS SDK
- Herramientas: Terraform >= 1.0, Go >= 1.21, Terratest library, AWS CLI
- Features: Module outputs, Go testing, AWS SDK for Go

## Arquitectura

El framework de testing valida modulos de Terraform de extremo a extremo:

- **Modulo Bajo Test**: Un modulo de Terraform que despliega recursos reales (por ejemplo, ALB + EC2 + Security Groups)
- **Test Suite (Go)**: Archivos `*_test.go` que usan Terratest para orquestar el ciclo init->apply->validate->destroy
- **Validaciones**:
  - Output validation: verificar que los outputs del modulo son correctos
  - AWS SDK checks: consultar la API de AWS para verificar atributos de recursos
  - HTTP checks: hacer requests a endpoints desplegados y verificar respuestas
- **Cleanup**: `defer terraform.Destroy()` garantiza la limpieza incluso si el test falla
- **CI Integration**: Los tests se ejecutan en GitHub Actions con credenciales AWS temporales

## Pasos sugeridos

1. Crear un modulo de Terraform de ejemplo (ALB + target group + EC2 con user data basico)
2. Inicializar un modulo Go con `go mod init` e importar Terratest como dependencia
3. Escribir el primer test: inicializar Terraform, aplicar y verificar que no hay errores
4. Agregar `defer terraform.Destroy()` para garantizar limpieza de recursos
5. Validar outputs del modulo usando `terraform.Output()` de Terratest
6. Usar AWS SDK para verificar atributos del recurso (por ejemplo, tags de la instancia)
7. Implementar un HTTP health check contra el endpoint del ALB desplegado
8. Agregar retry logic para esperar a que los recursos esten disponibles
9. Implementar aislamiento de tests usando nombres unicos con `random.UniqueId()`
10. Configurar GitHub Actions para ejecutar los tests con credenciales AWS via OIDC

## Validacion

- [ ] Los tests despliegan el modulo de Terraform exitosamente
- [ ] Los outputs del modulo se validan programaticamente
- [ ] Se verifican atributos de recursos via AWS SDK (tags, configuracion)
- [ ] Se valida la respuesta HTTP del endpoint desplegado
- [ ] Los recursos se limpian automaticamente al finalizar el test (incluso en caso de fallo)
- [ ] Los tests usan nombres unicos para evitar conflictos entre ejecuciones
- [ ] Los tests se ejecutan en CI con GitHub Actions
