# 14.01 Modulo VPC Composable con Subnets Flexibles

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Intermedio |
| **Modalidad** | Terraform |
| **Features TF** | `cidrsubnet()`, `for_each`, conditional resources, variable validation |

## Descripcion

Construir un modulo VPC que acepte una configuracion flexible de subnets (numero de AZs, capas de subnets, calculos de CIDR). Usar `cidrsubnet()` para asignacion automatica de CIDR. Soportar NAT Gateways opcionales, VPC endpoints y flow logs via feature flags.

## Objetivos de aprendizaje

- Disenar modulos de Terraform reutilizables y flexibles
- Dominar la funcion `cidrsubnet()` para matematica de CIDR
- Implementar creacion condicional de recursos con feature flags
- Disenar interfaces de modulos intuitivas y bien validadas

## Requisitos previos

- Conocimientos: Terraform intermedio, networking AWS (VPC, subnets, CIDR), modulos
- Herramientas: Terraform >= 1.3, AWS CLI
- Servicios/Features: VPC, Subnets, NAT Gateway, VPC Endpoints, Flow Logs

## Arquitectura

El modulo VPC es un componente fundamental de infraestructura con alta flexibilidad:

- **VPC Base**: CIDR configurable con DNS support y hostnames habilitados
- **Subnet Tiers**: Capas configurables (public, private, database) distribuidas por AZ
- **CIDR Calculation**: `cidrsubnet()` calcula automaticamente los rangos de cada subnet basado en el CIDR de la VPC
- **NAT Gateway** (opcional): Single NAT o NAT por AZ, controlado por feature flag
- **VPC Endpoints** (opcional): Gateway endpoints para S3 y DynamoDB, interface endpoints configurables
- **Flow Logs** (opcional): CloudWatch Logs o S3 como destino, controlado por feature flag
- **Outputs**: VPC ID, subnet IDs por capa, route table IDs, NAT Gateway IPs

## Pasos sugeridos

1. Definir la interfaz del modulo: variables de entrada con tipos complejos y validaciones
2. Implementar la creacion de la VPC con CIDR configurable
3. Usar `cidrsubnet()` para calcular CIDRs de subnets automaticamente segun el numero de AZs y capas
4. Implementar subnets publicas con `for_each` sobre las AZs disponibles
5. Implementar subnets privadas y de base de datos con la misma logica
6. Agregar Internet Gateway y route tables para subnets publicas
7. Implementar NAT Gateway condicional con flag `enable_nat_gateway` y opcion `single_nat_gateway`
8. Agregar VPC Endpoints opcionales (S3, DynamoDB) controlados por feature flags
9. Implementar Flow Logs opcionales con destino configurable
10. Definir outputs completos y escribir ejemplos de uso del modulo

## Validacion

- [ ] El modulo acepta configuracion flexible de AZs (1 a N)
- [ ] `cidrsubnet()` calcula los CIDRs automaticamente sin conflictos
- [ ] Los feature flags controlan correctamente la creacion de NAT Gateway, endpoints y flow logs
- [ ] Las variables tienen validaciones que rechazan configuraciones invalidas
- [ ] El modulo funciona con diferentes CIDRs y numeros de AZs
- [ ] Los outputs exponen toda la informacion necesaria para consumidores del modulo
- [ ] Se incluye al menos un ejemplo de uso completo
