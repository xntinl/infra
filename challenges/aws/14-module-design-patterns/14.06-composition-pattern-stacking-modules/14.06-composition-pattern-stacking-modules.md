# 14.06 Patron de Composicion: Stacking de Modulos

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform |
| **Features TF** | Module composition, output chaining, module `depends_on` |

## Descripcion

Construir una capa de composicion que apile multiples modulos juntos: modulo VPC + modulo ECS + modulo RDS + modulo de monitoring. Usar outputs de un modulo como inputs de otro. Implementar un modulo "stack" que encapsule un entorno de aplicacion completo.

## Objetivos de aprendizaje

- Dominar la composicion de modulos en Terraform
- Gestionar dependencias entre modulos correctamente
- Disenar abstracciones de stack que simplifiquen despliegues
- Crear interfaces limpias entre modulos

## Requisitos previos

- Conocimientos: Terraform modules avanzado, AWS networking, ECS, RDS, CloudWatch
- Herramientas: Terraform >= 1.0, AWS CLI
- Servicios/Features: VPC, ECS Fargate, RDS, CloudWatch, module composition

## Arquitectura

La composicion apila modulos en capas con dependencias claras:

- **Modulo VPC** (capa base): Crea VPC, subnets, NAT Gateways. Expone VPC ID, subnet IDs, security group IDs
- **Modulo RDS** (capa datos): Consume subnet IDs y SG IDs del modulo VPC. Crea RDS instance/cluster. Expone endpoint y credenciales
- **Modulo ECS** (capa compute): Consume VPC outputs y RDS endpoint. Crea ECS cluster, task definitions, services. Expone ALB DNS
- **Modulo Monitoring** (capa observabilidad): Consume outputs de todos los modulos anteriores. Crea dashboards, alarmas, log groups
- **Modulo Stack** (capa composicion): Encapsula todos los modulos anteriores con una interfaz simplificada. Un solo `module "stack"` despliega el entorno completo

## Pasos sugeridos

1. Disenar las interfaces de cada modulo: definir inputs y outputs claros
2. Implementar el modulo VPC con outputs bien definidos (vpc_id, subnet_ids, sg_ids)
3. Implementar el modulo RDS que consume outputs del VPC como inputs
4. Implementar el modulo ECS que consume outputs del VPC y RDS
5. Implementar el modulo de monitoring que consume outputs de todos los anteriores
6. Crear el modulo stack que instancia todos los modulos y conecta sus outputs/inputs
7. Usar `depends_on` a nivel de modulo donde las dependencias implicitas no son suficientes
8. Definir la interfaz simplificada del stack: pocas variables que controlan todo el entorno
9. Desplegar el stack completo y verificar que todos los componentes funcionan juntos
10. Documentar el grafo de dependencias y las interfaces entre modulos

## Validacion

- [ ] Cada modulo individual funciona de forma independiente con los inputs correctos
- [ ] Los outputs de un modulo se pasan correctamente como inputs al siguiente
- [ ] El modulo stack encapsula todos los modulos con una interfaz simplificada
- [ ] Las dependencias entre modulos se resuelven correctamente (implicitas y explicitas)
- [ ] El despliegue completo del stack funciona sin errores
- [ ] Los modulos de monitoring reciben informacion de todos los otros modulos
- [ ] El grafo de dependencias esta documentado
