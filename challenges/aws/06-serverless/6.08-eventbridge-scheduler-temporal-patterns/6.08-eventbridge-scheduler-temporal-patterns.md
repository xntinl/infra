# 6.08 EventBridge Scheduler Temporal Patterns

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Intermedio |
| **Modalidad** | Ambos |
| **Servicios AWS** | EventBridge Scheduler, Lambda, Step Functions, SQS, DLQ |

## Descripcion

Implementar diversos patrones de scheduling usando EventBridge Scheduler: tareas programadas de una sola vez, schedules cron recurrentes con ventanas de tiempo flexibles, schedules basados en rate. Apuntar a Lambda, Step Functions y SQS como targets. Implementar manejo de dead-letter para schedules fallidos.

## Objetivos de aprendizaje

- Comprender patrones de scheduling en arquitecturas serverless
- Configurar expresiones cron y rate para diferentes necesidades
- Implementar schedules de una sola vez vs recurrentes
- Garantizar la fiabilidad de schedules con dead-letter queues

## Requisitos previos

- Conocimientos: expresiones cron, Lambda, conceptos de scheduling
- Herramientas: AWS CLI, Terraform o CloudFormation, lenguaje para Lambda
- Servicios AWS: cuenta con permisos para EventBridge Scheduler, Lambda, Step Functions, SQS

## Arquitectura

La arquitectura implementa multiples patrones de scheduling:

1. **One-time Schedule**: schedule que se ejecuta una sola vez en una fecha/hora especifica (ej. enviar recordatorio)
2. **Cron Schedule**: schedule recurrente con expresion cron (ej. reporte diario a las 8 AM)
3. **Rate Schedule**: schedule basado en frecuencia (ej. cada 5 minutos para health check)
4. **Flexible Time Window**: ventana de tiempo que permite ejecutar el schedule dentro de un rango (reduce picos)
5. **Lambda Target**: funcion que ejecuta logica de negocio (generar reportes, limpiar datos)
6. **Step Functions Target**: workflow que ejecuta procesos multi-paso
7. **SQS Target**: cola que recibe mensajes programados para procesamiento asincronico
8. **DLQ**: cola de mensajes muertos para invocaciones fallidas de schedules

## Pasos sugeridos

1. Crear una funcion Lambda que registre la hora de invocacion y el payload recibido
2. Crear un schedule de una sola vez (one-time) que invoque la Lambda en una hora especifica
3. Crear un schedule cron recurrente que invoque la Lambda diariamente
4. Crear un schedule basado en rate que invoque la Lambda cada N minutos
5. Configurar una flexible time window en un schedule para distribuir la carga
6. Crear un schedule que apunte a un Step Functions workflow como target
7. Crear un schedule que envie mensajes a una cola SQS como target
8. Configurar una DLQ en SQS y asociarla a los schedules para capturar invocaciones fallidas
9. Probar el comportamiento de cada tipo de schedule y verificar los logs
10. Implementar un schedule group para organizar y gestionar schedules relacionados

## Validacion

- [ ] El schedule de una sola vez se ejecuta en la hora programada y no se repite
- [ ] El schedule cron se ejecuta segun la expresion configurada
- [ ] El schedule basado en rate se ejecuta con la frecuencia esperada
- [ ] La flexible time window distribuye ejecuciones dentro del rango configurado
- [ ] El target Lambda se invoca correctamente con el payload esperado
- [ ] El target Step Functions inicia el workflow correctamente
- [ ] El target SQS recibe los mensajes programados
- [ ] Las invocaciones fallidas se envian a la DLQ
- [ ] Los schedule groups organizan los schedules correctamente
