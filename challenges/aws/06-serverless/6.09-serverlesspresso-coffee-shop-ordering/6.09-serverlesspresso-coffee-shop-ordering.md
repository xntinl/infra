# 6.09 Serverlesspresso Coffee Shop Ordering

## Descripcion

Construir una aplicacion serverless orientada a eventos inspirada en el demo AWS Serverlesspresso. Implementar maquinas de estado de pedidos con Step Functions, enrutamiento de eventos con EventBridge, actualizaciones en tiempo real via IoT Core o WebSocket API, y un dashboard frontend.

## Objetivos de aprendizaje

- Disenar arquitecturas complejas orientadas a eventos
- Orquestar workflows con Step Functions
- Implementar comunicacion en tiempo real en aplicaciones serverless
- Construir una aplicacion serverless full-stack de principio a fin

## Requisitos previos

- Conocimientos: Step Functions, EventBridge, API Gateway, Lambda, DynamoDB, desarrollo frontend basico (HTML/JS)
- Herramientas: AWS CLI, Terraform o CloudFormation, Node.js, framework frontend basico
- Servicios AWS: cuenta con permisos para Step Functions, EventBridge, API Gateway, Lambda, DynamoDB, IoT Core, S3, Cognito

## Arquitectura

La arquitectura modela un sistema de pedidos de cafeteria con multiples componentes:

1. **API Gateway**: recibe pedidos de los clientes via REST API
2. **Step Functions - Order Workflow**: maquina de estados que gestiona el ciclo de vida del pedido (recibido -> en preparacion -> listo -> entregado)
3. **EventBridge**: bus de eventos central para comunicacion entre componentes
4. **Lambda Functions**: multiples funciones para logica de negocio (validar pedido, notificar barista, actualizar estado)
5. **DynamoDB**: tablas para pedidos, menu, configuracion de la tienda
6. **IoT Core / WebSocket API**: canal de comunicacion en tiempo real para actualizar el dashboard
7. **S3 + CloudFront**: hosting del frontend (dashboard del barista y pantalla del cliente)
8. **Cognito**: autenticacion para el dashboard del barista

El flujo: cliente hace pedido -> API Gateway -> Lambda valida -> Step Functions orquesta -> EventBridge notifica -> IoT Core/WebSocket actualiza dashboard en tiempo real.

## Pasos sugeridos

1. Disenar el modelo de datos: tablas DynamoDB para pedidos, menu y configuracion
2. Crear las funciones Lambda para logica de negocio (crear pedido, validar, actualizar estado)
3. Disenar la state machine de Step Functions para el ciclo de vida del pedido
4. Configurar EventBridge con reglas para enrutar eventos entre componentes
5. Implementar la API REST con API Gateway para recibir y consultar pedidos
6. Configurar IoT Core o WebSocket API para actualizaciones en tiempo real
7. Implementar la logica de publicacion de eventos al canal en tiempo real
8. Crear el dashboard del barista (frontend) que muestre pedidos y permita cambiar estados
9. Crear la pantalla del cliente que muestre el estado de su pedido en tiempo real
10. Hospedar el frontend en S3 con CloudFront y configurar Cognito para el dashboard del barista
11. Integrar todos los componentes y probar el flujo completo de extremo a extremo

## Validacion

- [ ] El cliente puede hacer un pedido via la API REST
- [ ] El Step Functions workflow gestiona el ciclo de vida del pedido correctamente
- [ ] Los eventos se emiten a EventBridge en cada cambio de estado
- [ ] Las actualizaciones en tiempo real llegan al dashboard del barista
- [ ] El barista puede cambiar el estado del pedido desde el dashboard
- [ ] La pantalla del cliente se actualiza en tiempo real con el estado del pedido
- [ ] El frontend esta hospedado en S3/CloudFront y es accesible
- [ ] La autenticacion de Cognito protege el dashboard del barista
- [ ] El flujo completo funciona: pedido -> preparacion -> listo -> entregado
- [ ] Los datos persisten correctamente en DynamoDB
