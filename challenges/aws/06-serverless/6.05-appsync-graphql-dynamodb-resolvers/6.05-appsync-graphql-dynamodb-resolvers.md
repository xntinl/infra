# 6.05 AppSync GraphQL DynamoDB Resolvers

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Ambos |
| **Servicios AWS** | AppSync, DynamoDB, Cognito, Lambda |

## Descripcion

Construir una API GraphQL usando AppSync con resolvers VTL y JavaScript pipeline conectados a DynamoDB. Implementar subscriptions para actualizaciones en tiempo real, autorizacion granular con grupos de Cognito, y resolucion de conflictos para modo offline.

## Objetivos de aprendizaje

- Comprender GraphQL sobre AWS con AppSync
- Implementar resolver pipelines con VTL y JavaScript
- Configurar subscriptions para actualizaciones en tiempo real
- Disenar modelos de autorizacion granular
- Implementar resolucion de conflictos para sincronizacion offline

## Requisitos previos

- Conocimientos: GraphQL (queries, mutations, subscriptions), DynamoDB, Cognito basico
- Herramientas: AWS CLI, Terraform o CloudFormation, cliente GraphQL (Postman, Apollo)
- Servicios AWS: cuenta con permisos para AppSync, DynamoDB, Cognito, Lambda

## Arquitectura

La arquitectura implementa una API GraphQL completa con AppSync:

1. **AppSync API**: API GraphQL con schema definiendo Types, Queries, Mutations y Subscriptions
2. **DynamoDB Tables**: tablas de datos (ej. Posts, Comments, Users) como data source principal
3. **VTL Resolvers**: resolvers en Velocity Template Language para operaciones CRUD simples
4. **JavaScript Pipeline Resolvers**: resolvers en JS para logica compleja (ej. autorizacion + query)
5. **Cognito User Pool**: autenticacion de usuarios con grupos (admin, editor, viewer)
6. **Lambda Data Source**: resolver Lambda para logica de negocio compleja
7. **Subscriptions**: actualizaciones en tiempo real via WebSocket cuando se crean/actualizan datos
8. **Conflict Resolution**: estrategia de resolucion para operaciones offline (Automerge, Optimistic Concurrency)

## Pasos sugeridos

1. Disenar el schema GraphQL con tipos (Post, Comment, User), queries, mutations y subscriptions
2. Crear las tablas DynamoDB con los indices necesarios (GSIs para queries por autor, fecha, etc.)
3. Crear la API AppSync con el schema y configurar Cognito como modo de autorizacion
4. Implementar resolvers VTL para operaciones CRUD basicas (createPost, getPost, listPosts)
5. Implementar un pipeline resolver en JavaScript con pasos de autorizacion y consulta
6. Configurar un Cognito User Pool con grupos (admin, editor, viewer)
7. Implementar autorizacion granular usando directivas @aws_auth y logica en resolvers
8. Configurar subscriptions para notificar cuando se crea o actualiza un Post
9. Implementar un resolver Lambda para logica compleja (ej. busqueda full-text)
10. Configurar conflict resolution para soportar operaciones offline

## Validacion

- [ ] La API AppSync se crea con el schema GraphQL completo
- [ ] Los resolvers VTL ejecutan operaciones CRUD en DynamoDB correctamente
- [ ] Los pipeline resolvers en JavaScript encadenan multiples pasos
- [ ] La autenticacion con Cognito funciona (tokens JWT validos requeridos)
- [ ] La autorizacion por grupos funciona (admin puede todo, viewer solo lectura)
- [ ] Las subscriptions notifican en tiempo real cuando se crean/actualizan datos
- [ ] El resolver Lambda procesa logica de negocio compleja
- [ ] La resolucion de conflictos maneja correctamente operaciones concurrentes
- [ ] Las queries con filtros y paginacion funcionan correctamente
