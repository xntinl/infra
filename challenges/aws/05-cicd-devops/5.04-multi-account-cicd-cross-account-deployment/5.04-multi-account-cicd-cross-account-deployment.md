# 5.04 Multi-Account CI/CD Cross-Account Deployment

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Ambos |
| **Servicios AWS** | CodePipeline, CodeBuild, IAM (cross-account roles), S3, KMS |

## Descripcion

Construir un pipeline en una cuenta de herramientas (tools account) que despliegue infraestructura en cuentas de dev, staging y produccion. Usar roles IAM cross-account asumidos por CodePipeline. Implementar overrides de parametros especificos por entorno y promover artefactos a traves de las etapas.

## Objetivos de aprendizaje

- Comprender patrones de despliegue cross-account en AWS
- Configurar encriptacion de artefactos con KMS para compartir entre cuentas
- Implementar role chaining para acceso cross-account seguro
- Disenar promocion de artefactos a traves de multiples entornos

## Requisitos previos

- Conocimientos: IAM roles y policies, conceptos de multi-account, KMS, CodePipeline
- Herramientas: AWS CLI con perfiles para multiples cuentas, Terraform o CloudFormation
- Servicios AWS: al menos 2 cuentas AWS (tools + target), permisos para IAM, CodePipeline, CodeBuild, S3, KMS

## Arquitectura

La arquitectura distribuye responsabilidades entre multiples cuentas AWS:

1. **Cuenta Tools**: aloja CodePipeline, CodeBuild, el bucket S3 de artefactos y la clave KMS
2. **Cuenta Dev**: recibe despliegues automaticos tras cada commit
3. **Cuenta Staging**: recibe despliegues tras aprobacion automatica o manual
4. **Cuenta Prod**: recibe despliegues tras aprobacion manual obligatoria

Cada cuenta target tiene un rol IAM que CodePipeline asume para ejecutar despliegues. La clave KMS es compartida para que todas las cuentas puedan leer los artefactos encriptados en S3.

## Pasos sugeridos

1. Configurar la cuenta Tools con CodePipeline, CodeBuild y un bucket S3 para artefactos
2. Crear una clave KMS en la cuenta Tools con policy que permita acceso desde las cuentas target
3. Crear roles IAM en cada cuenta target con permisos para CloudFormation/Terraform y los recursos necesarios
4. Configurar trust policies en los roles target para permitir que la cuenta Tools los asuma
5. Crear el pipeline con etapas Source, Build, DeployDev, DeployStagin y DeployProd
6. Configurar CodeBuild para compilar artefactos y parametrizar por entorno
7. Implementar parametros especificos por entorno (ej. tamano de instancia, replicas)
8. Agregar etapa de aprobacion manual antes de produccion
9. Validar que los artefactos encriptados se comparten correctamente entre cuentas
10. Probar el flujo completo de promocion: dev -> staging -> prod

## Validacion

- [ ] El pipeline en la cuenta Tools se ejecuta correctamente
- [ ] CodePipeline asume roles cross-account exitosamente en cada cuenta target
- [ ] Los artefactos se encriptan con KMS y son accesibles desde todas las cuentas
- [ ] El despliegue en dev se ejecuta automaticamente tras cada commit
- [ ] El despliegue en staging usa parametros especificos de ese entorno
- [ ] La aprobacion manual es requerida antes del despliegue a produccion
- [ ] Los parametros de cada entorno (tamano, replicas, etc.) se aplican correctamente
- [ ] El role chaining funciona sin problemas de permisos
