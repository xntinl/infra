# 5.05 Infrastructure Testing Terratest

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform + Go |
| **Servicios AWS** | CodeBuild, CodePipeline, EC2/Lambda (test targets), S3 |

## Descripcion

Escribir tests de integracion para modulos Terraform usando Terratest (Go). Los tests deben desplegar infraestructura real, validar que funciona correctamente (ej. un endpoint HTTP responde, un bucket S3 esta encriptado), y destruirla al finalizar. Integrar la ejecucion en un pipeline de CodeBuild.

## Objetivos de aprendizaje

- Comprender patrones de testing de infraestructura
- Implementar Infrastructure as Code orientado a tests (test-driven IaC)
- Integrar tests de infraestructura en pipelines de CI/CD
- Automatizar validacion de recursos desplegados

## Requisitos previos

- Conocimientos: Terraform (modulos, outputs), Go basico, conceptos de testing
- Herramientas: Go 1.21+, Terraform, AWS CLI, Git
- Servicios AWS: cuenta con permisos para EC2, Lambda, S3, CodeBuild, CodePipeline

## Arquitectura

La arquitectura tiene dos componentes principales:

1. **Modulos Terraform bajo test**: modulos reutilizables que despliegan recursos AWS (ej. un modulo de S3 con encriptacion, un modulo de Lambda con API Gateway)
2. **Suite de tests Terratest**: codigo Go que ejecuta `terraform apply`, valida los recursos creados mediante llamadas HTTP o SDK calls, y ejecuta `terraform destroy` al finalizar
3. **Pipeline CI**: CodeBuild ejecuta los tests en cada push, con un role IAM que tiene permisos para crear y destruir los recursos de test

## Pasos sugeridos

1. Crear un modulo Terraform simple (ej. S3 bucket con encriptacion y versionado)
2. Configurar el proyecto Go con Terratest como dependencia
3. Escribir un test que despliegue el modulo S3 y valide que la encriptacion esta activa
4. Crear un segundo modulo Terraform (ej. Lambda + API Gateway)
5. Escribir un test que despliegue la Lambda, invoque el endpoint HTTP y valide la respuesta
6. Implementar `defer terraform.Destroy()` para garantizar la limpieza de recursos
7. Agregar tests de validacion de tags, politicas IAM y configuraciones de seguridad
8. Crear el buildspec.yml para CodeBuild con los pasos de instalacion de Go y ejecucion de tests
9. Configurar el pipeline de CodePipeline con CodeBuild como etapa de test
10. Ejecutar el pipeline y verificar que los tests pasan y los recursos se limpian

## Validacion

- [ ] Los modulos Terraform se despliegan correctamente durante los tests
- [ ] El test de S3 valida que la encriptacion esta habilitada
- [ ] El test de Lambda valida que el endpoint HTTP responde correctamente
- [ ] `terraform destroy` se ejecuta siempre, incluso cuando los tests fallan
- [ ] Los tests se ejecutan exitosamente en CodeBuild
- [ ] El pipeline completo funciona de extremo a extremo
- [ ] Los recursos de test se limpian completamente tras cada ejecucion
- [ ] Los tests validan propiedades de seguridad (encriptacion, policies, tags)
