# 5.03 GitOps ArgoCD EKS

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Experto |
| **Modalidad** | Terraform (EKS Blueprints) + ArgoCD manifests |
| **Servicios AWS** | EKS, ArgoCD, Argo Rollouts, ECR, ALB |

## Descripcion

Desplegar ArgoCD sobre EKS usando EKS Blueprints. Configurar el patron App of Apps con repositorios separados para workloads y add-ons. Implementar politicas de sincronizacion automatica, health checks, y Progressive Delivery con Argo Rollouts (canary y blue/green).

## Objetivos de aprendizaje

- Comprender los principios de GitOps y su implementacion practica
- Gestionar infraestructura declarativa con ArgoCD
- Implementar el patron App of Apps para gestionar multiples aplicaciones
- Configurar Progressive Delivery con Argo Rollouts
- Dominar estrategias de despliegue en Kubernetes (canary, blue/green)

## Requisitos previos

- Conocimientos: Kubernetes (deployments, services, ingress), Helm, Terraform, conceptos de GitOps
- Herramientas: AWS CLI, kubectl, Terraform, Helm, Git
- Servicios AWS: cuenta con permisos para EKS, ECR, ALB, IAM, VPC

## Arquitectura

La arquitectura sigue el modelo GitOps donde Git es la unica fuente de verdad:

1. **EKS Cluster**: desplegado via Terraform usando EKS Blueprints con add-ons gestionados
2. **ArgoCD**: instalado como add-on del cluster, gestiona la sincronizacion de manifiestos
3. **App of Apps**: una aplicacion raiz de ArgoCD que despliega todas las demas aplicaciones
4. **Repo de workloads**: contiene los manifiestos de Kubernetes para las aplicaciones
5. **Repo de add-ons**: contiene configuraciones de herramientas del cluster (monitoring, ingress, etc.)
6. **Argo Rollouts**: controlador que reemplaza Deployments estandar con estrategias canary/blue-green
7. **ALB Ingress Controller**: gestiona el trafico externo hacia los servicios del cluster

## Pasos sugeridos

1. Crear el cluster EKS usando Terraform con EKS Blueprints y los add-ons necesarios
2. Instalar ArgoCD como add-on del cluster via EKS Blueprints
3. Configurar la autenticacion de ArgoCD con los repositorios Git
4. Crear el repositorio de add-ons con Helm charts para monitoring, ingress controller, etc.
5. Crear el repositorio de workloads con manifiestos de aplicaciones de ejemplo
6. Implementar el patron App of Apps con una aplicacion raiz que referencie las demas
7. Configurar politicas de sincronizacion automatica (auto-sync, self-heal, auto-prune)
8. Instalar Argo Rollouts en el cluster y crear un Rollout con estrategia canary
9. Crear un Rollout con estrategia blue/green y configurar promotion automatica
10. Validar health checks y Progressive Delivery haciendo cambios en los repositorios

## Validacion

- [ ] El cluster EKS se despliega correctamente via Terraform con EKS Blueprints
- [ ] ArgoCD esta instalado y accesible via la interfaz web
- [ ] El patron App of Apps despliega todas las aplicaciones al sincronizar
- [ ] Los cambios en Git se sincronizan automaticamente al cluster (auto-sync)
- [ ] Self-heal revierte cambios manuales hechos directamente en el cluster
- [ ] Argo Rollouts ejecuta un despliegue canary con analisis de metricas
- [ ] Argo Rollouts ejecuta un despliegue blue/green con promotion
- [ ] Los health checks de ArgoCD reflejan el estado real de las aplicaciones
