# 10.03 Cross-Region S3 Replication y Estrategia de Backup

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Ambos |
| **Servicios AWS** | S3 (CRR), AWS Backup, RDS, DynamoDB, EFS, CloudWatch |

## Descripcion

Implementar una estrategia de backup integral: S3 CRR con RTC para datos criticos, AWS Backup para RDS/DynamoDB/EFS con reglas de copia cross-region, y probar la restauracion en la region secundaria. Documentar y validar el RPO para cada almacen de datos.

## Objetivos de aprendizaje

- Disenar una estrategia de backup integral para multiples servicios AWS
- Configurar proteccion de datos cross-region con S3 CRR y RTC
- Utilizar AWS Backup para centralizar backups de RDS, DynamoDB y EFS
- Ejecutar pruebas de restauracion y validar tiempos de recuperacion
- Documentar y validar RPO para cada tipo de almacenamiento

## Requisitos previos

- Conocimientos: S3 replication, conceptos de backup/restore, RDS, DynamoDB, EFS basico
- Herramientas: AWS CLI, Terraform
- Servicios AWS: Cuenta con permisos en dos regiones para S3, AWS Backup, RDS, DynamoDB, EFS

## Arquitectura

La estrategia de backup cubre multiples capas de almacenamiento. S3 Cross-Region Replication (CRR) replica objetos criticos a un bucket en la region secundaria, con Replication Time Control (RTC) para garantizar replicacion en menos de 15 minutos (con metricas de monitoreo). AWS Backup centraliza la gestion de backups para RDS (snapshots automaticos), DynamoDB (backups on-demand y PITR), y EFS (backups incrementales). Las reglas de copia cross-region en AWS Backup envian copias de seguridad a un vault en la region secundaria. Se realizan pruebas periodicas de restauracion para validar la integridad de los backups.

## Pasos sugeridos

1. Crear buckets S3 en ambas regiones y configurar CRR con RTC habilitado
2. Configurar metricas de S3 Replication para monitorear el tiempo de replicacion
3. Crear una instancia RDS, una tabla DynamoDB con PITR, y un filesystem EFS con datos de prueba
4. Crear un vault de AWS Backup en la region primaria y otro en la secundaria
5. Definir un plan de backup en AWS Backup con reglas para RDS, DynamoDB y EFS
6. Configurar reglas de copia cross-region para enviar backups al vault secundario
7. Ejecutar backups manuales y verificar que las copias llegan a la region secundaria
8. Realizar pruebas de restauracion de cada tipo de recurso en la region secundaria
9. Medir y documentar el RPO real para cada servicio (S3, RDS, DynamoDB, EFS)
10. Crear alarmas CloudWatch para detectar fallos de replicacion o backups

## Validacion

- [ ] S3 CRR replica objetos a la region secundaria dentro del SLA de RTC (15 minutos)
- [ ] Las metricas de replicacion de S3 muestran el estado y tiempo de replicacion
- [ ] AWS Backup ejecuta backups programados para RDS, DynamoDB y EFS
- [ ] Las copias cross-region llegan al vault de la region secundaria
- [ ] La restauracion de RDS snapshot en la region secundaria es exitosa
- [ ] La restauracion de DynamoDB backup en la region secundaria es exitosa
- [ ] La restauracion de EFS backup en la region secundaria es exitosa
- [ ] El RPO documentado para cada servicio cumple con los objetivos definidos
