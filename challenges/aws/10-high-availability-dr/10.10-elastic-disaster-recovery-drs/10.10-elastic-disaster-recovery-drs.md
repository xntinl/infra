# 10.10 Elastic Disaster Recovery (DRS)

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Manual (consola DRS) + Terraform para configuracion de red |
| **Servicios AWS** | AWS DRS, EC2, VPC, S3, CloudWatch |

## Descripcion

Configurar AWS Elastic Disaster Recovery para replicar instancias on-premises o EC2 a una region destino. Configurar ajustes de replicacion, staging area y launch settings. Realizar un drill (prueba no disruptiva) y un ciclo completo de failover/failback.

## Objetivos de aprendizaje

- Comprender la replicacion a nivel de bloque y su funcionamiento continuo
- Ejecutar pruebas de DR (drills) sin afectar la produccion
- Orquestar el proceso completo de failover y failback
- Configurar procedimientos de failback para retornar a la region original
- Alcanzar RPO de sub-segundo con replicacion continua

## Requisitos previos

- Conocimientos: EC2, VPC, conceptos de replicacion, estrategias de DR
- Herramientas: AWS CLI, Terraform (para red), acceso a consola DRS
- Servicios AWS: Cuenta con permisos para DRS, EC2, VPC, S3, CloudWatch

## Arquitectura

AWS Elastic Disaster Recovery (DRS) replica continuamente los volumenes de las instancias origen a nivel de bloque hacia la region destino. Un agente de replicacion instalado en las maquinas origen captura los cambios de bloque y los transmite a la staging area en la region destino. La staging area consiste en instancias EC2 ligeras y volumenes EBS que mantienen los datos replicados. Los launch settings definen como se lanzaran las instancias de recuperacion (tipo de instancia, VPC, subnets, security groups). Durante un drill, DRS lanza instancias de prueba desde los datos replicados sin interrumpir la replicacion ni afectar la produccion. Durante un failover real, las instancias de recuperacion se lanzan y el trafico se redirige. El failback permite revertir la replicacion para retornar a la region original cuando se resuelve el incidente.

## Pasos sugeridos

1. Configurar la VPC y subnets en la region destino para la staging area y las instancias de recuperacion
2. Inicializar AWS DRS en la region destino
3. Instalar el agente de replicacion en las instancias origen (EC2 o on-premises)
4. Verificar que la replicacion inicial completa y la sincronizacion continua esta activa
5. Configurar los launch settings (tipo de instancia, red, security groups, tags)
6. Configurar la staging area (tipo de instancia de staging, cifrado)
7. Ejecutar un drill (prueba no disruptiva) y verificar que las instancias de prueba funcionan
8. Terminar las instancias del drill sin afectar la replicacion
9. Ejecutar un failover completo y verificar que las instancias de recuperacion sirven trafico
10. Ejecutar el proceso de failback para revertir la replicacion a la region original

## Validacion

- [ ] El agente de replicacion esta instalado y reportando estado saludable
- [ ] La replicacion inicial completa y la sincronizacion continua esta activa (lag < 1 segundo)
- [ ] Los launch settings estan configurados con la red y tipo de instancia correctos
- [ ] El drill lanza instancias de prueba funcionales sin interrumpir la replicacion
- [ ] Las instancias del drill se pueden terminar limpiamente
- [ ] El failover completo lanza instancias de recuperacion funcionales
- [ ] El failback inicia la replicacion inversa hacia la region original
- [ ] El RPO medido es de sub-segundo durante la replicacion continua
