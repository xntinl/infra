# 11.05 Budget Alerts & Anomaly Detection

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Intermedio |
| **Modalidad** | Ambos |
| **Servicios AWS** | AWS Budgets, Cost Anomaly Detection, SNS, Lambda |

## Descripcion

Configurar AWS Budgets con: presupuestos mensuales de costo por cuenta y servicio, presupuestos de uso para recursos especificos, presupuestos de utilizacion de reservas y monitores de deteccion de anomalias. Configurar alertas de multiples umbrales (50%, 80%, 100%) con acciones de SNS y Lambda.

## Objetivos de aprendizaje

- Disenar e implementar una estrategia de gestion de presupuestos en AWS
- Configurar deteccion de anomalias de costos para identificar gastos inesperados
- Implementar control proactivo de costos con alertas escalonadas
- Disenar flujos de alerta que escalen apropiadamente segun la severidad

## Requisitos previos

- **Conocimientos**: Modelo de costos de AWS, SNS topics y subscripciones, Lambda basics
- **Herramientas**: AWS CLI, Terraform
- **Servicios AWS**: Permisos para crear Budgets, Cost Anomaly Detection monitors, SNS topics y funciones Lambda

## Arquitectura

El sistema de alertas y deteccion de anomalias se compone de:

- **AWS Budgets**: Presupuestos configurados por cuenta, servicio y tipo de uso con umbrales multiples
- **Cost Anomaly Detection**: Monitores ML-based que detectan patrones de gasto inusuales automaticamente
- **SNS Topics**: Canales de notificacion diferenciados por severidad (informativo, advertencia, critico)
- **Lambda**: Funciones de accion automatica para responder a alertas criticas (ej: deshabilitar acceso a servicios costosos)
- **CloudWatch**: Dashboard para visualizar el estado de presupuestos y anomalias

## Pasos sugeridos

1. Definir la estrategia de presupuestos: presupuesto global mensual, por servicio (EC2, RDS, S3) y por cuenta
2. Crear presupuestos de costo con Terraform usando `aws_budgets_budget` con umbrales al 50%, 80% y 100%
3. Configurar SNS topics diferenciados por severidad y suscribir emails y/o canales de Slack
4. Crear presupuestos de uso para recursos especificos (horas de EC2, GB de almacenamiento S3)
5. Configurar un presupuesto de utilizacion de Savings Plans/Reserved Instances con alerta si cae por debajo del 80%
6. Habilitar Cost Anomaly Detection con monitores por servicio y por cuenta
7. Crear una funcion Lambda que se ejecute cuando el presupuesto alcance el 100% para tomar acciones automaticas
8. Probar las alertas simulando escenarios o ajustando temporalmente los umbrales
9. Crear un dashboard en CloudWatch que muestre el estado actual de todos los presupuestos

## Validacion

- [ ] Se crearon presupuestos mensuales de costo por cuenta y por servicio
- [ ] Las alertas se disparan correctamente en los umbrales de 50%, 80% y 100%
- [ ] Se configuraron SNS topics con suscriptores para cada nivel de severidad
- [ ] Se creo al menos un presupuesto de uso para un recurso especifico
- [ ] Cost Anomaly Detection esta habilitado con monitores configurados
- [ ] La funcion Lambda responde a alertas de presupuesto al 100%
- [ ] Se documento la estrategia de presupuestos y escalamiento de alertas
