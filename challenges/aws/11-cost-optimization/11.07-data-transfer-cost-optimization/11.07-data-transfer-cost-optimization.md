# 11.07 Data Transfer Cost Optimization

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Intermedio |
| **Modalidad** | Ambos |
| **Servicios AWS** | VPC Endpoints, NAT Gateway, CloudFront, S3, VPC Peering |

## Descripcion

Auditar los costos de data transfer en toda la arquitectura. Implementar optimizaciones: VPC endpoints para eliminar cargos de NAT Gateway, S3 gateway endpoints para acceso gratuito a S3, CloudFront para reducir costos de egress, y VPC peering para reducir trafico cross-region.

## Objetivos de aprendizaje

- Comprender el modelo de costos de data transfer en AWS y sus diferentes categorias
- Optimizar costos de NAT Gateway mediante VPC endpoints
- Evaluar la economia de los diferentes tipos de endpoints (gateway vs interface)
- Aprovechar CloudFront para reducir costos de egress a internet

## Requisitos previos

- **Conocimientos**: VPC networking, NAT Gateway, data transfer pricing, CloudFront basics
- **Herramientas**: AWS CLI, Terraform
- **Servicios AWS**: VPC con subnets privadas, NAT Gateway existente, bucket S3, permisos para crear VPC endpoints y distribuciones CloudFront

## Arquitectura

La optimizacion de data transfer aborda multiples puntos de la arquitectura:

- **VPC Gateway Endpoints**: Endpoints gratuitos para S3 y DynamoDB que eliminan el trafico a traves del NAT Gateway
- **VPC Interface Endpoints**: Endpoints para otros servicios AWS (ECR, CloudWatch, STS) que evitan el NAT Gateway
- **CloudFront**: CDN que reduce costos de egress al servir contenido desde edge locations con precios mas bajos
- **VPC Peering**: Conexion directa entre VPCs que reduce costos de trafico cross-region vs transito por internet
- **NAT Gateway**: El punto mas costoso de data transfer, objetivo principal de la optimizacion

## Pasos sugeridos

1. Analizar los costos de data transfer actuales en Cost Explorer, desglosando por tipo (inter-AZ, NAT Gateway, internet egress, cross-region)
2. Identificar el trafico que pasa por NAT Gateway usando VPC Flow Logs o metricas de CloudWatch
3. Crear un S3 Gateway Endpoint en la VPC para eliminar el trafico S3 a traves del NAT Gateway (gratuito)
4. Crear Interface Endpoints para los servicios AWS mas utilizados (ECR, CloudWatch Logs, STS, etc.)
5. Evaluar el costo de cada Interface Endpoint ($0.01/hr + data processing) vs el ahorro en NAT Gateway
6. Configurar una distribucion CloudFront frente a un bucket S3 o ALB para reducir el egress directo
7. Si existen VPCs en diferentes regiones, configurar VPC Peering para reducir costos de trafico cross-region
8. Implementar todos los cambios con Terraform de forma incremental
9. Medir el impacto en costos despues de 1 semana comparando con el periodo anterior

## Validacion

- [ ] Se auditaron y desglosaron los costos de data transfer por categoria
- [ ] Se creo un S3 Gateway Endpoint y se verifico que el trafico S3 ya no pasa por NAT Gateway
- [ ] Se crearon Interface Endpoints para al menos 2 servicios AWS frecuentes
- [ ] Se evaluo el ROI de cada Interface Endpoint creado
- [ ] Se configuro CloudFront para al menos un origen (S3 o ALB)
- [ ] Se documento el ahorro estimado o medido en cada optimizacion
