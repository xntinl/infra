# 11.01 Savings Plans Analysis & Purchase Strategy

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Intermedio |
| **Modalidad** | Manual (Cost Explorer + spreadsheet) |
| **Servicios AWS** | Cost Explorer, Savings Plans, Budgets |

## Descripcion

Analizar 90 dias de datos de Cost Explorer para identificar el uso de computo en estado estable. Modelar la cobertura de Compute Savings Plans vs EC2 Instance Savings Plans. Calcular los puntos de equilibrio (break-even) para terminos de 1 ano vs 3 anos. Crear una estrategia de compra que maximice los ahorros mientras minimiza el riesgo.

## Objetivos de aprendizaje

- Comprender el modelo de precios basado en compromisos (commitment-based pricing)
- Realizar analisis de cobertura de Savings Plans
- Modelar puntos de equilibrio (break-even) para diferentes terminos
- Evaluar riesgos asociados a compromisos de largo plazo
- Disenar una estrategia de compra escalonada

## Requisitos previos

- **Conocimientos**: Modelo de precios de AWS, tipos de instancias EC2, conceptos basicos de FinOps
- **Herramientas**: AWS CLI, acceso a Cost Explorer, hoja de calculo (Excel/Google Sheets)
- **Servicios AWS**: Cuenta con al menos 90 dias de historial de uso en Cost Explorer

## Arquitectura

La arquitectura de este desafio se centra en el analisis de datos de costos, no en infraestructura desplegada. Los componentes clave son:

- **Cost Explorer**: Fuente principal de datos de uso y costo historico de los ultimos 90 dias
- **Savings Plans Console**: Para evaluar recomendaciones y simular escenarios de cobertura
- **Budgets**: Para configurar alertas sobre la utilizacion de los Savings Plans adquiridos
- **Spreadsheet**: Modelo financiero donde se comparan escenarios On-Demand vs Compute SP vs EC2 Instance SP

## Pasos sugeridos

1. Acceder a Cost Explorer y exportar datos de costo y uso de los ultimos 90 dias, agrupados por servicio y tipo de instancia
2. Identificar el uso de computo en estado estable (baseline) separandolo de los picos temporales
3. Revisar las recomendaciones de Savings Plans en la consola de AWS para diferentes niveles de compromiso
4. Modelar en una hoja de calculo el costo On-Demand vs Compute Savings Plans vs EC2 Instance Savings Plans
5. Calcular el punto de equilibrio para terminos de 1 ano (No Upfront, Partial Upfront, All Upfront) y 3 anos
6. Evaluar la flexibilidad de cada tipo de plan: Compute SP permite cambiar familia de instancia, region y sistema operativo; EC2 Instance SP no
7. Disenar una estrategia de compra escalonada que cubra el baseline con compromisos y deje los picos para On-Demand o Spot
8. Configurar AWS Budgets con alertas de utilizacion de Savings Plans (umbral al 80% y 100%)
9. Documentar la estrategia final con justificacion de cada decision y proyeccion de ahorro anual

## Validacion

- [ ] Se calcularon los puntos de equilibrio para terminos de 1 y 3 anos
- [ ] Se documento la diferencia entre Compute SP y EC2 Instance SP
