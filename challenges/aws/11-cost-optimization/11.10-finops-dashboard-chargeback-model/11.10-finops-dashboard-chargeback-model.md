# 11.10 FinOps Dashboard & Chargeback Model

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Ambos |
| **Servicios AWS** | Cost Explorer, CUR, Athena, QuickSight, Lambda, Organizations |

## Descripcion

Construir un dashboard de FinOps a nivel organizacional: implementar cost allocation tags obligatorios, crear una Lambda de compliance de tags, construir queries de Athena basadas en el CUR para desglose de costos por equipo y por servicio, y crear un dashboard en QuickSight mostrando reportes de showback/chargeback.

## Objetivos de aprendizaje

- Aplicar practicas de FinOps en una organizacion multi-cuenta
- Disenar e implementar una estrategia de tagging para atribucion de costos
- Construir modelos de chargeback y showback basados en datos reales
- Crear visibilidad organizacional de costos con dashboards interactivos

## Requisitos previos

- **Conocimientos**: FinOps fundamentals, AWS Organizations, CUR schema, SQL intermedio, QuickSight basics
- **Herramientas**: AWS CLI, Terraform, SQL, QuickSight
- **Servicios AWS**: AWS Organizations (idealmente), CUR configurado, Athena, permisos para crear Lambda y QuickSight dashboards

## Arquitectura

El sistema de FinOps organizacional se compone de:

- **AWS Organizations**: Estructura multi-cuenta que agrupa equipos/proyectos
- **Cost Allocation Tags**: Tags obligatorios (team, project, environment, cost-center) activados en Billing
- **Tag Compliance Lambda**: Funcion que escanea recursos periodicamente y reporta/remedia tags faltantes
- **CUR + Athena**: Pipeline de datos de costos consultable con SQL para analisis detallado
- **QuickSight Dashboard**: Visualizacion interactiva con filtros por equipo, servicio, cuenta y periodo
- **SNS**: Notificaciones de compliance de tags y alertas de presupuesto por equipo

## Pasos sugeridos

1. Definir la estrategia de tagging: tags obligatorios (team, project, environment, cost-center) y opcionales
2. Activar los cost allocation tags en la consola de Billing & Cost Management
3. Crear una Tag Policy en AWS Organizations para enforcer los tags obligatorios
4. Desarrollar una Lambda que escanee recursos y reporte los que no cumplen con la politica de tags
5. Configurar la Lambda con EventBridge schedule (diario) y notificaciones SNS del reporte de compliance
6. Configurar el CUR con entrega a S3 si no esta hecho (ver challenge 11.03)
7. Crear queries de Athena para desglose de costos por: equipo, proyecto, servicio AWS y cuenta
8. Desarrollar queries para calcular costos compartidos (networking, soporte) y su distribucion proporcional
9. Conectar QuickSight a Athena y crear datasets con las queries preparadas
10. Construir el dashboard de QuickSight con: resumen ejecutivo, desglose por equipo, tendencias mensuales, top 10 recursos mas costosos y compliance de tags

## Validacion

- [ ] Los cost allocation tags obligatorios estan definidos y activados en Billing
- [ ] La Tag Policy de Organizations esta aplicada (si aplica)
- [ ] La Lambda de compliance escanea recursos y reporta tags faltantes
- [ ] El CUR esta configurado y consultable via Athena
- [ ] Se crearon queries para desglose de costos por equipo y servicio
- [ ] Se modelo la distribucion de costos compartidos
- [ ] El dashboard de QuickSight muestra datos reales con filtros interactivos
- [ ] Se documento el modelo de chargeback/showback con la metodologia de atribucion
