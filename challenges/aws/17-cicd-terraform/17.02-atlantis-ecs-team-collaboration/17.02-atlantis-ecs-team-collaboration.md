# 17.02 Atlantis en ECS para Colaboracion en Equipo

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |

## Descripcion

Desplegar Atlantis en ECS Fargate para workflows de Terraform en equipo. Configurar: `atlantis.yaml` a nivel de repositorio, plan/apply via comentarios en PR, soporte de workspaces, workflows personalizados para pre/post hooks, e integracion de webhooks.

## Objetivos de aprendizaje

- Disenar workflows de Terraform colaborativos con Atlantis
- Implementar applies basados en PR con control de acceso
- Dominar la configuracion de Atlantis (server-side y repo-side)
- Aplicar patrones de colaboracion para equipos de infraestructura

## Requisitos previos

- Conocimientos: ECS Fargate, Terraform intermedio, Git/PR workflows, webhooks
- Herramientas: Terraform CLI, AWS CLI, Docker, GitHub/GitLab con acceso a webhooks
- Servicios: ECS Fargate, ALB, Route 53, ACM, Secrets Manager

## Arquitectura

Atlantis corre como un servicio en ECS Fargate detras de un ALB con certificado TLS. GitHub envia webhooks al endpoint de Atlantis cuando se crean o actualizan PRs. Atlantis ejecuta `terraform plan` automaticamente y publica el resultado como comentario. Los desarrolladores pueden ejecutar `atlantis apply` via comentarios para aplicar los cambios.

```
GitHub Webhook -> ALB (HTTPS) -> ECS Fargate (Atlantis)
                                      |
                                      v
                               Terraform Plan/Apply
                                      |
                                      v
                                 AWS Resources
```

## Pasos sugeridos

1. Crear la infraestructura base: VPC, subnets, security groups para ECS
2. Configurar el servicio ECS Fargate con la imagen de Atlantis
3. Crear un ALB con listener HTTPS y certificado ACM
4. Almacenar el GitHub token y webhook secret en Secrets Manager
5. Configurar el webhook en GitHub apuntando al ALB
6. Crear el archivo `atlantis.yaml` en el repositorio con la configuracion de proyectos
7. Definir workflows personalizados con pre/post hooks (ej: `tflint`, `tfsec`)
8. Configurar soporte de workspaces para multiples entornos
9. Probar el flujo: crear PR, verificar plan automatico, aplicar via comentario
10. Configurar permisos y restricciones (quien puede hacer apply, branches permitidos)

## Validacion

- [ ] Atlantis corre como servicio estable en ECS Fargate
- [ ] Los PRs reciben automaticamente el output de `terraform plan`
- [ ] `atlantis apply` en comentarios ejecuta el apply correctamente
- [ ] Los workflows personalizados ejecutan pre/post hooks
- [ ] Multiples workspaces funcionan correctamente
- [ ] El webhook esta asegurado con secret validation
- [ ] Los logs de Atlantis son accesibles para debugging
