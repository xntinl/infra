# 17.04 Pre-commit Hooks para Terraform

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Intermedio |
| **Modalidad** | Terraform + pre-commit |
| **Features TF** | fmt, validate, docs generation |

## Descripcion

Configurar un sistema completo de pre-commit: `terraform fmt`, `terraform validate`, `tflint`, `tfsec`/`trivy`, `terraform-docs` para auto-generacion de documentacion, escaneo con `checkov`. Enforcer tanto en CI como en el flujo local del desarrollador.

## Objetivos de aprendizaje

- Automatizar la calidad de codigo en Terraform
- Implementar seguridad shift-left (detectar problemas lo antes posible)
- Configurar generacion automatica de documentacion con terraform-docs
- Integrar validaciones locales y de CI de forma consistente

## Requisitos previos

- Conocimientos: Terraform basico, Git hooks, linters y herramientas de analisis estatico
- Herramientas: pre-commit framework, tflint, trivy/tfsec, terraform-docs, checkov
- Features: `terraform fmt`, `terraform validate`

## Arquitectura

Los hooks se ejecutan en dos puntos: localmente antes de cada commit (via pre-commit framework) y en CI como paso de validacion en cada PR. Ambos usan la misma configuracion `.pre-commit-config.yaml` para garantizar consistencia.

```
Developer -> git commit -> pre-commit hooks
                              |
                   +----------+----------+
                   |          |          |
              terraform    tflint    trivy/tfsec
                fmt/validate          checkov
                   |          |          |
                   +----------+----------+
                              |
                        terraform-docs
                        (auto-update)
                              |
                        Commit (si todo pasa)
```

## Pasos sugeridos

1. Instalar el framework pre-commit (`pip install pre-commit` o `brew install pre-commit`)
2. Crear `.pre-commit-config.yaml` con hooks para `terraform fmt` y `terraform validate`
3. Agregar el hook de `tflint` con configuracion `.tflint.hcl`
4. Agregar hooks de seguridad: `tfsec`/`trivy` para escaneo de vulnerabilidades
5. Configurar `terraform-docs` para auto-generar documentacion en README entre marcadores
6. Agregar `checkov` como hook adicional de compliance
7. Ejecutar `pre-commit install` para activar los hooks localmente
8. Crear un workflow de CI que ejecute `pre-commit run --all-files`
9. Probar que cada hook detecta problemas y bloquea commits incorrectos

## Validacion

- [ ] `terraform fmt` reformatea automaticamente archivos mal formateados
- [ ] `terraform validate` detecta errores de sintaxis y configuracion
- [ ] `tflint` reporta best practices y errores comunes
- [ ] `trivy`/`tfsec` detecta vulnerabilidades de seguridad en el codigo
- [ ] `terraform-docs` actualiza automaticamente la documentacion
- [ ] `checkov` escanea contra benchmarks de seguridad
- [ ] El CI ejecuta los mismos hooks y falla si hay problemas
- [ ] Los desarrolladores pueden ejecutar `pre-commit run --all-files` localmente
