# 17.08 CI/CD con Workspaces Dinamicos

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform + GitHub Actions |
| **Features TF** | Workspaces, dynamic discovery |

## Descripcion

Construir un pipeline que descubra dinamicamente los workspaces de Terraform (a partir de la estructura de directorios o configuracion YAML) y los cree/gestione automaticamente. Nuevos directorios obtienen automaticamente la creacion de workspace y jobs de plan/apply.

## Objetivos de aprendizaje

- Implementar generacion dinamica de pipelines basada en la estructura del proyecto
- Gestionar workspaces de Terraform de forma automatizada
- Disenar CI/CD escalable que crece con la infraestructura
- Aplicar patrones de descubrimiento automatico

## Requisitos previos

- Conocimientos: Terraform workspaces, GitHub Actions (matrices dinamicas, outputs entre jobs), scripting bash/python
- Herramientas: Terraform CLI, GitHub Actions, jq o yq para parsing
- Features: Terraform workspaces, GitHub Actions `fromJSON()` para matrices dinamicas

## Arquitectura

Un job inicial escanea el repositorio para descubrir proyectos/workspaces (via estructura de directorios o archivo YAML). Genera una matrix dinamica que alimenta los jobs de plan/apply. Al agregar un nuevo directorio, el pipeline lo detecta automaticamente.

```
Discovery Job
  |
  +-- Escanear directorios / leer config YAML
  +-- Generar lista de workspaces
  +-- Output: JSON array para matrix
  |
  v
Matrix Jobs (dinamicos)
  |
  +-- Job por workspace descubierto
  +-- terraform workspace select/create
  +-- terraform plan / apply
```

## Pasos sugeridos

1. Definir la convencion: un directorio = un workspace (ej: `environments/dev/`, `environments/prod/`)
2. Crear un script que descubra los directorios con configuracion Terraform
3. Alternativamente, crear un archivo `workspaces.yaml` con la lista de workspaces y sus parametros
4. Implementar el job de discovery que genera un output JSON con los workspaces encontrados
5. Usar `fromJSON()` en la matrix strategy para crear jobs dinamicos
6. En cada job: seleccionar o crear el workspace con `terraform workspace select || terraform workspace new`
7. Ejecutar `terraform init`, `plan` y `apply` en el contexto del workspace
8. Implementar logica de deteccion de cambios: solo planificar workspaces con archivos modificados
9. Probar: agregar un nuevo directorio y verificar que el pipeline lo detecta automaticamente
10. Agregar manejo de eliminacion de workspaces cuando se borra un directorio

## Validacion

- [ ] El job de discovery encuentra todos los directorios/workspaces existentes
- [ ] La matrix se genera dinamicamente basada en el descubrimiento
- [ ] Nuevos directorios son detectados automaticamente en el siguiente run
- [ ] Cada workspace se crea si no existe
- [ ] Plan y apply se ejecutan correctamente por workspace
- [ ] Solo se planifican los workspaces con cambios (optimizacion)
- [ ] El pipeline escala sin necesidad de modificar la configuracion de CI
