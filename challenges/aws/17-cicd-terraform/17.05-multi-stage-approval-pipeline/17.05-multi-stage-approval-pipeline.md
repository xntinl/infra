# 17.05 Pipeline Multi-Stage con Aprobacion

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform + GitHub Actions/CodePipeline |
| **Features TF** | Plan JSON output, `-out` flag |

## Descripcion

Construir un pipeline con gates de aprobacion manual: plan -> escaneo de seguridad -> estimacion de costos (Infracost) -> aprobacion manual -> apply. Cada etapa produce artefactos. Implementar notificaciones Slack con botones de aprobar/rechazar.

## Objetivos de aprendizaje

- Disenar workflows de aprobacion para cambios de infraestructura
- Trabajar con artefactos de plan de Terraform (archivos binarios y JSON)
- Integrar estimacion de costos con Infracost
- Implementar notificaciones interactivas con Slack

## Requisitos previos

- Conocimientos: GitHub Actions (environments, approvals), Terraform plan/apply, integraciones de Slack
- Herramientas: Terraform CLI, Infracost, GitHub Actions, Slack (webhook o app)
- Features: `terraform plan -out`, `terraform show -json`, GitHub Environments

## Arquitectura

El pipeline se compone de etapas secuenciales, cada una produciendo artefactos que la siguiente consume. La aprobacion manual actua como gate entre la fase de validacion y la de apply.

```
Plan -> Artefacto (.tfplan)
  |
  v
Security Scan (trivy/tfsec sobre plan JSON)
  |
  v
Cost Estimate (Infracost sobre plan JSON)
  |
  v
Slack Notification (resumen + botones aprobar/rechazar)
  |
  v
Manual Approval (GitHub Environment protection rules)
  |
  v
Apply (consume artefacto .tfplan)
```

## Pasos sugeridos

1. Crear el job de plan con `terraform plan -out=tfplan` y subir el artefacto
2. Agregar step para generar la version JSON del plan: `terraform show -json tfplan`
3. Crear job de security scan que descargue el artefacto y ejecute trivy/tfsec
4. Configurar Infracost para generar estimacion de costos a partir del plan JSON
5. Implementar notificacion a Slack con el resumen del plan, resultados de seguridad y costo estimado
6. Configurar GitHub Environment con reglas de proteccion (reviewers requeridos)
7. Crear job de apply que requiera aprobacion del environment y consuma el artefacto de plan
8. Agregar botones interactivos en Slack para aprobar/rechazar (via Slack app o workflow)
9. Probar el flujo completo de extremo a extremo
10. Verificar que el apply usa exactamente el plan aprobado (no regenera)

## Validacion

- [ ] El plan genera un artefacto binario que se preserva entre jobs
- [ ] El escaneo de seguridad analiza el plan y reporta findings
- [ ] Infracost produce una estimacion de costos legible
- [ ] La notificacion de Slack incluye resumen del plan y costo
- [ ] El apply requiere aprobacion manual antes de ejecutarse
- [ ] El apply usa el artefacto de plan exacto (no regenera el plan)
- [ ] Si se rechaza, el pipeline se detiene sin aplicar cambios
- [ ] Los artefactos se preservan para auditoria
