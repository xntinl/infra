# 17.03 HCP Terraform con Sentinel Policies

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform + HCP Terraform |
| **Features TF** | Sentinel, policy sets, workspace VCS |

## Descripcion

Configurar HCP Terraform (anteriormente Terraform Cloud) con workspaces, integracion VCS y policy sets de Sentinel. Escribir politicas Sentinel que apliquen etiquetado obligatorio, restrinjan tipos de instancia y requieran encriptacion. Configurar niveles de politica (advisory, soft-mandatory, hard-mandatory).

## Objetivos de aprendizaje

- Dominar las funcionalidades de HCP Terraform (workspaces, VCS, runs)
- Aprender el lenguaje Sentinel para escribir politicas de infraestructura
- Comprender los niveles de enforcement de politicas
- Implementar policy-as-code en un flujo de Terraform gestionado

## Requisitos previos

- Conocimientos: Terraform intermedio, conceptos de policy-as-code, control de versiones
- Herramientas: HCP Terraform (cuenta gratuita o de pago), Sentinel CLI, Git
- Features: Sentinel policies, policy sets, workspaces con VCS

## Arquitectura

HCP Terraform gestiona el estado remoto y la ejecucion de runs. Los workspaces estan conectados a repositorios VCS y ejecutan plan/apply automaticamente al detectar cambios. Los policy sets de Sentinel se evaluan entre el plan y el apply, bloqueando cambios que no cumplan las reglas definidas.

```
VCS Push -> HCP Terraform Workspace -> Plan
                                         |
                                    Sentinel Policies
                                    (advisory / soft / hard)
                                         |
                                       Apply (si pasa)
```

## Pasos sugeridos

1. Crear una organizacion en HCP Terraform y conectarla al proveedor VCS
2. Crear workspaces vinculados a repositorios con configuracion de Terraform
3. Configurar variables de entorno (credenciales AWS) en los workspaces
4. Crear un repositorio de politicas Sentinel con la estructura requerida
5. Escribir politica de etiquetado obligatorio (`mandatory-tags.sentinel`)
6. Escribir politica de restriccion de tipos de instancia (`allowed-instance-types.sentinel`)
7. Escribir politica de encriptacion requerida (`require-encryption.sentinel`)
8. Configurar policy sets en HCP Terraform con los niveles apropiados
9. Probar cada politica: verificar que bloquea recursos no conformes
10. Verificar los diferentes niveles: advisory (aviso), soft-mandatory (override posible), hard-mandatory (sin override)

## Validacion

- [ ] Los workspaces estan conectados a VCS y ejecutan runs automaticamente
- [ ] La politica de etiquetado bloquea recursos sin tags requeridos
- [ ] La politica de instancias restringe a los tipos permitidos
- [ ] La politica de encriptacion requiere EBS/S3 encriptados
- [ ] Los niveles advisory, soft-mandatory y hard-mandatory funcionan correctamente
- [ ] Las politicas tienen tests unitarios con Sentinel CLI
- [ ] Los policy sets se aplican a los workspaces correctos
