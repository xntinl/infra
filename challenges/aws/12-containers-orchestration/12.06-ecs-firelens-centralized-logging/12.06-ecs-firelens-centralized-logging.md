# 12.06 ECS FireLens & Centralized Logging

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Intermedio |
| **Modalidad** | Ambos |
| **Servicios AWS** | ECS, FireLens/Fluent Bit, CloudWatch, S3, OpenSearch |

## Descripcion

Configurar tasks de ECS con FireLens (sidecar Fluent Bit) para enrutamiento avanzado de logs. Enrutar logs a multiples destinos: CloudWatch para logs operacionales, S3 para archivado y OpenSearch para busqueda. Implementar filtrado, parseo y enriquecimiento de logs.

## Objetivos de aprendizaje

- Disenar una arquitectura de logging centralizado para contenedores
- Configurar Fluent Bit como sidecar via FireLens en ECS
- Implementar enrutamiento de logs a multiples destinos simultaneamente
- Aplicar parseo y enriquecimiento de logs en el pipeline de ingesta

## Requisitos previos

- **Conocimientos**: ECS task definitions, Fluent Bit basics, formatos de log (JSON, Apache), CloudWatch Logs
- **Herramientas**: AWS CLI, Terraform, Docker
- **Servicios AWS**: ECS cluster, permisos para crear CloudWatch Log Groups, buckets S3, dominio OpenSearch (opcional)

## Arquitectura

La arquitectura de logging centralizado se compone de:

- **ECS Tasks**: Contenedores de aplicacion que generan logs a stdout/stderr
- **FireLens (Fluent Bit)**: Sidecar container que intercepta los logs del contenedor de aplicacion
- **CloudWatch Logs**: Destino para logs operacionales con retention policy y metric filters
- **S3**: Destino para archivado de largo plazo con lifecycle policies (Glacier despues de 90 dias)
- **OpenSearch**: Destino para busqueda y analisis interactivo de logs (con Kibana/OpenSearch Dashboards)
- **Fluent Bit Config**: Configuracion personalizada con filters (parsers, enrichment) y outputs (multi-destino)

## Pasos sugeridos

1. Crear un cluster ECS y un servicio de aplicacion simple que genere logs estructurados (JSON)
2. Modificar la task definition para agregar el sidecar FireLens con la imagen de Fluent Bit de AWS
3. Configurar el log driver de la aplicacion como `awsfirelens` en lugar de `awslogs`
4. Crear una configuracion custom de Fluent Bit para enrutar logs a CloudWatch Logs
5. Agregar un segundo output en la configuracion para enviar logs a S3 en formato Parquet o JSON comprimido
6. Implementar un filter de Fluent Bit para parsear logs no estructurados (ej: Apache access logs) a JSON
7. Agregar enrichment: incluir metadata como cluster name, service name y task ID en cada log entry
8. Configurar un tercer output hacia OpenSearch Service para logs que requieran busqueda interactiva
9. Implementar filtrado: enviar solo logs de nivel ERROR a OpenSearch, todos los niveles a S3
10. Validar que los logs llegan correctamente a los 3 destinos y que el parseo/filtrado funciona

## Validacion

- [ ] FireLens (Fluent Bit) esta configurado como sidecar en la task definition
- [ ] Los logs de la aplicacion se enrutan a CloudWatch Logs correctamente
- [ ] Los logs se archivan en S3 con el formato y particionado esperado
- [ ] El parseo de logs no estructurados a JSON funciona correctamente
- [ ] Los logs estan enriquecidos con metadata del cluster, servicio y task
- [ ] El filtrado por nivel de log funciona (ej: solo ERROR a OpenSearch)
- [ ] Los logs son buscables en OpenSearch/Kibana (si aplica)
- [ ] La configuracion de Fluent Bit esta externalizada y es mantenible
