# 12.03 ECR Image Lifecycle & Cross-Account Sharing

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Intermedio |
| **Modalidad** | Ambos |
| **Servicios AWS** | ECR, Inspector, IAM, EventBridge |

## Descripcion

Configurar ECR con: politicas de lifecycle de imagenes (mantener las ultimas N imagenes taggeadas, expirar imagenes sin tag despues de X dias), escaneo de vulnerabilidades (basico y enhanced con Inspector), replicacion cross-account para DR, y pull-through cache para imagenes publicas.

## Objetivos de aprendizaje

- Gestionar un container registry empresarial con politicas de lifecycle
- Implementar escaneo de vulnerabilidades en imagenes de contenedores
- Configurar patrones cross-account para compartir imagenes de forma segura
- Utilizar pull-through cache para optimizar el acceso a imagenes publicas

## Requisitos previos

- **Conocimientos**: Docker, ECR basics, IAM policies, Inspector
- **Herramientas**: AWS CLI, Terraform, Docker
- **Servicios AWS**: ECR, una segunda cuenta AWS (para cross-account) o al menos una segunda region

## Arquitectura

La solucion de gestion de ECR incluye:

- **ECR Repositories**: Repositorios privados con politicas de lifecycle configuradas
- **Lifecycle Policies**: Reglas automaticas que eliminan imagenes obsoletas (sin tag, antiguas)
- **ECR Scanning**: Escaneo basico (Clair) y enhanced (Inspector) para detectar vulnerabilidades
- **ECR Replication**: Replicacion automatica cross-account y cross-region para DR
- **Pull-Through Cache**: Cache local de imagenes publicas (Docker Hub, ECR Public, Quay) para reducir rate limits
- **EventBridge**: Eventos de scan completado para notificaciones automaticas de vulnerabilidades criticas

## Pasos sugeridos

1. Crear repositorios ECR via Terraform con configuracion de image tag immutability y encryption
2. Configurar lifecycle policies: mantener las ultimas 10 imagenes taggeadas, expirar imagenes untagged despues de 7 dias
3. Publicar varias imagenes de prueba con diferentes tags para validar las lifecycle policies
4. Habilitar el escaneo basico (scan on push) en los repositorios
5. Habilitar el escaneo enhanced con Amazon Inspector para obtener analisis continuo de vulnerabilidades
6. Configurar una regla de EventBridge que notifique via SNS cuando se detecten vulnerabilidades CRITICAL o HIGH
7. Configurar replicacion cross-region (y cross-account si se dispone de segunda cuenta) para los repositorios criticos
8. Configurar pull-through cache rules para Docker Hub y ECR Public Gallery
9. Validar que las imagenes publicas se cachean correctamente en el repositorio local
10. Documentar las politicas implementadas y el flujo de gestion de imagenes

## Validacion

- [ ] Los repositorios ECR estan creados con image tag immutability habilitado
- [ ] Las lifecycle policies eliminan imagenes sin tag despues de 7 dias
- [ ] Las lifecycle policies mantienen solo las ultimas N imagenes taggeadas
- [ ] El escaneo basico (scan on push) esta habilitado y reporta vulnerabilidades
- [ ] El escaneo enhanced con Inspector esta activo (si aplica)
- [ ] Las notificaciones de vulnerabilidades criticas se envian via EventBridge + SNS
- [ ] La replicacion cross-region funciona correctamente
- [ ] El pull-through cache descarga y cachea imagenes publicas
