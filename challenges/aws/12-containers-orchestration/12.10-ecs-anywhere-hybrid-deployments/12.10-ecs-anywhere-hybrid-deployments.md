# 12.10 ECS Anywhere & Hybrid Deployments

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Ambos |
| **Servicios AWS** | ECS Anywhere, SSM, EC2, ECR, CloudWatch |

## Descripcion

Configurar ECS Anywhere para gestionar contenedores en un servidor on-premises (simulado con una instancia EC2 en una VPC separada). Registrar la instancia externa con SSM, desplegar ECS tasks en ella, e implementar un pipeline de despliegue que apunte tanto a Fargate como a external launch types.

## Objetivos de aprendizaje

- Comprender la gestion hibrida de contenedores con ECS Anywhere
- Registrar y gestionar instancias externas con SSM
- Gestionar el ciclo de vida de instancias externas en ECS
- Implementar orquestacion unificada de contenedores entre cloud y on-premises

## Requisitos previos

- **Conocimientos**: ECS fundamentals, SSM Agent, networking (conectividad entre VPCs o instancias externas y AWS)
- **Herramientas**: AWS CLI, Terraform, Docker, acceso SSH a la instancia "on-premises"
- **Servicios AWS**: ECS cluster, SSM, permisos para registrar instancias externas, ECR para imagenes

## Arquitectura

La arquitectura hibrida con ECS Anywhere incluye:

- **ECS Cluster**: Cluster unificado que gestiona tanto tasks Fargate como tasks en instancias externas
- **Instancia "On-Premises"**: EC2 en una VPC separada (simulando un data center) con ECS Agent y SSM Agent
- **SSM**: Systems Manager para registrar y gestionar la instancia externa de forma segura
- **ECR**: Registro de imagenes accesible tanto desde Fargate como desde la instancia externa
- **ECS Services**: Servicios con launch type EXTERNAL para la instancia on-premises y FARGATE para cloud
- **CloudWatch**: Monitoreo unificado de metricas y logs de ambos entornos
- **Pipeline de Despliegue**: Pipeline que despliega la misma aplicacion a ambos launch types

## Pasos sugeridos

1. Crear una VPC separada (simulando el data center on-premises) con una instancia EC2 (Amazon Linux 2)
2. Crear el cluster ECS que soportara tanto Fargate como external instances
3. Generar el activation key y activation code para ECS Anywhere via la consola o CLI
4. Instalar SSM Agent en la instancia "on-premises" y registrarla como managed instance
5. Instalar Docker y el ECS Agent en la instancia, configurandolo con el activation key
6. Verificar que la instancia aparece como EXTERNAL en el cluster ECS
7. Crear una task definition compatible con el launch type EXTERNAL (considerar limitaciones vs Fargate)
8. Desplegar un servicio ECS con launch type EXTERNAL en la instancia on-premises
9. Desplegar el mismo servicio con launch type FARGATE en el cluster para comparar
10. Implementar un pipeline simple (CodePipeline o script) que despliegue actualizaciones a ambos launch types
11. Configurar CloudWatch para recolectar metricas y logs de ambos entornos de forma unificada
12. Documentar las diferencias operacionales entre FARGATE y EXTERNAL launch types

## Validacion

- [ ] La instancia "on-premises" esta registrada en SSM como managed instance
- [ ] La instancia aparece como EXTERNAL en el cluster ECS
- [ ] Se desplegaron tasks con launch type EXTERNAL en la instancia on-premises
- [ ] Se desplegaron tasks con launch type FARGATE en el mismo cluster
- [ ] Los contenedores en ambos entornos ejecutan la misma imagen de ECR
- [ ] El pipeline despliega actualizaciones a ambos launch types
- [ ] CloudWatch recolecta metricas y logs de ambos entornos
- [ ] Se documentaron las diferencias y limitaciones del launch type EXTERNAL
