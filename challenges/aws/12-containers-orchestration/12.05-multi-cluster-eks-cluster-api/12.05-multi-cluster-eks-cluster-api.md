# 12.05 Multi-Cluster EKS con Cluster API

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Experto |
| **Modalidad** | Terraform + Kubernetes |
| **Servicios AWS** | EKS, EC2, IAM, CAPA, ArgoCD |

## Descripcion

Desplegar multiples clusters EKS (dev, staging, prod) usando Cluster API Provider for AWS (CAPA). Implementar gestion de flota, observabilidad compartida y configuracion centralizada. Desplegar workloads usando ArgoCD ApplicationSets a traves de todos los clusters.

## Objetivos de aprendizaje

- Gestionar una flota de clusters Kubernetes con Cluster API
- Implementar Cluster API Provider for AWS (CAPA) para aprovisionar clusters EKS de forma declarativa
- Aplicar GitOps a escala con ArgoCD ApplicationSets para multiples clusters
- Disenar patrones multi-cluster para separacion de entornos

## Requisitos previos

- **Conocimientos**: Kubernetes avanzado, EKS, Cluster API concepts, GitOps con ArgoCD, IAM cross-account
- **Herramientas**: AWS CLI, Terraform, kubectl, clusterctl, Helm, ArgoCD CLI
- **Servicios AWS**: Permisos para crear multiples clusters EKS, VPCs, roles IAM. Idealmente multiples cuentas AWS

## Arquitectura

La arquitectura multi-cluster incluye:

- **Management Cluster**: Cluster EKS central que ejecuta Cluster API controllers y ArgoCD
- **Workload Clusters**: Clusters EKS para dev, staging y prod aprovisionados via CAPA
- **CAPA Controllers**: Controllers de Cluster API que gestionan el ciclo de vida de los clusters AWS
- **ArgoCD**: Servidor GitOps centralizado en el management cluster que despliega a todos los clusters
- **ApplicationSets**: Generadores de ArgoCD que crean Applications automaticamente para cada cluster
- **Observabilidad Compartida**: Metricas y logs centralizados de todos los clusters en un stack de observabilidad comun

## Pasos sugeridos

1. Crear el management cluster EKS con Terraform, incluyendo los permisos IAM necesarios para CAPA
2. Instalar clusterctl y los providers de Cluster API (core, bootstrap, control-plane, infrastructure-aws)
3. Configurar los credentials de CAPA para poder aprovisionar clusters EKS
4. Crear los manifiestos de Cluster API para el cluster de dev (AWSManagedCluster, EKSConfig, MachinePool)
5. Aplicar los manifiestos y verificar que CAPA aprovisiona el cluster de dev automaticamente
6. Repetir para staging y prod con configuraciones diferenciadas (tamano de nodos, numero de replicas)
7. Instalar ArgoCD en el management cluster via Helm
8. Registrar todos los workload clusters como destinos en ArgoCD
9. Crear un ApplicationSet con un cluster generator que despliegue una aplicacion de ejemplo a todos los clusters
10. Configurar observabilidad compartida: instalar Prometheus + Grafana centralizados con federation o remote write
11. Implementar un flujo de promocion: deploy a dev -> validar -> promover a staging -> validar -> promover a prod
12. Documentar los procedimientos de operaciones: agregar cluster, rotar credenciales, upgrade de version

## Validacion

- [ ] El management cluster esta funcionando con CAPA controllers instalados
- [ ] Al menos 2 workload clusters (dev, staging) fueron aprovisionados via Cluster API
- [ ] Los clusters se pueden gestionar declarativamente modificando los manifiestos de CAPA
- [ ] ArgoCD esta instalado y tiene todos los clusters registrados
- [ ] Un ApplicationSet despliega automaticamente a todos los clusters
- [ ] La observabilidad centralizada muestra metricas de todos los clusters
- [ ] El flujo de promocion entre entornos funciona correctamente
- [ ] Se puede agregar un nuevo cluster modificando solo manifiestos declarativos
