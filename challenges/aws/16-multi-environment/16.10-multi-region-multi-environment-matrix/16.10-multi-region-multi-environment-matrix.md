# 16.10 Matriz Multi-Region x Multi-Entorno

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Experto |
| **Modalidad** | Terraform + Terragrunt o GitHub Actions matrix |
| **Features TF** | Provider aliases, workspaces, matrix deployment |

## Descripcion

Despliega infraestructura a traves de una matriz de 3 entornos x 2 regiones (6 despliegues totales). Maneja recursos region-specific (AMIs, conteo de AZs), sizing por entorno, y recursos globales compartidos. Construye un pipeline de CI/CD que gestione los 6 despliegues.

## Objetivos de aprendizaje

- Disenar estrategias de despliegue matricial (region x entorno)
- Manejar diferencias regionales en la configuracion de infraestructura
- Mantener consistencia entre entornos a escala
- Construir pipelines que orquesten despliegues complejos

## Requisitos previos

- Conocimientos: Terraform avanzado, multi-region, provider aliases, CI/CD
- Herramientas: Terraform >= 1.5, Terragrunt (opcional), GitHub Actions, AWS CLI
- Features/Servicios: EC2, RDS, ALB, Route 53 (global), VPC, IAM, S3

## Arquitectura

```
Matriz de Despliegues (3 entornos x 2 regiones = 6):

                us-east-1           eu-west-1
  dev          dev-us-east-1       dev-eu-west-1
  staging      staging-us-east-1   staging-eu-west-1
  prod         prod-us-east-1      prod-eu-west-1

Recursos Regionales (por cada celda de la matriz):
  +-- VPC + Subnets (AZs varian por region)
  +-- EC2/ASG (AMI diferente por region)
  +-- RDS (sizing por entorno)
  +-- ALB

Recursos Globales (compartidos):
  +-- Route 53 Hosted Zone
  +-- IAM Roles (globales)
  +-- S3 Buckets de artefactos
  +-- CloudFront (opcional)

Estructura con Terragrunt:
  infra-live/
    +-- global/                    (recursos globales)
    +-- us-east-1/
    |     +-- dev/ staging/ prod/  (3 entornos)
    +-- eu-west-1/
          +-- dev/ staging/ prod/  (3 entornos)

Pipeline CI/CD (GitHub Actions matrix):
  strategy:
    matrix:
      environment: [dev, staging, prod]
      region: [us-east-1, eu-west-1]
```

## Pasos sugeridos

1. Disenar la estructura de directorios para la matriz region x entorno
2. Crear un modulo de Terraform parametrizable por region y entorno
3. Implementar descubrimiento automatico de AMIs por region usando data sources
4. Manejar el conteo variable de AZs por region (ej: us-east-1 tiene 6, eu-west-1 tiene 3)
5. Crear la capa de recursos globales (Route 53, IAM) independiente de las regiones
6. Implementar Route 53 con latency-based routing para dirigir trafico a la region mas cercana
7. Configurar backends de state independientes para cada celda de la matriz
8. Crear el pipeline de CI/CD con GitHub Actions matrix strategy para los 6 despliegues
9. Implementar el orden de ejecucion: globales primero, luego dev en paralelo, staging, y finalmente prod
10. Agregar validaciones de consistencia entre regiones del mismo entorno

## Validacion

- [ ] Los 6 despliegues (3 entornos x 2 regiones) se crean correctamente
- [ ] Cada despliegue usa la AMI correcta para su region
- [ ] El conteo de AZs se adapta a la region automaticamente
- [ ] El sizing de recursos varia segun el entorno (dev < staging < prod)
- [ ] Los recursos globales se crean una sola vez y son compartidos
- [ ] Route 53 dirige trafico a la region con menor latencia
- [ ] El state de cada celda de la matriz esta aislado
- [ ] El pipeline de CI/CD ejecuta los 6 despliegues con el orden correcto
- [ ] Se valida consistencia entre regiones del mismo entorno
- [ ] El rollback es posible por celda individual sin afectar las demas
