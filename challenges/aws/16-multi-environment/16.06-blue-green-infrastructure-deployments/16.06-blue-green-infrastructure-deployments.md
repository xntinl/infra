# 16.06 Despliegues Blue-Green a Nivel de Infraestructura

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform |
| **Features TF** | Workspaces, Route 53, conditional logic |

## Descripcion

Implementa blue-green a nivel de infraestructura: despliega un nuevo entorno "green" junto al "blue" existente, valida, y luego cambia el DNS de Route 53. Usa Terraform workspaces o convenciones de nombrado para gestionar ambos. Implementa rollback cambiando de vuelta a blue.

## Objetivos de aprendizaje

- Implementar el patron blue-green a nivel de infraestructura completa
- Comprender el cambio de trafico via DNS con Route 53
- Disenar patrones de rollback para infraestructura
- Lograr cambios de infraestructura con zero-downtime

## Requisitos previos

- Conocimientos: Terraform intermedio, Route 53, DNS, ALB, EC2/Lambda
- Herramientas: Terraform >= 1.5, AWS CLI, dominio registrado o zona de Route 53
- Features/Servicios: Route 53, ALB, EC2 o Lambda, VPC

## Arquitectura

```
Estado Inicial (Blue activo):
  Route 53 (app.example.com)
    -> ALB Blue (10.0.1.x)
         -> EC2 instances Blue (v1)
    [ALB Green no existe aun]

Despliegue Green:
  Route 53 (app.example.com)
    -> ALB Blue (10.0.1.x) [ACTIVO - peso 100]
         -> EC2 instances Blue (v1)
    -> ALB Green (10.0.2.x) [STANDBY - peso 0]
         -> EC2 instances Green (v2)

Switch a Green:
  Route 53 (app.example.com)
    -> ALB Blue (10.0.1.x) [STANDBY - peso 0]
    -> ALB Green (10.0.2.x) [ACTIVO - peso 100]

Terraform:
  +-- variable "active_environment" = "blue" | "green"
  +-- aws_route53_record con weighted routing
  +-- Modulo "stack" instanciado para blue y green
```

## Pasos sugeridos

1. Crear un modulo de stack que despliegue ALB + EC2 (o Lambda) con un parametro de color (blue/green)
2. Instanciar el modulo para "blue" con la version actual de la aplicacion
3. Configurar Route 53 con un weighted routing policy apuntando al ALB blue (peso 100)
4. Crear la variable `active_environment` que controle cual entorno recibe trafico
5. Instanciar el modulo para "green" con la nueva version de la aplicacion
6. Agregar un segundo record Route 53 weighted para green (inicialmente peso 0)
7. Implementar la logica condicional: si `active_environment == "green"`, green peso 100, blue peso 0
8. Probar el switch cambiando la variable y ejecutando `terraform apply`
9. Implementar rollback: cambiar `active_environment` de vuelta a "blue"
10. Agregar un paso de limpieza que destruya el entorno inactivo tras validacion

## Validacion

- [ ] Ambos entornos (blue y green) coexisten simultaneamente
- [ ] Route 53 dirige todo el trafico al entorno activo
- [ ] Cambiar `active_environment` redirige el trafico al otro entorno
- [ ] El cambio de DNS se propaga correctamente (verificar con `dig`)
- [ ] El rollback funciona cambiando la variable de vuelta
- [ ] No hay downtime durante el switch (las peticiones siempre tienen destino)
- [ ] El entorno inactivo puede destruirse de forma segura
- [ ] El patron es reproducible para futuros despliegues
