# 16.03 Terragrunt: Configuracion DRY Multi-Entorno

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform + Terragrunt |
| **Features TF** | Terragrunt include, dependency, generate blocks |

## Descripcion

Reestructura un proyecto con Terragrunt: `terragrunt.hcl` en cada nivel (account/region/env/stack), bloques `include` para configuracion compartida, bloques `dependency` para referencias inter-stack. Implementa `run-all` para aplicar entornos completos.

## Objetivos de aprendizaje

- Dominar los patrones de Terragrunt para configuracion DRY
- Comprender la orquestacion de dependencias entre stacks
- Implementar jerarquias de configuracion con `include` y `read_terragrunt_config()`
- Usar `run-all` para gestionar entornos completos de forma eficiente

## Requisitos previos

- Conocimientos: Terraform intermedio, estructura de proyectos, backends
- Herramientas: Terraform >= 1.5, Terragrunt >= 0.55, AWS CLI, cuenta AWS
- Features/Servicios: S3 (backend), DynamoDB (lock), VPC, EC2, RDS

## Arquitectura

```
infra-live/
  +-- terragrunt.hcl                    (root: backend config, provider generation)
  +-- account.hcl                       (account-level vars)
  +-- us-east-1/
  |     +-- region.hcl                  (region-level vars)
  |     +-- dev/
  |     |     +-- env.hcl              (env-level vars)
  |     |     +-- vpc/
  |     |     |     +-- terragrunt.hcl (include + inputs)
  |     |     +-- database/
  |     |     |     +-- terragrunt.hcl (include + dependency on vpc)
  |     |     +-- app/
  |     |           +-- terragrunt.hcl (include + dependency on vpc, database)
  |     +-- staging/
  |     |     +-- env.hcl
  |     |     +-- vpc/ database/ app/
  |     +-- prod/
  |           +-- env.hcl
  |           +-- vpc/ database/ app/

infra-modules/
  +-- vpc/         (modulo Terraform reutilizable)
  +-- database/
  +-- app/
```

## Pasos sugeridos

1. Crear la estructura de directorios con la jerarquia account/region/env/stack
2. Escribir el `terragrunt.hcl` raiz con la configuracion de backend remoto (S3 + DynamoDB) usando `generate`
3. Crear `account.hcl`, `region.hcl` y `env.hcl` con variables a cada nivel
4. Implementar `include` en cada stack para heredar configuracion del root
5. Usar `read_terragrunt_config()` para leer variables de los archivos de nivel superior
6. Crear los modulos de Terraform (vpc, database, app) en un directorio separado
7. Configurar `dependency` blocks para que database dependa de vpc y app dependa de ambos
8. Usar `dependency.vpc.outputs.vpc_id` para pasar outputs entre stacks
9. Probar `terragrunt run-all plan` para planificar un entorno completo respetando dependencias
10. Probar `terragrunt run-all apply` y verificar el orden de ejecucion

## Validacion

- [ ] La estructura de directorios sigue la jerarquia account/region/env/stack
- [ ] El backend se configura automaticamente con `generate` desde el root
- [ ] `include` hereda configuracion del root sin duplicar codigo
- [ ] Las variables de cada nivel se leen correctamente con `read_terragrunt_config()`
- [ ] `dependency` establece el orden correcto entre stacks
- [ ] Los outputs de un stack se pasan como inputs a otro via `dependency`
- [ ] `run-all plan` muestra planes para todos los stacks en orden correcto
- [ ] `run-all apply` crea la infraestructura completa respetando dependencias
- [ ] Cada entorno (dev, staging, prod) es independiente pero comparte modulos
