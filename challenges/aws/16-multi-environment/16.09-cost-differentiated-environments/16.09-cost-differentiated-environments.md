# 16.09 Entornos Diferenciados por Costo

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Intermedio |
| **Modalidad** | Terraform |
| **Features TF** | Variable maps, `lookup()`, conditional resources |

## Descripcion

Crea entornos con niveles de costo: dev (minimo: t3.micro, single AZ, sin HA), staging (moderado: t3.small, 2 AZs), prod (completo: m5.large, 3 AZs, Multi-AZ RDS, ElastiCache). Usa un unico modulo con variable maps sensibles al entorno.

## Objetivos de aprendizaje

- Optimizar costos por entorno usando configuracion variable-driven
- Implementar sizing condicional de recursos basado en el entorno
- Disenar alta disponibilidad condicional (HA solo en prod)
- Crear modulos flexibles que soporten multiples niveles de servicio

## Requisitos previos

- Conocimientos: EC2 instance types, RDS, ElastiCache, Multi-AZ, ALB
- Herramientas: Terraform >= 1.5, AWS CLI, cuenta AWS
- Features/Servicios: EC2, RDS, ElastiCache, ALB, Auto Scaling Groups, VPC

## Arquitectura

```
variables.tf
  +-- variable "environment" { type = string }
  +-- variable "config" {
        default = {
          dev = {
            instance_type  = "t3.micro"
            az_count       = 1
            rds_multi_az   = false
            rds_instance   = "db.t3.micro"
            enable_cache   = false
            asg_min        = 1
            asg_max        = 1
          }
          staging = {
            instance_type  = "t3.small"
            az_count       = 2
            rds_multi_az   = false
            rds_instance   = "db.t3.small"
            enable_cache   = false
            asg_min        = 1
            asg_max        = 3
          }
          prod = {
            instance_type  = "m5.large"
            az_count       = 3
            rds_multi_az   = true
            rds_instance   = "db.m5.large"
            enable_cache   = true
            asg_min        = 2
            asg_max        = 10
          }
        }
      }

main.tf
  +-- locals { env = var.config[var.environment] }
  +-- VPC con N subnets segun local.env.az_count
  +-- ASG con min/max segun entorno
  +-- RDS con multi_az condicional
  +-- ElastiCache solo en prod (count = local.env.enable_cache ? 1 : 0)
```

## Pasos sugeridos

1. Definir la variable `config` como un map de objetos con la configuracion de cada nivel de entorno
2. Crear un local que seleccione la configuracion del entorno actual: `local.env = var.config[var.environment]`
3. Implementar el VPC con un numero variable de subnets/AZs usando `local.env.az_count`
4. Crear el Auto Scaling Group con `min_size` y `max_size` del mapa de configuracion
5. Implementar RDS con `multi_az = local.env.rds_multi_az` y `instance_class` variable
6. Crear ElastiCache condicionalmente: `count = local.env.enable_cache ? 1 : 0`
7. Configurar el ALB con health checks y listeners
8. Agregar outputs que muestren el costo estimado por entorno
9. Desplegar en dev y verificar que solo se crean los recursos minimos
10. Comparar los planes de dev, staging y prod para visualizar las diferencias

## Validacion

- [ ] Dev usa t3.micro, 1 AZ, sin Multi-AZ RDS, sin ElastiCache
- [ ] Staging usa t3.small, 2 AZs, sin Multi-AZ RDS, sin ElastiCache
- [ ] Prod usa m5.large, 3 AZs, Multi-AZ RDS habilitado, ElastiCache habilitado
- [ ] Un unico modulo/configuracion soporta los tres niveles
- [ ] `lookup()` o acceso directo al map selecciona la configuracion correcta
- [ ] Los recursos condicionales (ElastiCache) solo se crean cuando estan habilitados
- [ ] El ASG respeta los limites min/max de cada entorno
- [ ] Cambiar de entorno cambia el sizing sin modificar la logica
