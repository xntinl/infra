# 7.06 Container Insights en ECS y EKS

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Intermedio |
| **Modalidad** | Ambos |
| **Servicios AWS** | CloudWatch Container Insights, ECS, EKS, CloudWatch Agent |

## Descripcion

Habilitar Container Insights en clusters ECS y EKS. Desplegar el CloudWatch Agent como DaemonSet en EKS. Analizar metricas a nivel de contenedor (CPU, memoria, red, almacenamiento por pod/task). Crear dashboards de rendimiento y alarmas de utilizacion de recursos.

## Objetivos de aprendizaje

- Configurar Container Insights en ECS y EKS para monitoreo de contenedores
- Analizar metricas de utilizacion de recursos a nivel de contenedor, pod y task
- Comprender las diferencias de observabilidad entre ECS y EKS
- Diagnosticar problemas de rendimiento usando metricas de contenedores

## Requisitos previos

- Conocimientos: Docker, ECS (tasks, services), EKS (pods, deployments, DaemonSets), CloudWatch
- Herramientas: AWS CLI, Docker, ecs-cli o copilot (para ECS), kubectl (para EKS), Terraform (opcional)
- Servicios AWS: Cuenta AWS con permisos para ECS, EKS, CloudWatch, IAM

## Arquitectura

Dos entornos de contenedores en paralelo: un cluster ECS con Fargate/EC2 y un cluster EKS. En ECS, Container Insights se habilita a nivel de cluster y recopila metricas automaticamente. En EKS, el CloudWatch Agent se despliega como DaemonSet para recopilar metricas de cada nodo y pod. Ambos envian metricas a CloudWatch Container Insights, donde se visualizan en dashboards automaticos y personalizados. Alarmas de utilizacion de recursos notifican cuando los contenedores se acercan a sus limites.

## Pasos sugeridos

1. Crear un cluster ECS y habilitar Container Insights en la configuracion del cluster
2. Desplegar 2-3 servicios en ECS con limites de CPU y memoria definidos
3. Crear un cluster EKS y desplegar el CloudWatch Agent como DaemonSet
4. Desplegar 2-3 aplicaciones en EKS con resource requests y limits configurados
5. Generar carga en ambos entornos para producir metricas significativas
6. Explorar los dashboards automaticos de Container Insights en la consola de CloudWatch
7. Crear dashboards personalizados comparando metricas de ECS y EKS
8. Configurar alarmas por utilizacion de CPU y memoria a nivel de task/pod
9. Analizar metricas de red y almacenamiento para identificar cuellos de botella
10. Documentar las diferencias de observabilidad entre ECS y EKS

## Validacion

- [ ] Container Insights esta habilitado y recopilando metricas en el cluster ECS
- [ ] El CloudWatch Agent esta desplegado como DaemonSet en EKS y recopilando metricas
- [ ] Las metricas a nivel de contenedor (CPU, memoria, red) son visibles en CloudWatch
- [ ] Los dashboards automaticos de Container Insights muestran datos de ambos clusters
- [ ] Los dashboards personalizados comparan metricas de ECS vs EKS
- [ ] Las alarmas de utilizacion de recursos se disparan correctamente
- [ ] Se pueden identificar contenedores con alto consumo de recursos
