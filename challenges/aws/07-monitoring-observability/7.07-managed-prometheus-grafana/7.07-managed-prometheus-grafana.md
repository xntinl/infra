# 7.07 Managed Prometheus y Grafana

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform + Kubernetes |
| **Servicios AWS** | AMP, AMG, EKS, ADOT, SNS |

## Descripcion

Configurar Amazon Managed Prometheus (AMP) y Amazon Managed Grafana (AMG). Configurar Prometheus remote write desde EKS usando ADOT. Construir dashboards en Grafana con queries PromQL, configurar reglas de alertas e integrar con PagerDuty/Slack via SNS.

## Objetivos de aprendizaje

- Desplegar y configurar el ecosistema Prometheus en AWS con servicios gestionados
- Escribir queries PromQL para consultar metricas de aplicaciones y cluster
- Disenar dashboards en Grafana efectivos y reutilizables
- Configurar rutas de alertas desde Grafana a multiples destinos
- Integrar servicios gestionados (AMP + AMG) con EKS y ADOT

## Requisitos previos

- Conocimientos: Kubernetes, Prometheus (conceptos basicos), metricas, PromQL basico, Grafana
- Herramientas: AWS CLI, kubectl, Helm, Terraform, navegador web
- Servicios AWS: Cuenta AWS con permisos para AMP, AMG, EKS, IAM, SNS, IAM Identity Center (SSO)

## Arquitectura

Un cluster EKS ejecuta aplicaciones que exponen metricas en formato Prometheus. El ADOT Collector, desplegado como DaemonSet, scrapea las metricas de los pods y las envia via remote write a Amazon Managed Prometheus (AMP). Amazon Managed Grafana (AMG) se conecta a AMP como data source y muestra dashboards con queries PromQL. Las alerting rules en Grafana envian notificaciones a SNS, que a su vez las reenvian a Slack o PagerDuty.

## Pasos sugeridos

1. Crear un workspace de Amazon Managed Prometheus (AMP) con Terraform
2. Crear un workspace de Amazon Managed Grafana (AMG) con autenticacion via IAM Identity Center
3. Desplegar un cluster EKS con aplicaciones que exponen metricas Prometheus (/metrics endpoint)
4. Configurar el ADOT Collector con Prometheus receiver y AMP remote write exporter
5. Verificar que las metricas llegan a AMP usando la API de query de AMP
6. Configurar AMP como data source en Grafana
7. Construir dashboards con queries PromQL (rate, histogram_quantile, sum by, etc.)
8. Crear recording rules en AMP para queries costosas que se ejecutan frecuentemente
9. Configurar alerting rules en Grafana con umbrales y condiciones
10. Integrar las alertas con SNS y configurar destinos (email, Slack webhook)

## Validacion

- [ ] El workspace de AMP esta creado y recibe metricas correctamente
- [ ] El workspace de AMG esta accesible via IAM Identity Center
- [ ] El ADOT Collector envia metricas a AMP via remote write
- [ ] AMP aparece como data source funcional en Grafana
- [ ] Los dashboards muestran metricas con queries PromQL complejas
- [ ] Las alerting rules se evaluan y disparan correctamente
- [ ] Las notificaciones llegan al destino configurado (SNS -> Slack/email)
- [ ] Las recording rules funcionan y mejoran el rendimiento de queries
