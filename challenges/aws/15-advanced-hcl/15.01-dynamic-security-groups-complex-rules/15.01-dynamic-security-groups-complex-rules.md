# 15.01 Dynamic Security Groups con Reglas Complejas

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Intermedio |
| **Modalidad** | Terraform |
| **Features TF** | `dynamic` blocks, `flatten()`, `for` expressions, complex maps |

## Descripcion

Utiliza bloques `dynamic` para crear security groups a partir de un map de reglas. Soporta ingress/egress con CIDR blocks, referencias a otros security groups y prefix lists. Aplana estructuras anidadas usando `flatten()` y expresiones `for`.

## Objetivos de aprendizaje

- Dominar el uso de bloques `dynamic` para generar bloques repetidos dentro de un recurso
- Comprender como aplanar datos anidados con `flatten()` para alimentar `for_each`
- Trabajar con tipos de variables complejos (maps de objetos con listas anidadas)
- Generar reglas de security groups de forma declarativa y mantenible

## Requisitos previos

- Conocimientos: HCL basico, security groups de AWS, CIDR notation
- Herramientas: Terraform >= 1.5, AWS CLI, cuenta AWS con permisos de VPC/EC2
- Features/Servicios: VPC, Security Groups, Prefix Lists

## Arquitectura

```
variables.tf
  |
  v
map de reglas (ingress/egress)
  |
  v
locals.tf (flatten + for expressions)
  |
  v
main.tf
  +-- aws_security_group "this" (for_each)
        +-- dynamic "ingress" { ... }
        +-- dynamic "egress" { ... }
```

Se define un mapa donde cada clave es el nombre de un security group y su valor contiene listas de reglas de ingress y egress. Cada regla puede referenciar CIDR blocks, otros security groups o prefix lists. Se usa `flatten()` para normalizar las estructuras antes de pasarlas a los bloques `dynamic`.

## Pasos sugeridos

1. Definir una variable de tipo `map(object(...))` que represente multiples security groups con sus reglas de ingress y egress
2. Crear un VPC base con `aws_vpc` y subnets necesarias
3. Implementar `locals` que usen `flatten()` y expresiones `for` para transformar el mapa anidado en una lista plana de reglas
4. Crear los security groups usando `for_each` sobre el mapa de configuracion
5. Implementar bloques `dynamic "ingress"` y `dynamic "egress"` dentro de cada security group
6. Agregar soporte para reglas que referencian otros security groups (usando `security_group_id`)
7. Agregar soporte para reglas que referencan prefix lists (usando `prefix_list_ids`)
8. Crear outputs que muestren los security groups creados y sus reglas
9. Probar con `terraform plan` para verificar que las reglas se generan correctamente
10. Validar aplicando y verificando en la consola de AWS

## Validacion

- [ ] La variable acepta un mapa complejo con multiples security groups y reglas
- [ ] Los bloques `dynamic` generan las reglas de ingress y egress correctamente
- [ ] Se soportan reglas con CIDR blocks, referencias a security groups y prefix lists
- [ ] `flatten()` aplana correctamente las estructuras anidadas
- [ ] Las expresiones `for` transforman los datos al formato esperado
- [ ] `terraform plan` muestra los recursos esperados sin errores
- [ ] Los security groups se crean correctamente en AWS
