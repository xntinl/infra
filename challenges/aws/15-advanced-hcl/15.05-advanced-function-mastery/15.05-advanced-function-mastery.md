# 15.05 Dominio Avanzado de Funciones HCL

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Intermedio |
| **Modalidad** | Terraform |
| **Features TF** | Built-in functions, `terraform console` |

## Descripcion

Practica funciones avanzadas de HCL: `cidrsubnet()` para calculo de redes, `templatefile()` para plantillas complejas, `merge()`/`lookup()`/`coalesce()` para manipulacion de maps, `regex()`/`regexall()` para parsing de strings, y `jsondecode()`/`yamldecode()` para archivos de configuracion.

## Objetivos de aprendizaje

- Dominar la biblioteca de funciones built-in de HCL
- Componer expresiones complejas combinando multiples funciones
- Usar `terraform console` como herramienta de depuracion y experimentacion
- Aplicar funciones de red, string, coleccion y encoding en escenarios reales

## Requisitos previos

- Conocimientos: HCL basico, CIDR notation, JSON/YAML, expresiones regulares
- Herramientas: Terraform >= 1.5, `terraform console`
- Features/Servicios: VPC, subnets (para ejemplos de red)

## Arquitectura

```
config/
  +-- app-config.yaml        (configuracion de aplicacion)
  +-- templates/
        +-- user-data.sh.tpl (template para EC2)
        +-- policy.json.tpl  (template para IAM)

main.tf
  +-- Funciones de red:    cidrsubnet(), cidrhost(), cidrnetmask()
  +-- Funciones de string: regex(), regexall(), replace(), split(), join()
  +-- Funciones de map:    merge(), lookup(), coalesce(), zipmap()
  +-- Funciones de lista:  flatten(), chunklist(), distinct(), sort()
  +-- Funciones de encode: jsondecode(), yamldecode(), jsonencode()
  +-- Templates:           templatefile() con loops y condicionales
```

## Pasos sugeridos

1. Usar `terraform console` para experimentar con `cidrsubnet()`: calcular subnets a partir de un CIDR base, entender los parametros newbits y netnum
2. Implementar un esquema de subnetting automatico que genere subnets publicas y privadas por AZ usando `cidrsubnet()`
3. Crear archivos de configuracion YAML y cargarlos con `yamldecode(file(...))` para alimentar recursos
4. Implementar `templatefile()` con templates que usen loops `%{ for ... }` y condicionales `%{ if ... }`
5. Usar `merge()` para combinar maps de configuracion base con overrides por entorno
6. Implementar `lookup()` con valores por defecto para configuracion resiliente
7. Usar `regex()` y `regexall()` para validar y extraer datos de strings (ARNs, nombres de recursos)
8. Combinar `zipmap()` con `keys()` y `values()` para transformar estructuras de datos
9. Crear locals que compongan multiples funciones en expresiones complejas
10. Documentar cada funcion con ejemplos de entrada/salida usando comentarios

## Validacion

- [ ] `cidrsubnet()` genera los rangos de red esperados para cada subnet
- [ ] `yamldecode()` carga configuracion externa correctamente
- [ ] `templatefile()` renderiza templates con loops y condicionales
- [ ] `merge()` combina maps dando prioridad al ultimo argumento
- [ ] `lookup()` retorna valores por defecto cuando la clave no existe
- [ ] `regex()` extrae datos de strings con formato conocido
- [ ] Las expresiones compuestas producen los resultados esperados
- [ ] `terraform console` se usa para verificar cada funcion interactivamente
