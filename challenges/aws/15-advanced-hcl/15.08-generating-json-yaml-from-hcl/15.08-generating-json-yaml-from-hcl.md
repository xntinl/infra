# 15.08 Generacion de JSON/YAML desde HCL

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Intermedio |
| **Modalidad** | Terraform |
| **Features TF** | `jsonencode()`, `yamlencode()`, `templatefile()`, heredoc |

## Descripcion

Usa `jsonencode()` y `yamlencode()` para generar archivos de configuracion a partir de estructuras HCL. Genera documentos de IAM policy, definiciones de ECS task, ASL de Step Functions y manifiestos de Kubernetes desde variables de Terraform.

## Objetivos de aprendizaje

- Generar configuraciones JSON/YAML complejas desde estructuras nativas de HCL
- Dominar las funciones de encoding y sus particularidades
- Usar patrones de templates para configuraciones dinamicas
- Comprender cuando usar `jsonencode()` vs `templatefile()` vs heredoc

## Requisitos previos

- Conocimientos: JSON, YAML, IAM policies, ECS task definitions, Step Functions ASL
- Herramientas: Terraform >= 1.5, AWS CLI, cuenta AWS
- Features/Servicios: IAM, ECS, Step Functions, S3

## Arquitectura

```
templates/
  +-- step-function.json.tpl   (template ASL con placeholders)
  +-- task-definition.json.tpl (template ECS)

locals.tf
  +-- iam_policy = jsonencode({
  |     Version = "2012-10-17"
  |     Statement = [ ... ]
  |   })
  +-- ecs_task_def = jsonencode({ ... })
  +-- step_function_asl = templatefile("...", { ... })

main.tf
  +-- aws_iam_policy "this"          (policy generada con jsonencode)
  +-- aws_ecs_task_definition "this" (container_definitions con jsonencode)
  +-- aws_sfn_state_machine "this"   (definition con templatefile)
  +-- local_file "config"            (YAML generado con yamlencode)
```

## Pasos sugeridos

1. Crear una IAM policy usando `jsonencode()` con la estructura HCL nativa (Version, Statement, etc.)
2. Comparar con el enfoque de `data "aws_iam_policy_document"` y entender las diferencias
3. Generar una ECS task definition usando `jsonencode()` para `container_definitions`
4. Crear un template ASL para Step Functions usando `templatefile()` con placeholders de Lambda ARN
5. Usar heredoc (`<<-EOF ... EOF`) para JSON inline simple y comparar legibilidad
6. Generar un archivo YAML con `yamlencode()` para configuracion de aplicacion
7. Usar `local_file` para escribir el output generado a disco (util para depuracion)
8. Combinar `jsondecode()` con `jsonencode()` para transformar configuraciones existentes
9. Implementar generacion condicional de secciones dentro del JSON/YAML
10. Verificar que los documentos generados son validos usando las herramientas correspondientes

## Validacion

- [ ] `jsonencode()` genera IAM policies validas desde estructuras HCL
- [ ] Las ECS task definitions generadas son aceptadas por AWS
- [ ] `templatefile()` renderiza ASL de Step Functions con ARNs dinamicos
- [ ] `yamlencode()` produce YAML valido desde maps y listas de HCL
- [ ] Los heredocs generan JSON inline legible
- [ ] `jsondecode()` parsea JSON externo para su uso en Terraform
- [ ] Las configuraciones generadas son validas y funcionales en AWS
- [ ] Se entiende cuando usar cada enfoque (encode vs template vs heredoc)
