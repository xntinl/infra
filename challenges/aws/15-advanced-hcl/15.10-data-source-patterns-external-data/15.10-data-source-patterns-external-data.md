# 15.10 Patrones de Data Sources y Datos Externos

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Intermedio |
| **Modalidad** | Terraform |
| **Features TF** | Data sources, `external`, `http`, provider data sources |

## Descripcion

Domina los data sources: `aws_caller_identity`, `aws_region`, `aws_availability_zones` para descubrimiento. Usa el data source `external` para llamar scripts. Implementa el data source `http` para llamadas a APIs. Encadena data sources para lookups complejos.

## Objetivos de aprendizaje

- Dominar patrones de data sources para descubrimiento de infraestructura existente
- Integrar datos externos mediante scripts y llamadas HTTP
- Comprender patrones de descubrimiento automatico (discovery patterns)
- Encadenar data sources para resolver lookups complejos

## Requisitos previos

- Conocimientos: Data sources basicos de Terraform, AWS CLI, scripting
- Herramientas: Terraform >= 1.5, AWS CLI, Python o bash (para scripts externos), jq
- Features/Servicios: EC2, VPC, IAM, SSM Parameter Store

## Arquitectura

```
scripts/
  +-- get-latest-ami.sh        (script para data source external)
  +-- fetch-config.py          (script Python para external)

main.tf
  +-- Data Sources de Descubrimiento:
  |     +-- data "aws_caller_identity" "current"
  |     +-- data "aws_region" "current"
  |     +-- data "aws_availability_zones" "available"
  |     +-- data "aws_vpc" "selected"
  |     +-- data "aws_subnets" "private"
  |
  +-- Data Sources Encadenados:
  |     +-- data "aws_ami" "latest" (filtros dinamicos)
  |     +-- data "aws_ssm_parameter" "config"
  |
  +-- Data Sources Externos:
  |     +-- data "external" "custom_lookup"
  |     +-- data "http" "api_config"
  |
  +-- Recursos que usan los datos descubiertos
```

## Pasos sugeridos

1. Usar `aws_caller_identity` para obtener el account ID y ARN del caller actual
2. Usar `aws_region` y `aws_availability_zones` para descubrimiento automatico de la region y AZs disponibles
3. Encadenar `aws_vpc` con `aws_subnets` usando filtros para descubrir subnets de un VPC existente
4. Implementar `aws_ami` con filtros para encontrar la AMI mas reciente de un tipo especifico
5. Usar `aws_ssm_parameter` para leer configuracion almacenada en Parameter Store
6. Crear un script bash que retorne JSON y usarlo con el data source `external`
7. Implementar el data source `http` para obtener datos de una API externa (ej: IP publica)
8. Encadenar multiples data sources: descubrir VPC -> encontrar subnets -> filtrar por tag -> usar en recurso
9. Combinar data sources con `for` expressions para transformar los resultados
10. Implementar un patron de "lookup or create" usando `try()` con data sources

## Validacion

- [ ] `aws_caller_identity` retorna el account ID correcto
- [ ] `aws_availability_zones` descubre las AZs disponibles en la region
- [ ] El encadenamiento VPC -> subnets resuelve correctamente las subnets privadas
- [ ] `aws_ami` encuentra la AMI mas reciente con los filtros especificados
- [ ] El data source `external` ejecuta el script y parsea el JSON de respuesta
- [ ] El data source `http` obtiene datos de una URL externa
- [ ] Los data sources encadenados resuelven lookups complejos sin errores
- [ ] Los recursos usan los datos descubiertos correctamente
