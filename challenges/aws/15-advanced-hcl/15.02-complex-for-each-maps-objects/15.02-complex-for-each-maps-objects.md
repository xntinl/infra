# 15.02 for_each Complejo con Maps de Objetos

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform |
| **Features TF** | `for_each`, complex types, `each.key`/`each.value`, cross-references |

## Descripcion

Utiliza `for_each` con maps de objetos para crear multiples recursos relacionados (por ejemplo, para cada microservicio: Lambda + API Gateway route + DynamoDB table + IAM role). Maneja referencias cruzadas entre recursos dentro de la misma iteracion de `for_each`.

## Objetivos de aprendizaje

- Dominar iteracion avanzada con `for_each` sobre maps de objetos complejos
- Manejar tipos complejos en variables y locals
- Resolver referencias cruzadas entre recursos que comparten la misma clave de iteracion
- Disenar estructuras de datos que representen stacks completos de microservicios

## Requisitos previos

- Conocimientos: `for_each` basico, Lambda, API Gateway, DynamoDB, IAM
- Herramientas: Terraform >= 1.5, AWS CLI, cuenta AWS
- Features/Servicios: Lambda, API Gateway, DynamoDB, IAM

## Arquitectura

```
variable "microservices" = map(object({
  runtime, memory, routes, table_config, ...
}))
  |
  v
for_each = var.microservices
  +-- aws_lambda_function     "this"[each.key]
  +-- aws_apigatewayv2_route  "this"[each.key]
  +-- aws_dynamodb_table      "this"[each.key]
  +-- aws_iam_role            "this"[each.key]
  +-- aws_iam_policy          "this"[each.key]
```

Cada microservicio se define como una entrada en un mapa. Terraform itera sobre el mapa y crea todos los recursos necesarios para cada servicio, conectandolos entre si mediante `each.key`.

## Pasos sugeridos

1. Disenar la variable `microservices` como `map(object(...))` con campos para runtime, memoria, rutas HTTP, configuracion de tabla, etc.
2. Crear el IAM role con `for_each` que genere un rol por microservicio
3. Crear la policy de IAM que permita acceso a la tabla DynamoDB correspondiente (referencia cruzada usando `each.key`)
4. Crear las funciones Lambda con `for_each`, referenciando el IAM role del mismo `each.key`
5. Crear las tablas DynamoDB con `for_each`, usando la configuracion de cada microservicio
6. Crear un API Gateway HTTP y las rutas con `for_each`, integrando con la Lambda correspondiente
7. Implementar outputs que agrupen la informacion de cada microservicio
8. Probar agregando y eliminando microservicios del mapa para verificar que `for_each` maneja correctamente los cambios
9. Validar que las referencias cruzadas entre recursos son correctas

## Validacion

- [ ] La variable acepta un mapa de objetos con la configuracion completa de cada microservicio
- [ ] `for_each` crea todos los recursos necesarios para cada microservicio
- [ ] Las referencias cruzadas entre recursos del mismo microservicio funcionan correctamente
- [ ] Agregar un nuevo microservicio al mapa crea todos sus recursos sin afectar los existentes
- [ ] Eliminar un microservicio del mapa destruye solo sus recursos asociados
- [ ] Los outputs agrupan la informacion por microservicio
- [ ] `terraform plan` no muestra cambios inesperados en recursos existentes
