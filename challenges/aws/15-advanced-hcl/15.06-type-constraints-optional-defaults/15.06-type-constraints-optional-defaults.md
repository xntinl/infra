# 15.06 Type Constraints con optional() y Defaults

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform |
| **Features TF** | `optional()`, `object()`, type constraints, `defaults()` |

## Descripcion

Disena tipos de variables complejos usando `object()` con atributos `optional()` y valores por defecto. Implementa objetos anidados con validacion. Usa la funcion `defaults()` para deep merging de objetos de configuracion.

## Objetivos de aprendizaje

- Dominar el sistema de tipos de Terraform para variables complejas
- Implementar atributos opcionales con `optional()` y valores por defecto
- Disenar interfaces de modulos con tipos bien definidos y auto-documentados
- Aplicar deep merging de configuraciones con `defaults()`

## Requisitos previos

- Conocimientos: Variables de Terraform, tipos basicos y complejos, modulos
- Herramientas: Terraform >= 1.5 (soporte completo de `optional()`)
- Features/Servicios: Cualquier servicio AWS (los tipos se aplican a variables)

## Arquitectura

```
variables.tf
  +-- variable "services" {
  |     type = map(object({
  |       runtime     = string
  |       memory      = optional(number, 128)
  |       timeout     = optional(number, 30)
  |       environment = optional(map(string), {})
  |       scaling = optional(object({
  |         min_capacity = optional(number, 1)
  |         max_capacity = optional(number, 10)
  |       }), {})
  |       tags = optional(map(string))
  |     }))
  |   }

locals.tf
  +-- Merge de defaults con configuracion del usuario
  +-- Validaciones personalizadas

main.tf
  +-- Recursos creados a partir de la variable tipada
```

El objetivo es disenar variables que acepten configuracion parcial del usuario y completen los valores faltantes con defaults sensatos, creando una interfaz ergonomica para el consumidor del modulo.

## Pasos sugeridos

1. Definir una variable con `type = object({...})` usando tipos basicos (string, number, bool)
2. Agregar `optional()` a atributos que no son obligatorios, con valores por defecto
3. Anidar objetos dentro de objetos, usando `optional()` en cada nivel
4. Implementar `optional(type, default_value)` con la sintaxis de dos argumentos para defaults inline
5. Crear bloques `validation` que verifiquen rangos, formatos y restricciones de negocio
6. Usar `defaults()` en locals para aplicar deep merging de configuraciones base
7. Probar con inputs parciales: solo campos obligatorios, verificar que los opcionales toman sus defaults
8. Probar con inputs completos: verificar que los valores del usuario sobreescriben los defaults
9. Crear un modulo que use estas variables tipadas como su interfaz publica
10. Documentar la interfaz del modulo con `description` en cada variable

## Validacion

- [ ] Las variables aceptan inputs parciales sin error
- [ ] Los atributos `optional()` toman sus valores por defecto cuando se omiten
- [ ] Los objetos anidados con `optional()` funcionan en multiples niveles
- [ ] Las validaciones rechazan valores fuera de rango con mensajes claros
- [ ] `defaults()` aplica deep merging correctamente
- [ ] El input completo del usuario sobreescribe todos los defaults
- [ ] La interfaz del modulo es clara y auto-documentada
- [ ] `terraform plan` funciona con distintas combinaciones de inputs
