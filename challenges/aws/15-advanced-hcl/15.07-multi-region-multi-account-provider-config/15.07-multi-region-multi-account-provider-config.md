# 15.07 Configuracion Multi-Region y Multi-Account con Providers

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform |
| **Features TF** | Provider aliases, `configuration_aliases`, `assume_role`, provider passing |

## Descripcion

Configura multiples providers de AWS con aliases para despliegues multi-region. Usa el meta-argumento `provider` en modulos. Implementa patrones de assume-role para recursos cross-account. Pasa providers a modulos hijos.

## Objetivos de aprendizaje

- Dominar patrones multi-provider con aliases para despliegues en varias regiones
- Implementar assume role para acceso cross-account
- Comprender la herencia de providers y como pasarlos a modulos hijos
- Disenar modulos que soporten `configuration_aliases`

## Requisitos previos

- Conocimientos: Providers de Terraform, IAM roles, assume role, regiones de AWS
- Herramientas: Terraform >= 1.5, AWS CLI, al menos una cuenta AWS (idealmente dos para cross-account)
- Features/Servicios: IAM, S3, Route 53 (recursos multi-region), STS

## Arquitectura

```
providers.tf
  +-- provider "aws" { region = "us-east-1" }              # default
  +-- provider "aws" { alias = "eu" region = "eu-west-1" }  # Europa
  +-- provider "aws" { alias = "target_account"             # cross-account
  |     assume_role { role_arn = "arn:aws:iam::123:role/x" }
  |   }

modules/replica/
  +-- terraform { required_providers {
  |     aws = { configuration_aliases = [aws.primary, aws.replica] }
  |   }}
  +-- Recursos que usan provider = aws.primary o aws.replica

main.tf
  +-- module "s3_replica" {
  |     providers = {
  |       aws.primary = aws
  |       aws.replica = aws.eu
  |     }
  |   }
  +-- Recursos con provider = aws.eu para la region europea
  +-- Recursos con provider = aws.target_account para otra cuenta
```

## Pasos sugeridos

1. Configurar el provider default de AWS para la region principal
2. Agregar un segundo provider con `alias` para una region secundaria (ej: `eu-west-1`)
3. Crear recursos en la region secundaria usando `provider = aws.eu`
4. Crear un modulo que declare `configuration_aliases` en su bloque `required_providers`
5. Pasar multiples providers al modulo usando el bloque `providers` en la llamada al modulo
6. Implementar replicacion S3 cross-region como caso de uso practico
7. Configurar un provider con `assume_role` para acceso cross-account
8. Crear recursos en la cuenta destino usando el provider con assume role
9. Implementar un modulo que reciba providers dinamicamente y cree recursos en multiples regiones
10. Verificar que cada recurso se crea en la region/cuenta correcta

## Validacion

- [ ] Los providers con alias apuntan a regiones distintas
- [ ] Los recursos se crean en la region correcta segun el provider asignado
- [ ] `configuration_aliases` permite que los modulos reciban multiples providers
- [ ] El bloque `providers` pasa correctamente los providers a modulos hijos
- [ ] `assume_role` permite crear recursos en otra cuenta de AWS
- [ ] La replicacion S3 cross-region funciona con dos providers distintos
- [ ] `terraform plan` muestra los recursos en las regiones/cuentas esperadas
- [ ] Los modulos son reutilizables con diferentes combinaciones de providers
