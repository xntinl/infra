# 18.03 Escaneo de Compliance CIS con Checkov

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |

## Descripcion

Ejecutar Checkov con checks de benchmark CIS contra el codigo Terraform. Implementar politicas personalizadas en Python o YAML. Generar reportes de compliance. Integrar con CI/CD y usar `#checkov:skip` con justificaciones documentadas.

## Objetivos de aprendizaje

- Comprender los benchmarks CIS y su aplicacion a infraestructura como codigo
- Crear politicas personalizadas en Python y YAML para Checkov
- Generar y analizar reportes de compliance
- Implementar excepciones justificadas de forma controlada

## Requisitos previos

- Conocimientos: Terraform intermedio, CIS benchmarks (conceptos), Python basico (para custom policies)
- Herramientas: Checkov, Python 3, Terraform CLI, CI/CD pipeline
- Features: Escaneo HCL, CIS AWS Foundations Benchmark

## Arquitectura

Checkov analiza los archivos Terraform contra cientos de checks predefinidos basados en CIS benchmarks y best practices. Los resultados se pueden exportar como reportes de compliance y usarse como gate en CI/CD.

```
Codigo Terraform
  |
  v
Checkov Scanner
  |
  +-- CIS AWS Foundations Benchmark
  +-- Best practices (networking, IAM, encryption)
  +-- Custom policies (Python / YAML)
  |
  v
Reporte de Compliance
  |
  +-- Passed checks
  +-- Failed checks (con CIS reference)
  +-- Skipped checks (con justificacion)
  |
  v
CI Gate + Dashboard de Compliance
```

## Pasos sugeridos

1. Instalar Checkov: `pip install checkov`
2. Ejecutar un escaneo inicial: `checkov -d . --framework terraform`
3. Revisar los resultados agrupados por categoria CIS
4. Corregir los checks fallidos mas criticos (ej: CK_AWS_* para S3, IAM, VPC)
5. Para checks que no aplican, agregar `#checkov:skip=CKV_AWS_XXX: justificacion`
6. Crear una politica personalizada en YAML para un requisito especifico
7. Crear una politica personalizada en Python para logica mas compleja
8. Generar reporte en formato JSON o JUnit: `checkov -d . -o json > report.json`
9. Integrar en CI con `--hard-fail-on` para checks criticos
10. Configurar un dashboard o almacenar reportes historicos para tracking de compliance

## Validacion

- [ ] Checkov se ejecuta exitosamente contra todo el codebase
- [ ] Los checks CIS criticos estan corregidos (encriptacion, acceso publico, logging)
- [ ] Las politicas personalizadas en YAML detectan violaciones correctamente
- [ ] Las politicas personalizadas en Python implementan logica avanzada
- [ ] Los skips tienen justificaciones documentadas y revisadas
- [ ] El reporte de compliance se genera en formato estructurado (JSON/JUnit)
- [ ] El CI falla cuando se introduce una violacion critica
- [ ] El porcentaje de compliance mejora progresivamente
