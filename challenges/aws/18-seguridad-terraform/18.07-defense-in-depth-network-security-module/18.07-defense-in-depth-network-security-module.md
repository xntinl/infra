# 18.07 Modulo de Seguridad de Red con Defensa en Profundidad

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform |
| **Servicios AWS** | WAF, CloudFront, ALB, Security Groups, NACLs, Network Firewall |

## Descripcion

Construir un modulo de seguridad que implemente defensas por capas: WAF -> CloudFront -> ALB con security groups -> NACLs -> VPC Flow Logs -> Network Firewall. Cada capa es configurable de forma independiente. Validar que todas las capas funcionan en conjunto.

## Objetivos de aprendizaje

- Implementar el patron de defensa en profundidad para infraestructura de red
- Disenar seguridad por capas donde cada capa es independiente
- Construir modulos de seguridad reutilizables y configurables
- Validar que multiples capas de seguridad operan correctamente en conjunto

## Requisitos previos

- Conocimientos: Redes AWS (VPC, subnets, routing), WAF rules, CloudFront distributions, seguridad de red en capas
- Herramientas: Terraform CLI, AWS CLI
- Servicios: WAF, CloudFront, ALB, VPC (Security Groups, NACLs, Flow Logs), Network Firewall

## Arquitectura

Cada capa de seguridad protege contra diferentes tipos de amenazas. El trafico atraviesa multiples capas antes de llegar a la aplicacion, y cada capa puede bloquear trafico independientemente.

```
Internet
  |
  v
WAF (Capa 7)
  +-- Rate limiting
  +-- SQL injection protection
  +-- XSS protection
  +-- Geo-blocking
  |
  v
CloudFront (CDN + Capa 7)
  +-- TLS termination
  +-- Origin access control
  +-- Cache (reduce carga)
  |
  v
Network Firewall (Capa 3-7)
  +-- Stateful inspection
  +-- IPS rules
  +-- Domain filtering
  |
  v
ALB + Security Groups (Capa 4)
  +-- Solo HTTPS (443)
  +-- Solo desde CloudFront
  |
  v
NACLs (Capa 3)
  +-- Stateless filtering
  +-- Deny lists
  |
  v
Aplicacion (EC2/ECS/Lambda)
  |
  v
VPC Flow Logs (Monitoreo)
  +-- Registro de todo el trafico
  +-- Analisis y alertas
```

## Pasos sugeridos

1. Disenar la interfaz del modulo: variables para habilitar/deshabilitar cada capa
2. Implementar la capa de VPC Flow Logs (monitoreo pasivo, siempre activo)
3. Implementar NACLs con reglas basicas (deny lists, restricciones por CIDR)
4. Implementar Security Groups para el ALB (solo HTTPS, restringido por origen)
5. Configurar el ALB con listeners HTTPS y certificado ACM
6. Implementar CloudFront con origin access y headers personalizados para validar origen
7. Configurar WAF con reglas: rate limiting, SQL injection, XSS, geo-blocking
8. Implementar Network Firewall con reglas stateful para inspeccion profunda
9. Conectar todas las capas: WAF -> CloudFront -> Network Firewall -> ALB -> App
10. Probar cada capa individualmente y luego en conjunto

## Validacion

- [ ] Cada capa se puede habilitar/deshabilitar independientemente
- [ ] WAF bloquea solicitudes maliciosas (SQL injection, XSS)
- [ ] CloudFront solo sirve trafico via HTTPS
- [ ] El ALB solo acepta trafico desde CloudFront (no directo)
- [ ] Los Security Groups restringen el trafico al minimo necesario
- [ ] Los NACLs proporcionan filtrado adicional stateless
- [ ] Network Firewall inspecciona y filtra trafico segun reglas
- [ ] VPC Flow Logs registran todo el trafico para auditoria
- [ ] El modulo es reutilizable con variables claras y documentadas
