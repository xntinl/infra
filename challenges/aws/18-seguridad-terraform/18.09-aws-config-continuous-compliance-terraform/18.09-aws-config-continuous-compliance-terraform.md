# 18.09 Compliance Continuo con AWS Config via Terraform

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform |
| **Servicios AWS** | AWS Config, SSM Automation, Lambda |

## Descripcion

Desplegar reglas de AWS Config via Terraform que enforzen compliance: tags requeridos, encriptacion habilitada, acceso publico bloqueado. Desplegar acciones de remediacion (SSM Automation) que auto-corrijan recursos no conformes. Monitorear el dashboard de compliance.

## Objetivos de aprendizaje

- Implementar compliance continuo con controles detectivos
- Configurar remediacion automatica para recursos no conformes
- Disenar un sistema de compliance que opera en tiempo real
- Monitorear y reportar el estado de compliance de la infraestructura

## Requisitos previos

- Conocimientos: AWS Config (rules, recorders), SSM Automation documents, Lambda, Terraform intermedio
- Herramientas: Terraform CLI, AWS CLI
- Servicios: AWS Config, SSM (Systems Manager Automation), Lambda, S3 (para Config delivery), SNS (notificaciones)

## Arquitectura

AWS Config monitorea continuamente la configuracion de los recursos. Las reglas evaluan si los recursos cumplen con las politicas definidas. Cuando se detecta un recurso no conforme, se ejecuta automaticamente una accion de remediacion via SSM Automation o Lambda.

```
Recursos AWS
  |
  v
AWS Config Recorder (monitoreo continuo)
  |
  v
Config Rules
  |
  +-- required-tags (tags obligatorios)
  +-- encrypted-volumes (EBS encriptado)
  +-- s3-bucket-public-read-prohibited
  +-- restricted-ssh (puerto 22 restringido)
  |
  v
Evaluacion: COMPLIANT / NON_COMPLIANT
  |
  +-- COMPLIANT -> OK
  +-- NON_COMPLIANT -> Remediacion automatica
        |
        +-- SSM Automation (agregar tags, habilitar encriptacion)
        +-- Lambda (logica personalizada)
        |
        v
  SNS Notification (alerta al equipo)
  |
  v
Config Dashboard (vista de compliance)
```

## Pasos sugeridos

1. Configurar AWS Config Recorder y Delivery Channel con Terraform
2. Crear un S3 bucket para almacenar los snapshots de configuracion
3. Desplegar la regla `required-tags`: verificar que todos los recursos tengan tags obligatorios
4. Desplegar la regla `encrypted-volumes`: verificar que los EBS esten encriptados
5. Desplegar la regla `s3-bucket-public-read-prohibited`: bloquear S3 publico
6. Desplegar la regla `restricted-ssh`: verificar que el puerto 22 este restringido
7. Crear un SSM Automation document para agregar tags faltantes automaticamente
8. Crear una Lambda de remediacion para logica mas compleja (ej: habilitar encriptacion)
9. Vincular las acciones de remediacion a las reglas de Config
10. Configurar SNS para notificaciones cuando se detectan recursos no conformes
11. Verificar el dashboard de compliance en la consola de AWS Config

## Validacion

- [ ] AWS Config Recorder esta activo y registrando cambios
- [ ] Las reglas de Config evaluan correctamente los recursos
- [ ] Los recursos conformes aparecen como COMPLIANT
- [ ] Los recursos no conformes aparecen como NON_COMPLIANT
- [ ] La remediacion automatica corrige recursos no conformes (ej: agrega tags)
- [ ] Las notificaciones SNS se envian cuando hay no-compliance
- [ ] El dashboard de compliance muestra el estado actual de todos los recursos
- [ ] Las reglas se pueden agregar/modificar facilmente via Terraform
