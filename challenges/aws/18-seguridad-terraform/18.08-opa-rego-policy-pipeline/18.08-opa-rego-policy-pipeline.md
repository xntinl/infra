# 18.08 Pipeline de Politicas con OPA y Rego

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform + OPA |
| **Features TF** | `terraform show -json`, plan output |

## Descripcion

Implementar OPA (Open Policy Agent) para enforcement de politicas en Terraform. Escribir politicas Rego que evaluen el output JSON de `terraform plan`. Integrar en CI/CD como gate. Probar politicas con `opa test`. Construir una libreria de politicas con documentacion.

## Objetivos de aprendizaje

- Aprender el lenguaje OPA/Rego para policy-as-code
- Evaluar planes de Terraform programaticamente con OPA
- Implementar testing de politicas con `opa test`
- Disenar una libreria de politicas reutilizable y documentada

## Requisitos previos

- Conocimientos: Terraform plan JSON format, logica de evaluacion de politicas, JSON path querying
- Herramientas: OPA CLI, Terraform CLI, jq, CI/CD pipeline
- Features: `terraform show -json`, `terraform plan -out`

## Arquitectura

OPA evalua el output JSON del plan de Terraform contra politicas escritas en Rego. Las politicas se organizan en una libreria con tests automatizados. El pipeline CI/CD usa OPA como gate entre plan y apply.

```
terraform plan -out=tfplan
  |
  v
terraform show -json tfplan > plan.json
  |
  v
OPA Evaluation
  |
  +-- policies/
  |     +-- tags/mandatory_tags.rego
  |     +-- security/no_public_access.rego
  |     +-- cost/instance_limits.rego
  |     +-- network/vpc_required.rego
  |
  +-- tests/
  |     +-- tags/mandatory_tags_test.rego
  |     +-- security/no_public_access_test.rego
  |     +-- ...
  |
  v
opa eval -d policies/ -i plan.json "data.terraform.deny"
  |
  +-- [] (vacio) -> Pasa -> Apply
  +-- ["violacion 1", ...] -> Falla -> Bloquear
```

## Pasos sugeridos

1. Instalar OPA CLI: `brew install opa`
2. Generar un plan JSON de ejemplo: `terraform plan -out=tfplan && terraform show -json tfplan > plan.json`
3. Estudiar la estructura del JSON del plan (resource_changes, planned_values, etc.)
4. Escribir la primera politica Rego: tags obligatorios en todos los recursos
5. Escribir politica de seguridad: prohibir security groups con ingress 0.0.0.0/0
6. Escribir politica de costos: limitar tipos de instancia permitidos
7. Escribir politica de red: requerir VPC para recursos que lo soporten
8. Crear tests para cada politica con `_test.rego` y datos de prueba
9. Ejecutar tests: `opa test policies/ tests/ -v`
10. Integrar en CI/CD: ejecutar OPA eval como step entre plan y apply
11. Documentar cada politica con descripcion, justificacion y ejemplos

## Validacion

- [ ] OPA evalua correctamente el JSON del plan de Terraform
- [ ] La politica de tags detecta recursos sin tags obligatorios
- [ ] La politica de seguridad bloquea security groups abiertos
- [ ] La politica de costos restringe tipos de instancia no autorizados
- [ ] Todos los tests de Rego pasan con `opa test`
- [ ] El pipeline CI/CD bloquea applies que violan politicas
- [ ] Las politicas estan documentadas con ejemplos de uso
- [ ] La libreria es extensible (facil agregar nuevas politicas)
