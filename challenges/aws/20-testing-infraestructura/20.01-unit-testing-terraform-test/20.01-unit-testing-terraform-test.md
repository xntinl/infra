# 20.01 Unit Testing con terraform test

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Intermedio |
| **Modalidad** | Terraform |
| **Features TF** | `terraform test`, `.tftest.hcl`, `assert` |

## Descripcion

Escribe unit tests usando `terraform test` (1.6+) para un modulo. Crea archivos `.tftest.hcl` con bloques `run` que validan output values, conteo de recursos y valores de atributos. Usa `command = plan` para validacion rapida sin desplegar recursos reales.

## Objetivos de aprendizaje

- Dominar el framework nativo de testing de Terraform
- Escribir assertions a nivel de plan sin necesidad de desplegar infraestructura
- Organizar tests de forma estructurada y mantenible

## Requisitos previos

- Conocimientos: Terraform intermedio, concepto de modulos, outputs y variables
- Herramientas: Terraform >= 1.6
- Features: `terraform test`, archivos `.tftest.hcl`, bloques `run` y `assert`

## Arquitectura

Estructura de un modulo con unit tests:

```
module/
  main.tf
  variables.tf
  outputs.tf
  tests/
    unit.tftest.hcl
    naming.tftest.hcl
    validation.tftest.hcl
```

El modulo bajo test puede ser un modulo sencillo (por ejemplo, un modulo de S3 bucket con naming conventions, tags obligatorios y configuracion de cifrado). Los tests validan la logica del modulo sin crear recursos reales.

## Pasos sugeridos

1. Crear un modulo de Terraform reutilizable con variables, logica condicional y outputs
2. Escribir el primer archivo `.tftest.hcl` con un bloque `run` basico usando `command = plan`
3. Agregar assertions para validar que los outputs tienen los valores esperados
4. Crear tests que verifiquen el conteo de recursos generados por el modulo
5. Escribir tests para validar atributos especificos de los recursos (tags, nombres, configuraciones)
6. Agregar tests negativos que verifiquen que inputs invalidos producen errores de validacion
7. Organizar los tests en multiples archivos por categoria (naming, security, outputs)
8. Ejecutar `terraform test` y verificar que todos los tests pasan

## Validacion

- [ ] Los archivos `.tftest.hcl` estan correctamente estructurados con bloques `run`
- [ ] Los tests usan `command = plan` para ejecucion rapida sin despliegue
- [ ] Las assertions validan output values correctamente
- [ ] Los tests verifican el conteo de recursos esperados
- [ ] Los tests validan atributos especificos de los recursos
- [ ] Los tests negativos detectan inputs invalidos correctamente
- [ ] `terraform test` ejecuta todos los tests y reporta resultados
- [ ] Los tests estan organizados en archivos separados por categoria
