# 20.07 Chaos Engineering para Infraestructura

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform + FIS |
| **Servicios AWS** | FIS, EC2, ECS, RDS, ALB |

## Descripcion

Despues de desplegar infraestructura con Terraform, usa AWS FIS para inyectar fallos y verificar la resiliencia. Testea: recuperacion ante terminacion de instancias, manejo de fallos de AZ y comportamiento ante particiones de red. Documenta las brechas de resiliencia de la infraestructura.

## Objetivos de aprendizaje

- Comprender los principios de chaos engineering aplicados a infraestructura
- Disenar y ejecutar experimentos de inyeccion de fallos con AWS FIS
- Verificar la resiliencia de la infraestructura desplegada con Terraform
- Documentar brechas de resiliencia y planes de mejora

## Requisitos previos

- Conocimientos: Terraform intermedio, arquitectura multi-AZ, conceptos de alta disponibilidad
- Herramientas: Terraform >= 1.0, AWS CLI, jq
- Servicios AWS: FIS, EC2, ECS, RDS Multi-AZ, ALB, CloudWatch

## Arquitectura

Desplegar una aplicacion 3-tier con Terraform (ALB + ECS/EC2 + RDS Multi-AZ) distribuida en 3 AZs. Crear experiment templates en FIS que inyecten fallos a nivel de instancia, AZ y red. CloudWatch monitorea la disponibilidad durante cada experimento.

## Pasos sugeridos

1. Desplegar infraestructura base con Terraform: VPC multi-AZ, ALB, Auto Scaling Group o ECS Service, RDS Multi-AZ
2. Crear un IAM role para FIS con permisos sobre los recursos objetivo
3. Configurar CloudWatch Alarms como stop conditions para los experimentos
4. Crear experiment template para terminacion de instancia EC2 y verificar que ASG la reemplaza
5. Crear experiment template para simulacion de fallo de AZ y verificar que el trafico se redirige
6. Crear experiment template para failover de RDS y medir el tiempo de indisponibilidad
7. Ejecutar cada experimento y monitorear metricas de disponibilidad en CloudWatch
8. Documentar resultados: tiempo de recuperacion, impacto en usuarios, brechas encontradas

## Validacion

- [ ] La infraestructura se despliega correctamente con Terraform
- [ ] Los experiment templates de FIS se crean sin errores
- [ ] La aplicacion se recupera automaticamente tras terminacion de instancia
- [ ] El trafico se redirige correctamente durante fallo simulado de AZ
- [ ] El failover de RDS completa en tiempo aceptable
- [ ] Las stop conditions detienen el experimento si se exceden umbrales
