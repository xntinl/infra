# 20.05 OPA Policy Testing

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform + OPA |
| **Features TF** | `terraform show -json`, Rego testing |

## Descripcion

Escribe tests comprehensivos de OPA para tus politicas de Terraform. Usa `opa test` con datos de plan simulados. Testea escenarios permitidos y denegados. Alcanza cobertura completa de las reglas de politica. Integra los tests en CI.

## Objetivos de aprendizaje

- Testear politicas de infraestructura escritas en Rego de forma exhaustiva
- Crear datos de plan simulados para reproducir escenarios especificos
- Alcanzar cobertura completa de reglas de politica
- Integrar tests de politicas en el pipeline de CI

## Requisitos previos

- Conocimientos: Terraform intermedio, OPA y lenguaje Rego basico, conceptos de policy-as-code, CI/CD
- Herramientas: Terraform >= 1.6, OPA CLI, conftest (opcional), herramienta de CI (GitHub Actions, etc.)
- Features: `terraform show -json` para generar plan JSON, `opa test` para ejecutar tests

## Arquitectura

Estructura de politicas OPA con tests:

```
policies/
  terraform/
    deny_public_s3.rego
    require_tags.rego
    restrict_instance_types.rego
    enforce_encryption.rego
  terraform_test/
    deny_public_s3_test.rego
    require_tags_test.rego
    restrict_instance_types_test.rego
    enforce_encryption_test.rego
  testdata/
    plan_public_s3.json
    plan_private_s3.json
    plan_missing_tags.json
    plan_valid_tags.json
```

Cada politica tiene su archivo de test correspondiente con mock plan data que simula outputs de `terraform show -json`. Los tests cubren escenarios allowed y denied.

## Pasos sugeridos

1. Escribir politicas Rego para Terraform: deny public S3, require tags, restrict instance types, enforce encryption
2. Generar un plan JSON real con `terraform plan -out=plan.tfplan && terraform show -json plan.tfplan`
3. Crear datos de plan simulados (testdata) para escenarios permitidos y denegados
4. Escribir tests Rego para cada politica usando los datos simulados
5. Ejecutar `opa test` y verificar que todos los tests pasan
6. Agregar tests negativos: verificar que configuraciones validas NO son denegadas
7. Ejecutar `opa test --coverage` y asegurar cobertura completa de todas las reglas
8. Integrar `opa test` en el pipeline de CI (GitHub Actions o similar)
9. Configurar conftest para evaluar politicas contra plans reales en el pipeline
10. Documentar las politicas y sus tests para el equipo

## Validacion

- [ ] Las politicas Rego evaluan correctamente planes de Terraform
- [ ] Los datos de plan simulados representan escenarios realistas (allowed y denied)
- [ ] Los tests positivos verifican que las politicas deniegan configuraciones inseguras
- [ ] Los tests negativos verifican que las configuraciones validas son permitidas
- [ ] `opa test` ejecuta todos los tests sin fallos
- [ ] `opa test --coverage` muestra cobertura completa (o cercana al 100%)
- [ ] Los tests estan integrados en el pipeline de CI y bloquean merges si fallan
- [ ] conftest evalua planes reales contra las politicas en el pipeline
