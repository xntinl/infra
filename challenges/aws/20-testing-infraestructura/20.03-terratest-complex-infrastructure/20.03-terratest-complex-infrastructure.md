# 20.03 Terratest para infraestructura compleja

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform + Go |
| **Features TF** | Terratest stages, retry, AWS SDK validation |

## Descripcion

Escribe tests con Terratest (Go) para infraestructura multi-modulo: despliega VPC + ECS + RDS, valida conectividad entre capas, testea comportamiento de auto-scaling y verifica security group rules. Usa retry logic y test stages para eficiencia.

## Objetivos de aprendizaje

- Escribir tests avanzados con Terratest en Go para infraestructura compleja
- Implementar test stages para reutilizar infraestructura entre tests
- Aplicar patrones de retry y validacion con AWS SDK
- Validar conectividad y comportamiento de red entre componentes

## Requisitos previos

- Conocimientos: Terraform avanzado, Go basico-intermedio, Terratest fundamentals, networking AWS
- Herramientas: Terraform >= 1.6, Go >= 1.21, AWS CLI, AWS SDK for Go
- Features: Terratest (terraform, aws, retry, stages), AWS SDK for Go v2

## Arquitectura

Tests de Terratest para una infraestructura multi-capa:

```
project/
  modules/
    vpc/
    ecs/
    rds/
  test/
    infra_test.go
    vpc_test.go
    connectivity_test.go
    go.mod
    go.sum
```

La infraestructura bajo test consiste en VPC (3 AZs) + ECS Fargate + RDS PostgreSQL. Los tests validan no solo que los recursos existen, sino que funcionan correctamente juntos (conectividad, auto-scaling, security groups).

## Pasos sugeridos

1. Configurar el proyecto Go con las dependencias de Terratest y AWS SDK
2. Crear los modulos de Terraform (VPC, ECS, RDS) que seran testeados
3. Implementar test stages para separar el deploy de la validacion (evitar redespliegues)
4. Escribir tests que desplieguen la infraestructura completa con `terraform.InitAndApply`
5. Validar la VPC con AWS SDK: verificar subnets, route tables, NAT Gateways
6. Testear conectividad entre capas: ECS puede alcanzar RDS en el puerto configurado
7. Verificar security group rules programaticamente con AWS SDK
8. Implementar tests de auto-scaling: simular carga y verificar que ECS escala
9. Agregar retry logic para validaciones que requieren tiempo de convergencia
10. Ejecutar la suite completa y medir tiempos, optimizar con paralelismo donde sea posible

## Validacion

- [ ] El proyecto Go compila y las dependencias de Terratest estan correctamente configuradas
- [ ] Los test stages permiten reutilizar infraestructura desplegada entre tests
- [ ] Los tests despliegan la infraestructura multi-modulo exitosamente
- [ ] La validacion con AWS SDK verifica propiedades reales de los recursos
- [ ] Los tests de conectividad confirman que ECS puede comunicarse con RDS
- [ ] Las security group rules son verificadas programaticamente
- [ ] El retry logic maneja correctamente los tiempos de convergencia
- [ ] La infraestructura se destruye completamente al finalizar los tests
- [ ] Los tiempos de ejecucion son razonables gracias a stages y paralelismo
