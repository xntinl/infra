# 20.09 Post-Deploy Application Testing

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Intermedio |
| **Modalidad** | Terraform + scripts |
| **Features TF** | `local-exec`, `terraform_data`, outputs |

## Descripcion

Despues de ejecutar `terraform apply`, ejecuta automaticamente tests a nivel de aplicacion: verificacion de endpoints de health check, smoke tests via curl/wget, conectividad a base de datos y validacion de certificados SSL. Usa el provisioner `local-exec` o una etapa separada de CI para orquestar las validaciones.

## Objetivos de aprendizaje

- Implementar validaciones automaticas post-despliegue
- Usar provisioners y `terraform_data` para ejecutar tests tras el apply
- Disenar smoke tests que verifiquen la salud de la aplicacion
- Integrar validaciones post-deploy en pipelines CI/CD

## Requisitos previos

- Conocimientos: Terraform intermedio, scripting (bash/python), HTTP/HTTPS
- Herramientas: Terraform >= 1.0, AWS CLI, curl, jq
- Features TF: Outputs, provisioners, `terraform_data`

## Arquitectura

Desplegar infraestructura con Terraform (ALB + ECS/Lambda + RDS). Usar outputs de Terraform para obtener endpoints. Ejecutar scripts de validacion via `local-exec` o `terraform_data` que verifican health checks, conectividad y certificados SSL.

## Pasos sugeridos

1. Desplegar una aplicacion con Terraform que exponga un endpoint HTTP/HTTPS
2. Definir outputs para los endpoints (ALB DNS, API URL, etc.)
3. Crear un script de smoke test que valide: HTTP 200, response body, latencia
4. Implementar `terraform_data` con `local-exec` que ejecute el script post-apply
5. Agregar validacion de certificado SSL con openssl/curl
6. Agregar validacion de conectividad a base de datos (si aplica)
7. Configurar reintentos con backoff para esperar que los servicios esten listos
8. Integrar los tests como un stage separado en el pipeline CI/CD

## Validacion

- [ ] Los outputs de Terraform proporcionan los endpoints correctos
- [ ] El smoke test se ejecuta automaticamente despues de `terraform apply`
- [ ] El health check endpoint responde con HTTP 200
- [ ] La validacion SSL verifica el certificado correctamente
- [ ] Los reintentos manejan el tiempo de arranque de los servicios
- [ ] Un fallo en los tests se reporta claramente en la salida
