# 22.02 Comprehensive Tagging con default_tags

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Intermedio |
| **Modalidad** | Terraform |
| **Features TF** | default_tags, tag validation, aws_resourcegroupstaggingapi |

## Descripcion

Implementa una estrategia de etiquetado usando `default_tags` del provider. Define tags obligatorios (project, environment, owner, cost-center). Construye una verificacion de cumplimiento que falle si los recursos carecen de tags requeridos. Genera reportes de asignacion de costos por tag.

## Objetivos de aprendizaje

- Estrategia de etiquetado para infraestructura
- Asignacion de costos basada en tags
- Cumplimiento y validacion de tags

## Requisitos previos

- Conocimientos: Terraform basico, conceptos de cost allocation en AWS
- Herramientas: Terraform CLI, AWS CLI
- Servicios/Features: AWS Provider default_tags, Resource Groups Tagging API, AWS Cost Explorer

## Arquitectura

La estrategia de tagging se implementa en tres capas. La primera es la capa de provider con `default_tags`, que aplica automaticamente los tags obligatorios a todos los recursos. La segunda es la capa de validacion, que usa `precondition` blocks o un script externo para verificar que ningun recurso se despliegue sin los tags requeridos. La tercera es la capa de reporte, que usa la Resource Groups Tagging API y Cost Explorer para generar reportes de asignacion de costos agrupados por tags.

## Pasos sugeridos

1. Configurar el bloque `default_tags` en el provider de AWS con los tags obligatorios (project, environment, owner, cost-center)
2. Desplegar varios tipos de recursos para verificar que los tags se aplican automaticamente
3. Implementar variables de Terraform para los valores de cada tag obligatorio
4. Crear validaciones con `precondition` o `validation` blocks que impidan valores vacios en los tags
5. Usar la Resource Groups Tagging API para listar recursos y verificar el cumplimiento de tags
6. Desarrollar un script o funcion Lambda que identifique recursos sin los tags requeridos
7. Configurar AWS Cost Explorer para agrupar costos por los tags definidos
8. Generar un reporte de asignacion de costos por project y cost-center
9. Probar el escenario de fallo: intentar desplegar sin un tag obligatorio y verificar que Terraform rechaza el plan
10. Documentar la estrategia de tagging y los valores permitidos para cada tag

## Validacion

- [ ] El bloque `default_tags` aplica los cuatro tags obligatorios a todos los recursos
- [ ] Las validaciones de Terraform impiden desplegar recursos sin tags requeridos
- [ ] La Resource Groups Tagging API confirma que todos los recursos tienen los tags correctos
- [ ] Se genera un reporte de costos agrupado por tag de project
- [ ] Se genera un reporte de costos agrupado por tag de cost-center
- [ ] Los recursos que no soportan `default_tags` reciben los tags explicitamente
