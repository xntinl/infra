# 22.01 Infracost CI Integration

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Intermedio |
| **Modalidad** | Terraform + Infracost |
| **Features TF** | Plan JSON, Infracost CLI |

## Descripcion

Integra Infracost en tu pipeline de CI. Genera estimaciones de costo para cada PR. Muestra el diff de costos (antes/despues) como comentario en el PR. Configura politicas de costos que rechacen PRs que excedan un umbral. Realiza seguimiento de tendencias de costos por modulo.

## Objetivos de aprendizaje

- Estimacion de costos para IaC
- Visibilidad de costos basada en PRs
- Politicas de costos automatizadas

## Requisitos previos

- Conocimientos: Terraform basico, CI/CD (GitHub Actions o similar), YAML
- Herramientas: Terraform CLI, Infracost CLI, GitHub Actions o GitLab CI
- Servicios/Features: Infracost API, plan JSON de Terraform, GitHub API (para comentarios en PRs)

## Arquitectura

El pipeline de CI se extiende con un paso de Infracost. Cuando se abre un PR con cambios en Terraform, el pipeline genera el plan JSON, Infracost lo analiza para estimar costos, calcula el diff respecto a la rama principal, publica el resultado como comentario en el PR, y evalua las politicas de costos definidas. Si el incremento de costo supera el umbral configurado, el pipeline falla e impide el merge.

## Pasos sugeridos

1. Instalar Infracost CLI localmente y configurar la API key
2. Crear una configuracion de Terraform con recursos que tengan costo (EC2, RDS, NAT Gateway)
3. Ejecutar `infracost breakdown` localmente para ver la estimacion base
4. Configurar el workflow de GitHub Actions (o el CI de tu preferencia) con el paso de Infracost
5. Implementar la generacion del diff de costos entre la rama principal y el PR
6. Configurar la publicacion automatica del comentario de costos en el PR
7. Definir politicas de costos: umbral maximo de incremento mensual permitido
8. Crear un PR que exceda el umbral y verificar que el pipeline falla
9. Configurar el tracking de tendencias de costos por modulo usando Infracost Cloud o archivos JSON
10. Documentar el flujo completo y los umbrales definidos

## Validacion

- [ ] Infracost genera estimaciones de costo correctas para la configuracion de Terraform
- [ ] El pipeline de CI ejecuta Infracost automaticamente en cada PR
- [ ] Se publica un comentario en el PR con el diff de costos (antes/despues)
- [ ] Un PR que excede el umbral de costos falla en el pipeline
- [ ] Un PR dentro del umbral pasa exitosamente
- [ ] Se puede ver el desglose de costos por modulo/recurso
