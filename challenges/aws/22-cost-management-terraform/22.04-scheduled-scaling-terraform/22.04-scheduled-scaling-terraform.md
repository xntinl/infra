# 22.04 Scheduled Scaling con Terraform

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Intermedio |
| **Modalidad** | Terraform |
| **Servicios AWS** | Auto Scaling, RDS, ECS, EventBridge, Lambda |

## Descripcion

Implementa escalado programado para ahorro de costos: reduce los entornos de dev/staging fuera del horario laboral usando acciones programadas de Auto Scaling, stop/start de RDS via Lambda, y ajustes del desired count de servicios ECS. Todo gestionado mediante Terraform.

## Objetivos de aprendizaje

- Patrones de escalado programado
- Ahorro de costos en entornos no productivos
- Automatizacion de escalado via IaC

## Requisitos previos

- Conocimientos: Terraform basico, Auto Scaling, ECS, RDS, expresiones cron
- Herramientas: Terraform CLI, AWS CLI
- Servicios/Features: Auto Scaling Groups, RDS, ECS, EventBridge Scheduler, Lambda

## Arquitectura

La solucion gestiona tres tipos de escalado programado. Para EC2/Auto Scaling, se configuran scheduled actions que reducen min/max/desired a valores minimos fuera de horario (ej: 19:00-07:00) y los restauran en horario laboral. Para RDS, una funcion Lambda invocada por EventBridge detiene las instancias por la noche y las reinicia por la manana. Para ECS, otra Lambda o scheduled action ajusta el desired count de los servicios a cero fuera de horario. Todo se define en Terraform con variables configurables para los horarios.

## Pasos sugeridos

1. Crear un Auto Scaling Group con Terraform y una launch template basica
2. Configurar `aws_autoscaling_schedule` para reducir la capacidad fuera de horario laboral
3. Configurar otra schedule para restaurar la capacidad en horario laboral
4. Desplegar una instancia RDS de desarrollo via Terraform
5. Crear la funcion Lambda que detenga/inicie la instancia RDS usando el SDK
6. Configurar las reglas de EventBridge Scheduler para invocar la Lambda en los horarios definidos
7. Desplegar un servicio ECS con Terraform
8. Implementar el ajuste programado del desired count del servicio ECS
9. Parametrizar todos los horarios con variables de Terraform (expresiones cron)
10. Calcular el ahorro estimado basado en las horas de reduccion/detencion

## Validacion

- [ ] El Auto Scaling Group reduce su capacidad fuera del horario laboral
- [ ] El Auto Scaling Group restaura su capacidad al inicio del horario laboral
- [ ] La instancia RDS se detiene automaticamente por la noche
- [ ] La instancia RDS se reinicia automaticamente por la manana
- [ ] El servicio ECS reduce su desired count a cero fuera de horario
- [ ] Los horarios son configurables mediante variables de Terraform
