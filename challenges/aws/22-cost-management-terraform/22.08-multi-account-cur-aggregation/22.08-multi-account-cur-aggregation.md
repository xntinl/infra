# 22.08 Multi-Account CUR Aggregation

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform |
| **Servicios AWS** | CUR, S3, Glue, Athena, QuickSight, Organizations |

## Descripcion

Configura Cost and Usage Reports a traves de una organizacion. Despliega Glue crawler y tablas de Athena via Terraform. Construye consultas de asignacion de costos por cuenta, equipo y servicio. Crea un dashboard de QuickSight para visibilidad de costos organizacional.

## Objetivos de aprendizaje

- Visibilidad de costos multi-cuenta
- Analisis de CUR con Athena
- FinOps organizacional

## Requisitos previos

- Conocimientos: Terraform intermedio, AWS Organizations, SQL basico, conceptos de data lake
- Herramientas: Terraform CLI, AWS CLI, SQL
- Servicios/Features: Cost and Usage Reports, S3, Glue, Athena, QuickSight, AWS Organizations

## Arquitectura

El Cost and Usage Report (CUR) se configura en la cuenta de gestion (management account) de la organizacion y entrega archivos Parquet a un bucket S3 centralizado. Un Glue Crawler descubre el esquema de los archivos CUR y crea/actualiza la tabla en el Glue Data Catalog. Athena permite ejecutar consultas SQL sobre los datos CUR para analizar costos por cuenta, servicio, region y tags. QuickSight se conecta a Athena como fuente de datos y presenta dashboards interactivos con desglose de costos por equipo, cuenta y servicio, incluyendo tendencias y comparaciones mes a mes.

## Pasos sugeridos

1. Crear el bucket S3 centralizado para recibir los datos de CUR con la politica de acceso apropiada
2. Configurar el Cost and Usage Report via Terraform con formato Parquet y granularidad horaria
3. Desplegar el Glue Database y el Glue Crawler que procese los archivos CUR del bucket S3
4. Ejecutar el crawler y verificar que la tabla se crea correctamente en el Glue Data Catalog
5. Configurar el workgroup de Athena con un bucket de resultados y cifrado
6. Escribir consultas Athena para desglose de costos por cuenta y servicio
7. Escribir consultas para asignacion de costos por tags (equipo, proyecto, cost-center)
8. Crear named queries en Athena para las consultas mas comunes via Terraform
9. Configurar QuickSight con Athena como data source y crear el dataset
10. Construir el dashboard de QuickSight con visualizaciones de costos por cuenta, servicio, equipo y tendencias

## Validacion

- [ ] El CUR se configura y entrega archivos Parquet al bucket S3 centralizado
- [ ] El Glue Crawler descubre el esquema y crea la tabla en el Data Catalog
- [ ] Las consultas Athena retornan datos de costos desglosados por cuenta y servicio
- [ ] Se puede filtrar y agrupar costos por tags de asignacion (equipo, proyecto)
- [ ] El dashboard de QuickSight muestra visualizaciones interactivas de costos
- [ ] Toda la infraestructura del pipeline de datos se gestiona via Terraform
