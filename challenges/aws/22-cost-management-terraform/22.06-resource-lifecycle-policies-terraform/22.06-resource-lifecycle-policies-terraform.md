# 22.06 Resource Lifecycle Policies con Terraform

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Intermedio |
| **Modalidad** | Terraform |
| **Servicios AWS** | S3, EBS, ECR, RDS, CloudWatch |

## Descripcion

Implementa ciclos de vida automatizados para recursos: reglas de lifecycle en S3, lifecycle de snapshots de EBS, lifecycle de imagenes en ECR, retencion de snapshots de RDS, y retencion de logs en CloudWatch. Construye un modulo que encapsule las mejores practicas de lifecycle.

## Objetivos de aprendizaje

- Gestion del ciclo de vida de recursos
- Optimizacion de costos de almacenamiento
- Politicas de retencion automatizadas

## Requisitos previos

- Conocimientos: Terraform basico, S3 storage classes, conceptos de snapshots, ECR
- Herramientas: Terraform CLI, AWS CLI
- Servicios/Features: S3 Lifecycle Rules, EBS Snapshots, ECR Lifecycle Policies, RDS Snapshots, CloudWatch Logs

## Arquitectura

Se construye un modulo de Terraform reutilizable que encapsula las mejores practicas de lifecycle para cada servicio. El modulo incluye: reglas de S3 que transicionan objetos entre storage classes (Standard -> IA -> Glacier -> Delete), Data Lifecycle Manager para snapshots de EBS con retencion configurable, politicas de ECR que eliminan imagenes antiguas o sin tag, configuracion de retencion de backups automaticos de RDS, y retencion de log groups de CloudWatch. Cada politica es configurable mediante variables.

## Pasos sugeridos

1. Crear un bucket S3 con reglas de lifecycle que transicionen objetos: 30 dias a IA, 90 dias a Glacier, 365 dias eliminacion
2. Configurar reglas de lifecycle para versiones no actuales del bucket S3
3. Crear un DLM (Data Lifecycle Manager) policy para snapshots de EBS con retencion de 7 dias
4. Configurar politicas de lifecycle en un repositorio ECR: mantener solo las ultimas 10 imagenes taggeadas
5. Configurar la retencion de backups automaticos de RDS a 7 dias y snapshots manuales
6. Establecer la retencion de CloudWatch Log Groups a 30 dias para dev y 90 dias para produccion
7. Encapsular todas las politicas en un modulo de Terraform reutilizable con variables configurables
8. Agregar outputs del modulo que muestren las politicas configuradas
9. Desplegar el modulo y verificar que todas las politicas se aplican correctamente
10. Estimar el ahorro de costos comparando almacenamiento sin lifecycle vs con lifecycle

## Validacion

- [ ] Las reglas de lifecycle de S3 transicionan objetos entre storage classes segun los plazos definidos
- [ ] DLM crea y elimina snapshots de EBS automaticamente segun la retencion configurada
- [ ] ECR elimina imagenes antiguas segun la politica de lifecycle
- [ ] Los backups automaticos de RDS respetan la retencion configurada
- [ ] Los CloudWatch Log Groups tienen la retencion correcta segun el entorno
- [ ] El modulo de Terraform es reutilizable y configurable mediante variables
