# 22.07 Cost Anomaly Detection y Budget Enforcement

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform |
| **Servicios AWS** | AWS Budgets, Cost Anomaly Detection, IAM, SNS |

## Descripcion

Despliega AWS Budgets y Cost Anomaly Detection via Terraform. Configura budget actions que automaticamente restrinjan politicas IAM cuando se excedan los presupuestos. Configura monitores de anomalias por servicio con alertas SNS.

## Objetivos de aprendizaje

- Automatizacion de presupuestos en AWS
- Deteccion de anomalias de costos
- Controles automaticos de costos mediante IAM

## Requisitos previos

- Conocimientos: Terraform intermedio, AWS Budgets, IAM policies, SNS
- Herramientas: Terraform CLI, AWS CLI
- Servicios/Features: AWS Budgets, Cost Anomaly Detection, IAM, SNS, Cost Explorer

## Arquitectura

La solucion tiene dos componentes principales. El primero es AWS Budgets con acciones automaticas: se definen presupuestos mensuales por cuenta o por servicio, y cuando el gasto supera umbrales definidos (80%, 100%, 120%), se ejecutan acciones progresivas: notificacion SNS, aplicacion de una SCP restrictiva o IAM policy que impida la creacion de nuevos recursos, y notificacion de emergencia. El segundo componente es Cost Anomaly Detection: monitores configurados por servicio que detectan patrones de gasto inusuales y envian alertas inmediatas via SNS con el detalle de la anomalia.

## Pasos sugeridos

1. Crear un topico SNS y suscripciones para las alertas de costos
2. Desplegar un AWS Budget mensual con Terraform definiendo el monto limite
3. Configurar notificaciones en multiples umbrales (50%, 80%, 100%)
4. Crear una IAM policy restrictiva que bloquee la creacion de recursos costosos
5. Configurar una budget action que aplique la IAM policy restrictiva al superar el 100% del presupuesto
6. Desplegar un Cost Anomaly Detection monitor para los servicios principales (EC2, RDS, S3)
7. Configurar las suscripciones de anomalias con umbral de alerta en dolares
8. Crear monitores de anomalias separados por equipo o proyecto usando tags de cost allocation
9. Probar el flujo de budget action: verificar que la politica restrictiva se aplica correctamente
10. Configurar un dashboard de CloudWatch que muestre el estado de los presupuestos

## Validacion

- [ ] Los AWS Budgets estan desplegados con los montos y umbrales correctos
- [ ] Las notificaciones SNS se envian al alcanzar cada umbral de presupuesto
- [ ] La budget action aplica la IAM policy restrictiva al exceder el presupuesto
- [ ] Los monitores de Cost Anomaly Detection estan activos por servicio
- [ ] Las alertas de anomalias se envian via SNS con el detalle del gasto inusual
- [ ] Toda la configuracion se gestiona exclusivamente via Terraform
