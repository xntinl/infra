# 22.05 Spot Instance Integration con Terraform

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform |
| **Servicios AWS** | EC2 Spot, ASG, ECS, EKS, EMR |

## Descripcion

Implementa instancias Spot en diferentes casos de uso: politica de instancias mixtas en ASG, Capacity Providers con Spot en ECS, node groups con Spot en EKS, y nodos de tarea en EMR. Compara el ahorro de costos para cada enfoque.

## Objetivos de aprendizaje

- Patrones de integracion de instancias Spot
- Medicion de ahorro de costos con Spot
- Manejo de interrupciones de instancias Spot

## Requisitos previos

- Conocimientos: Terraform intermedio, EC2 Spot, conceptos de ASG, ECS, EKS y EMR
- Herramientas: Terraform CLI, AWS CLI, kubectl (para EKS)
- Servicios/Features: EC2 Spot Instances, Auto Scaling Groups, ECS, EKS, EMR

## Arquitectura

Se implementan cuatro patrones de uso de Spot en paralelo. El primero es un ASG con mixed instances policy que combina On-Demand y Spot con multiples tipos de instancia. El segundo es un cluster ECS con Capacity Providers que incluye un provider Spot para tareas tolerantes a fallos. El tercero es un cluster EKS con un node group dedicado a Spot instances para workloads batch. El cuarto son clusters EMR con task nodes en Spot para procesamiento de datos. Cada patron incluye configuraciones de manejo de interrupciones apropiadas.

## Pasos sugeridos

1. Crear un ASG con mixed instances policy que combine 30% On-Demand y 70% Spot
2. Configurar multiples tipos de instancia en el override para maximizar disponibilidad de Spot
3. Implementar la allocation strategy (capacity-optimized) y el manejo de rebalance
4. Desplegar un cluster ECS con un Capacity Provider basado en un ASG Spot
5. Configurar un servicio ECS que use el Capacity Provider Spot para tareas no criticas
6. Crear un cluster EKS con un managed node group de Spot instances
7. Configurar taints y tolerations para dirigir workloads especificos a los nodos Spot
8. Desplegar un cluster EMR con task nodes en Spot para procesamiento batch
9. Configurar el manejo de terminacion de Spot en cada caso (lifecycle hooks, draining, etc.)
10. Comparar los precios On-Demand vs Spot para cada caso y calcular el ahorro estimado

## Validacion

- [ ] El ASG tiene una mixed instances policy funcional con instancias Spot activas
- [ ] El ECS Capacity Provider Spot ejecuta tareas correctamente
- [ ] El EKS node group Spot registra nodos en el cluster
- [ ] El cluster EMR tiene task nodes Spot ejecutando pasos
- [ ] Se configuraron estrategias de manejo de interrupciones en cada caso
- [ ] Se documenta el ahorro estimado comparando precios On-Demand vs Spot
