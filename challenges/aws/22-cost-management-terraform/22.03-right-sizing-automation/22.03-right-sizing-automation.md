# 22.03 Right-Sizing Automation

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform + Lambda |
| **Servicios AWS** | Compute Optimizer, EC2, Lambda |

## Descripcion

Construye una configuracion de Terraform que lea las recomendaciones de Compute Optimizer mediante data sources y ajuste automaticamente los tipos de instancia. Implementa un flujo de revision y aprobacion antes de aplicar los cambios de tamano.

## Objetivos de aprendizaje

- Right-sizing automatizado de instancias
- Consumo de recomendaciones de AWS Compute Optimizer
- Flujos de trabajo de optimizacion de costos

## Requisitos previos

- Conocimientos: Terraform intermedio, AWS Lambda, Compute Optimizer, tipos de instancia EC2
- Herramientas: Terraform CLI, AWS CLI, Python para Lambda
- Servicios/Features: AWS Compute Optimizer, EC2, Lambda, SNS, SSM Parameter Store

## Arquitectura

Una funcion Lambda se ejecuta periodicamente y consulta las recomendaciones de Compute Optimizer para todas las instancias EC2. Las recomendaciones se almacenan en SSM Parameter Store como pares instancia-ID/tipo-recomendado. La configuracion de Terraform lee estos parametros via data sources y los usa para definir el `instance_type` de cada EC2. Un pipeline de CI genera el plan, lo envia para aprobacion humana via SNS, y solo despues de la aprobacion se ejecuta el apply.

## Pasos sugeridos

1. Desplegar instancias EC2 con Terraform usando tipos de instancia deliberadamente sobredimensionados
2. Habilitar Compute Optimizer en la cuenta de AWS y esperar a que genere recomendaciones
3. Crear la funcion Lambda que consulte las recomendaciones de Compute Optimizer via SDK
4. Almacenar las recomendaciones en SSM Parameter Store con un esquema consistente
5. Modificar la configuracion de Terraform para leer los tipos de instancia recomendados via `aws_ssm_parameter` data source
6. Implementar la logica condicional: usar la recomendacion solo si existe, sino mantener el tipo actual
7. Configurar el pipeline de CI que genere el plan de cambios de tamano
8. Implementar el flujo de aprobacion: enviar notificacion SNS con el detalle del cambio y esperar confirmacion
9. Aplicar los cambios aprobados y verificar que las instancias se redimensionan correctamente
10. Medir el ahorro de costos comparando los tipos antes y despues del right-sizing

## Validacion

- [ ] Compute Optimizer genera recomendaciones para las instancias desplegadas
- [ ] La funcion Lambda extrae las recomendaciones y las almacena en SSM Parameter Store
- [ ] Terraform lee los parametros de SSM y genera un plan con los cambios de tipo de instancia
- [ ] El flujo de aprobacion funciona: se envia notificacion y se espera confirmacion
- [ ] Las instancias se redimensionan correctamente despues del apply
- [ ] Se puede cuantificar el ahorro de costos estimado por el cambio de tipo
