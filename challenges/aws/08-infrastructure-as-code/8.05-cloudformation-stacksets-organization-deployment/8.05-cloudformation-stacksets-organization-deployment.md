# 8.05 CloudFormation StackSets - Organization Deployment

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | CloudFormation |
| **Servicios AWS** | CloudFormation StackSets, Organizations, CloudTrail, Config, GuardDuty |

## Descripcion

Usar StackSets con permisos service-managed para desplegar baselines de seguridad en todas las cuentas de una Organization. Desplegar CloudTrail, Config rules, habilitacion de GuardDuty y un topic SNS de notificaciones de seguridad en todas las cuentas miembro.

## Objetivos de aprendizaje

- Gestionar infraestructura como codigo en entornos multi-cuenta con StackSets
- Disenar estrategias de despliegue con StackSets (concurrencia, orden regional, tolerancia a fallos)
- Configurar permisos service-managed para despliegues automaticos en Organizations
- Comprender el ordenamiento regional en despliegues multi-region

## Requisitos previos

- Conocimientos: CloudFormation (templates, stacks), AWS Organizations (OUs, SCPs), seguridad AWS basica
- Herramientas: AWS CLI, AWS Console, editor de YAML/JSON
- Servicios AWS: Cuenta de management de Organizations con permisos para StackSets, CloudTrail, Config, GuardDuty

## Arquitectura

Desde la cuenta de management de AWS Organizations, se despliegan StackSets con permisos service-managed. Cada StackSet contiene un template de CloudFormation que define los recursos de seguridad baseline. Los StackSets se asocian a OUs especificas, y automaticamente despliegan stack instances en todas las cuentas miembro de esas OUs (incluyendo cuentas nuevas). Los templates despliegan: CloudTrail con logs centralizados en S3, AWS Config con rules de compliance, GuardDuty habilitado con findings centralizados, y un topic SNS para notificaciones de seguridad.

## Pasos sugeridos

1. Verificar la estructura de Organizations (OUs existentes) y habilitar trusted access para StackSets
2. Crear el template de CloudFormation para CloudTrail (trail, bucket S3 centralizado, policy)
3. Crear el template para AWS Config (recorder, delivery channel, rules basicas de compliance)
4. Crear el template para GuardDuty (detector habilitado)
5. Crear el template para el topic SNS de notificaciones de seguridad con subscripciones
6. Crear un StackSet con permisos service-managed y asociarlo a las OUs target
7. Configurar los parametros de despliegue: concurrencia maxima, tolerancia a fallos, orden regional
8. Desplegar los StackSets y monitorear el progreso en cada cuenta/region
9. Verificar que los recursos se crearon correctamente en las cuentas miembro
10. Probar el despliegue automatico creando una nueva cuenta en la OU y verificando que recibe el baseline

## Validacion

- [ ] Los StackSets se crean con permisos service-managed correctamente
- [ ] Los stack instances se despliegan en todas las cuentas de las OUs target
- [ ] CloudTrail esta habilitado en todas las cuentas miembro
- [ ] AWS Config recorder esta activo con las rules de compliance configuradas
- [ ] GuardDuty esta habilitado en todas las cuentas
- [ ] Los topics SNS se crean con las subscripciones correctas
- [ ] Las cuentas nuevas en la OU reciben automaticamente el baseline de seguridad
- [ ] Los despliegues multi-region siguen el orden configurado
