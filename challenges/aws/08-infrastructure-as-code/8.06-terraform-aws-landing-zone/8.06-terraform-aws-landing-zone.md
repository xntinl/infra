# 8.06 Terraform AWS Landing Zone

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Experto |
| **Modalidad** | Terraform |
| **Servicios AWS** | Organizations, Control Tower, IAM Identity Center, Transit Gateway, CloudTrail, GuardDuty |

## Descripcion

Construir una landing zone empresarial con Terraform: AWS Organizations, OUs, cuentas provisionadas via Account Factory, SCPs, logging centralizado (CloudTrail + Config), baselines de seguridad (GuardDuty, Security Hub) y una VPC de networking compartida con Transit Gateway.

## Objetivos de aprendizaje

- Disenar una arquitectura AWS empresarial multi-cuenta siguiendo las mejores practicas
- Implementar una estrategia multi-cuenta con Organizations, OUs y SCPs
- Automatizar el aprovisionamiento de cuentas y la aplicacion de baselines a escala
- Configurar baselines de seguridad y networking compartido para toda la organizacion

## Requisitos previos

- Conocimientos: Terraform avanzado, AWS Organizations, networking (VPC, Transit Gateway), seguridad AWS, IAM
- Herramientas: Terraform >= 1.5, AWS CLI, Git, cuenta AWS con billing habilitado
- Servicios AWS: Cuenta root de Organizations con permisos completos para todos los servicios de la landing zone

## Arquitectura

La landing zone se estructura en capas. La capa de Organizations define la jerarquia de OUs (Security, Infrastructure, Workloads, Sandbox) y SCPs que limitan acciones peligrosas. La capa de identidad configura IAM Identity Center con permission sets y grupos de acceso. La capa de seguridad despliega GuardDuty, Security Hub y CloudTrail con logs centralizados en una cuenta de Log Archive. La capa de networking crea una VPC compartida con Transit Gateway que conecta todas las cuentas de workload, con subnets compartidas via RAM. Account Factory automatiza la creacion de nuevas cuentas con baselines aplicados automaticamente.

## Pasos sugeridos

1. Crear la cuenta de management y habilitar AWS Organizations
2. Disenar y crear la estructura de OUs (Security, Infrastructure, Workloads/Dev, Workloads/Prod, Sandbox)
3. Definir y aplicar SCPs por OU (denegar regiones no autorizadas, proteger roles de seguridad)
4. Crear las cuentas foundacionales: Log Archive, Security Audit, Shared Services, Network
5. Configurar IAM Identity Center con permission sets (Admin, ReadOnly, PowerUser) y asignaciones por OU
6. Desplegar CloudTrail organizacional con logs centralizados en la cuenta de Log Archive
7. Habilitar GuardDuty y Security Hub con administrador delegado en la cuenta de Security Audit
8. Crear la VPC de networking compartido con Transit Gateway en la cuenta de Network
9. Configurar AWS RAM para compartir subnets con las cuentas de workload
10. Implementar Account Factory con Terraform para provisionar nuevas cuentas con baselines automaticos

## Validacion

- [ ] La estructura de Organizations con OUs esta creada correctamente
- [ ] Las SCPs se aplican y bloquean acciones no autorizadas
- [ ] Las cuentas foundacionales estan creadas y configuradas
- [ ] IAM Identity Center permite acceso federado a todas las cuentas
- [ ] CloudTrail centralizado captura eventos de todas las cuentas
- [ ] GuardDuty y Security Hub estan habilitados en todas las cuentas
- [ ] Transit Gateway conecta las VPCs de las cuentas de workload
- [ ] Las subnets compartidas son accesibles desde las cuentas de workload
- [ ] Account Factory puede provisionar nuevas cuentas con baselines aplicados
- [ ] La infraestructura completa se gestiona con Terraform y es reproducible
