# 8.04 Terraform Drift Detection y Remediation Pipeline

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform + CodeBuild |
| **Servicios AWS** | CodeBuild, EventBridge (schedule), SNS, S3, Lambda |

## Descripcion

Construir un pipeline automatizado de deteccion de drift: un job programado en CodeBuild ejecuta `terraform plan` y parsea el output. Si se detecta drift, crear una notificacion con el diff. Opcionalmente auto-remediar aplicando si el drift coincide con patrones conocidos.

## Objetivos de aprendizaje

- Comprender el concepto de infrastructure drift y sus causas comunes
- Automatizar la deteccion de drift con pipelines de CI/CD
- Disenar estrategias de remediacion (manual vs automatica) segun el tipo de drift
- Parsear y analizar la salida de `terraform plan` para tomar decisiones automatizadas
- Implementar governance operacional sobre la infraestructura

## Requisitos previos

- Conocimientos: Terraform intermedio, CodeBuild (buildspec), EventBridge schedules, scripting (bash/Python)
- Herramientas: Terraform >= 1.5, AWS CLI, jq, Python (opcional)
- Servicios AWS: Cuenta AWS con permisos para CodeBuild, EventBridge, SNS, S3, Lambda, IAM

## Arquitectura

EventBridge Scheduler ejecuta un proyecto de CodeBuild en intervalos programados (cada hora o diariamente). CodeBuild clona el repositorio de Terraform, inicializa el backend y ejecuta `terraform plan -detailed-exitcode`. Si el exit code indica cambios (drift), el pipeline parsea el output JSON del plan para identificar los recursos afectados. Una funcion Lambda procesa el resultado, genera una notificacion detallada con el diff y la envia via SNS. Para patrones de drift conocidos (ejemplo: tags modificados manualmente), el pipeline puede auto-remediar ejecutando `terraform apply` de forma selectiva.

## Pasos sugeridos

1. Crear una infraestructura base con Terraform (VPC, security groups, S3 buckets) como target de deteccion
2. Crear un proyecto de CodeBuild con un buildspec que ejecute `terraform plan -detailed-exitcode -json`
3. Configurar el IAM role de CodeBuild con permisos para acceder al state y los recursos de AWS
4. Crear una regla de EventBridge Schedule que ejecute el proyecto de CodeBuild periodicamente
5. Implementar el parsing del output JSON de `terraform plan` para extraer recursos con drift
6. Crear una funcion Lambda que reciba el resultado del plan y genere notificaciones legibles
7. Configurar SNS para enviar notificaciones con el detalle del drift detectado
8. Introducir drift manualmente (modificar un security group, agregar un tag) y verificar la deteccion
9. Implementar logica de auto-remediacion para patrones de drift conocidos (con aprobacion o automatico)
10. Crear un dashboard o log que mantenga el historial de drift detectado

## Validacion

- [ ] El proyecto de CodeBuild ejecuta `terraform plan` exitosamente
- [ ] EventBridge ejecuta el CodeBuild en el schedule programado
- [ ] El pipeline detecta drift cuando se modifican recursos manualmente
- [ ] El parsing del plan identifica correctamente los recursos afectados
- [ ] Las notificaciones incluyen un diff claro y legible del drift
- [ ] La auto-remediacion corrige el drift para patrones conocidos
- [ ] No se generan notificaciones falsas cuando no hay drift
- [ ] El historial de drift se mantiene para auditoria
