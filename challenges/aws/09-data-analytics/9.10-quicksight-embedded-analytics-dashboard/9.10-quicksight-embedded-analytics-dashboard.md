# 9.10 QuickSight Embedded Analytics Dashboard

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Manual (consola QuickSight) + codigo para embedding |
| **Servicios AWS** | QuickSight, Athena/Redshift, S3, Cognito (para autenticacion de embedding) |

## Descripcion

Construir un dashboard en QuickSight conectado a fuentes de datos Athena/Redshift. Crear campos calculados, parametros y filtros interactivos. Implementar row-level security (RLS) para aislamiento de datos multi-tenant. Embeber el dashboard en una aplicacion web.

## Objetivos de aprendizaje

- Comprender Business Intelligence en AWS con QuickSight
- Implementar embedded analytics en aplicaciones web
- Configurar row-level security para aislamiento multi-tenant
- Utilizar el motor SPICE para acelerar consultas
- Aplicar patrones de diseno de dashboards efectivos

## Requisitos previos

- Conocimientos: SQL basico, conceptos de BI y visualizacion, desarrollo web basico (HTML/JS)
- Herramientas: AWS CLI, navegador web, editor de codigo para la aplicacion web
- Servicios AWS: Cuenta QuickSight Enterprise Edition, permisos para Athena/Redshift, S3, Cognito

## Arquitectura

QuickSight se conecta a fuentes de datos (Athena para datos en el data lake, Redshift para el data warehouse) y carga datos en el motor SPICE para consultas rapidas. El dashboard incluye campos calculados, parametros con controles interactivos y filtros encadenados. Row-level security (RLS) se implementa con un dataset de reglas que mapea usuarios/grupos a los datos que pueden ver, habilitando aislamiento multi-tenant. Para el embedding, Cognito autentica a los usuarios de la aplicacion web, y la API de QuickSight genera URLs de embedding firmadas con permisos apropiados segun el tenant.

## Pasos sugeridos

1. Configurar QuickSight Enterprise Edition y registrar las fuentes de datos (Athena y/o Redshift)
2. Crear datasets en QuickSight e importar datos al motor SPICE
3. Crear campos calculados (metricas derivadas, categorias condicionales)
4. Construir visualizaciones variadas (graficos de barras, lineas, KPIs, tablas pivot)
5. Agregar parametros con controles interactivos y filtros encadenados
6. Crear un dataset de reglas RLS que mapee usuarios a filtros de datos
7. Aplicar RLS al dashboard y verificar que cada usuario ve solo sus datos
8. Configurar un Cognito User Pool para autenticacion de la aplicacion web
9. Implementar el codigo de embedding usando la QuickSight Embedding SDK
10. Probar el dashboard embebido con diferentes usuarios y verificar el aislamiento RLS

## Validacion

- [ ] El dashboard se conecta y muestra datos correctos desde Athena/Redshift
- [ ] Los datos estan cargados en SPICE y las consultas son rapidas
- [ ] Los campos calculados y parametros funcionan correctamente
- [ ] Los filtros interactivos permiten explorar los datos dinamicamente
- [ ] RLS restringe los datos visibles segun el usuario/tenant
- [ ] El dashboard embebido se muestra correctamente en la aplicacion web
- [ ] Diferentes usuarios ven datos diferentes segun las reglas RLS
