# 9.05 EMR con Spot Instances y Spark

## Descripcion

Desplegar un cluster EMR con una combinacion de instancias On-Demand (master/core) y Spot (task). Ejecutar jobs PySpark para procesamiento batch. Configurar politicas de auto-scaling basadas en metricas de YARN. Implementar un flujo multi-step con EMR Steps.

## Objetivos de aprendizaje

- Comprender la arquitectura de un cluster EMR (master, core, task nodes)
- Utilizar instancias Spot en cargas de trabajo de big data para optimizar costos
- Ajustar y optimizar jobs de Spark (particionado, caching, shuffle)
- Configurar auto-scaling basado en metricas de YARN
- Orquestar flujos de trabajo con EMR Steps

## Requisitos previos

- Conocimientos: Fundamentos de Spark/PySpark, conceptos de clusters distribuidos, S3
- Herramientas: AWS CLI, Terraform, Python 3.x, conocimientos basicos de YARN
- Servicios AWS: Cuenta con permisos para EMR, EC2, S3, CloudWatch

## Arquitectura

El cluster EMR se compone de tres tipos de nodos. El nodo master (On-Demand) gestiona el cluster y YARN ResourceManager. Los nodos core (On-Demand) almacenan datos HDFS y ejecutan tareas. Los nodos task (Spot) solo ejecutan tareas sin almacenar datos, lo que los hace tolerantes a interrupciones Spot. Auto-scaling ajusta la cantidad de nodos task segun metricas de YARN (YARNMemoryAvailablePercentage, ContainerPendingRatio). EMR Steps orquesta la ejecucion secuencial de multiples jobs PySpark. Los datos de entrada y salida residen en S3.

## Pasos sugeridos

1. Crear un bucket S3 para datos de entrada, salida y logs del cluster EMR
2. Subir datasets de ejemplo y scripts PySpark al bucket S3
3. Configurar y lanzar un cluster EMR con nodos master/core On-Demand y task Spot
4. Configurar instance fleets con multiples tipos de instancia para nodos task (diversificacion Spot)
5. Escribir y ejecutar un job PySpark basico que lea de S3 y escriba resultados transformados
6. Configurar politicas de auto-scaling basadas en metricas de YARN
7. Implementar un flujo multi-step con varios jobs PySpark encadenados como EMR Steps
8. Monitorear el cluster via la consola EMR y metricas de CloudWatch
9. Probar el comportamiento ante interrupciones Spot y verificar que el procesamiento continua
10. Optimizar los jobs Spark (ajustar particionado, configuracion de executor memory/cores)

## Validacion

- [ ] El cluster EMR se lanza correctamente con nodos On-Demand y Spot
- [ ] Los jobs PySpark se ejecutan y producen resultados correctos en S3
- [ ] El auto-scaling agrega y remueve nodos task segun la carga de trabajo
- [ ] Los EMR Steps se ejecutan secuencialmente sin errores
- [ ] El cluster tolera la perdida de nodos task Spot sin interrumpir el procesamiento
- [ ] Las metricas de YARN y Spark son visibles en CloudWatch
- [ ] Los costos de Spot son significativamente menores que On-Demand para los nodos task
