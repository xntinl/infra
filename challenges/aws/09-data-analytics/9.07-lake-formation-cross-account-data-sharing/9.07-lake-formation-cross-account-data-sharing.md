# 9.07 Lake Formation Cross-Account Data Sharing

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Ambos |
| **Servicios AWS** | Lake Formation, Organizations, Glue Data Catalog, Athena, S3 |

## Descripcion

Configurar Lake Formation para compartir datasets curados desde una cuenta central de data lake con cuentas consumidoras. Implementar control de acceso basado en etiquetas (LF-Tags), configurar filtros a nivel de fila y columna, y verificar que los consumidores pueden consultar los datos compartidos via Athena.

## Objetivos de aprendizaje

- Comprender patrones de data mesh y su implementacion en AWS
- Configurar comparticion de datos cross-account con Lake Formation
- Implementar control de acceso basado en LF-Tags
- Aplicar gobernanza de datos granular (filtros de fila y columna)
- Establecer un modelo productor/consumidor de datos

## Requisitos previos

- Conocimientos: Lake Formation basico, conceptos de data governance, AWS Organizations
- Herramientas: AWS CLI, Terraform, acceso a dos o mas cuentas AWS
- Servicios AWS: Cuentas con permisos para Lake Formation, Glue, Athena, S3, Organizations

## Arquitectura

La arquitectura sigue un modelo de data mesh con una cuenta central (productora) y cuentas consumidoras. La cuenta central aloja el data lake en S3 con datos catalogados en Glue Data Catalog y gobernados por Lake Formation. LF-Tags clasifican las tablas y columnas por dominio, sensibilidad y nivel de acceso. Lake Formation comparte recursos con cuentas consumidoras a traves de AWS RAM (Resource Access Manager) o invitaciones directas. Las cuentas consumidoras crean resource links en sus propios catalogs para acceder a los datos compartidos. Los filtros de fila y columna garantizan que cada consumidor solo vea los datos que le corresponden.

## Pasos sugeridos

1. Configurar la cuenta central como administradora de Lake Formation con los datos en S3
2. Catalogar los datos con Glue Crawlers y registrar el data lake en Lake Formation
3. Definir LF-Tags (ej: dominio=ventas/finanzas, sensibilidad=publica/confidencial)
4. Asignar LF-Tags a bases de datos, tablas y columnas del catalogo
5. Crear politicas de acceso basadas en LF-Tags para diferentes roles
6. Configurar la comparticion cross-account via AWS RAM o Lake Formation grants
7. En la cuenta consumidora, aceptar la comparticion y crear resource links
8. Configurar filtros de fila para restringir registros visibles por cuenta consumidora
9. Configurar filtros de columna para ocultar columnas sensibles a ciertos consumidores
10. Verificar desde la cuenta consumidora que las consultas Athena respetan los permisos

## Validacion

- [ ] Los LF-Tags estan definidos y asignados correctamente a tablas y columnas
- [ ] La comparticion cross-account esta configurada y activa
- [ ] La cuenta consumidora puede ver los resource links en su catalogo
- [ ] Las consultas Athena desde la cuenta consumidora retornan datos correctos
- [ ] Los filtros de fila restringen los registros visibles segun la configuracion
- [ ] Los filtros de columna ocultan las columnas sensibles a consumidores no autorizados
- [ ] Los permisos basados en LF-Tags funcionan segun las politicas definidas
