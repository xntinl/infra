# 13.08 Pipeline de Auditoria y Compliance del State

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform + Python/Go |
| **Features TF** | `terraform show -json`, plan JSON output |

## Descripcion

Construir un pipeline que audite los state files de Terraform: verificar recursos sin tags, identificar security groups excesivamente permisivos, detectar recursos en regiones no autorizadas. Usar `terraform show -json` para parsear plan/state y generar reportes de compliance.

## Objetivos de aprendizaje

- Realizar introspeccion programatica del state de Terraform
- Automatizar verificaciones de compliance sobre infraestructura gestionada
- Comprender el formato JSON del plan y state de Terraform
- Interactuar con Terraform de forma programatica

## Requisitos previos

- Conocimientos: Terraform state, JSON parsing, Python o Go basico, politicas de compliance
- Herramientas: Terraform >= 1.0, Python 3.x o Go, jq, CI/CD runner
- Features: `terraform show -json`, `terraform plan -out`, JSON plan format

## Arquitectura

El pipeline de auditoria opera como una capa de validacion sobre el state:

- **Extraccion**: `terraform show -json` genera una representacion JSON completa del state actual
- **Parser**: Script en Python/Go que procesa el JSON y extrae informacion de cada recurso
- **Reglas de Compliance**: Conjunto configurable de verificaciones (tags obligatorios, regiones permitidas, reglas de SG)
- **Reporte**: Salida estructurada con violaciones encontradas, severidad y recomendaciones
- **CI Integration**: El pipeline se ejecuta en cada PR/merge para validar compliance continuo

## Pasos sugeridos

1. Desplegar infraestructura de ejemplo con mezcla de recursos compliant y no-compliant
2. Generar el JSON del state con `terraform show -json` y explorar su estructura
3. Generar un plan JSON con `terraform plan -out=plan.bin && terraform show -json plan.bin`
4. Escribir un script que parsee el state JSON y extraiga todos los recursos con sus atributos
5. Implementar regla de auditoria: verificar que todos los recursos tienen tags obligatorios (Name, Environment, Owner)
6. Implementar regla: detectar security groups con ingress 0.0.0.0/0 en puertos sensibles
7. Implementar regla: verificar que todos los recursos estan en regiones autorizadas
8. Generar un reporte de compliance en formato JSON/HTML con las violaciones encontradas
9. Integrar el script en un pipeline CI que falle si hay violaciones criticas
10. Agregar reglas adicionales segun las necesidades del equipo

## Validacion

- [ ] El script parsea correctamente el JSON del state de Terraform
- [ ] Se detectan recursos sin tags obligatorios
- [ ] Se identifican security groups con reglas excesivamente permisivas
- [ ] Se detectan recursos en regiones no autorizadas
- [ ] Se genera un reporte de compliance legible y estructurado
- [ ] El pipeline CI falla cuando hay violaciones criticas
- [ ] Las reglas de compliance son configurables y extensibles
