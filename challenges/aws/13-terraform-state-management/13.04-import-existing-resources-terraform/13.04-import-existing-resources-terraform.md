# 13.04 Import de Recursos Existentes a Terraform

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform |
| **Features TF** | `import` block, `terraform import`, resource addressing |

## Descripcion

Tomar un conjunto de recursos AWS creados manualmente (VPC, subnets, security groups, instancias EC2) y ponerlos bajo gestion de Terraform. Usar `terraform import` y el bloque `import` (1.5+). Escribir la configuracion HCL correspondiente. Verificar con `terraform plan` que no muestra cambios.

## Objetivos de aprendizaje

- Dominar el flujo de trabajo de importacion de recursos
- Generar configuracion HCL a partir de recursos existentes
- Reconciliar el state con la infraestructura real
- Adoptar Terraform en entornos brownfield

## Requisitos previos

- Conocimientos: Terraform intermedio, recursos AWS (VPC, EC2, SG), resource addressing
- Herramientas: Terraform >= 1.5, AWS CLI, consola AWS para crear recursos manualmente
- Servicios/Features: VPC, EC2, Security Groups, `import` block

## Arquitectura

El escenario simula una adopcion brownfield donde la infraestructura ya existe:

- **Recursos Pre-existentes**: VPC con CIDR personalizado, subnets publicas y privadas, security groups con reglas especificas, instancias EC2 en ejecucion
- **Configuracion HCL**: Archivos `.tf` que replican exactamente la configuracion de los recursos existentes
- **Import Blocks**: Bloques `import` declarativos (Terraform 1.5+) que mapean recursos AWS a direcciones de Terraform
- **Validacion**: `terraform plan` debe mostrar cero cambios, confirmando que la configuracion HCL coincide exactamente con la realidad

## Pasos sugeridos

1. Crear manualmente en AWS: una VPC, dos subnets, un internet gateway, un security group y una instancia EC2
2. Documentar los IDs de todos los recursos creados (vpc-xxx, subnet-xxx, sg-xxx, i-xxx)
3. Escribir la estructura base de archivos Terraform (main.tf, variables.tf, outputs.tf)
4. Usar `terraform import` (CLI) para importar la VPC al state y observar el resultado
5. Escribir la configuracion HCL de la VPC que coincida con los atributos reales
6. Migrar al enfoque declarativo: usar bloques `import` en HCL para los recursos restantes
7. Ejecutar `terraform plan` e iterar ajustando la configuracion hasta lograr cero cambios
8. Verificar que cada recurso importado tiene todos sus atributos correctamente reflejados
9. Ejecutar `terraform apply` para confirmar que no se realizan cambios
10. Probar modificar un atributo via Terraform para confirmar que la gestion funciona correctamente

## Validacion

- [ ] Todos los recursos pre-existentes fueron importados al state de Terraform
- [ ] Se utilizaron tanto `terraform import` (CLI) como bloques `import` (HCL)
- [ ] La configuracion HCL refleja exactamente los atributos de los recursos reales
- [ ] `terraform plan` muestra cero cambios despues de la importacion completa
- [ ] Se puede modificar un recurso via Terraform y el cambio se aplica correctamente
- [ ] El proceso de importacion esta documentado paso a paso
