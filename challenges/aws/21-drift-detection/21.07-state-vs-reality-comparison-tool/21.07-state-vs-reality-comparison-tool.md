# 21.07 State vs Reality Comparison Tool

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform + Python/Go |
| **Features TF** | terraform show -json, AWS SDK |

## Descripcion

Construye una herramienta que compare el estado de Terraform con los recursos reales en AWS usando el SDK. Identifica recursos que estan en el estado pero fueron eliminados en AWS, recursos que existen en AWS pero no estan en el estado, y diferencias de atributos. Genera un reporte de reconciliacion detallado.

## Objetivos de aprendizaje

- Introspeccion del estado de Terraform
- Validacion basada en SDK contra la realidad de AWS
- Patrones de reconciliacion de infraestructura

## Requisitos previos

- Conocimientos: Terraform intermedio, Python o Go, AWS SDK (boto3 o aws-sdk-go)
- Herramientas: Terraform CLI, Python 3 o Go, AWS CLI
- Servicios/Features: terraform show -json, AWS SDK (EC2, S3, IAM, etc.)

## Arquitectura

La herramienta tiene tres fases. Primero, extrae el estado actual de Terraform usando `terraform show -json` para obtener la lista completa de recursos gestionados con sus atributos. Segundo, consulta la API de AWS usando el SDK para obtener el estado real de cada recurso. Tercero, compara ambas fuentes y genera un reporte con tres categorias: recursos fantasma (en estado pero eliminados en AWS), recursos huerfanos (en AWS pero no en estado), y diferencias de atributos (el recurso existe en ambos pero con valores distintos).

## Pasos sugeridos

1. Desplegar una variedad de recursos con Terraform (EC2, S3, security groups, IAM roles)
2. Implementar el modulo de extraccion de estado usando `terraform show -json`
3. Parsear el JSON de estado y extraer tipo de recurso, ID y atributos relevantes
4. Implementar los adaptadores de AWS SDK para consultar cada tipo de recurso (EC2, S3, IAM, etc.)
5. Desarrollar la logica de comparacion: detectar recursos ausentes, extras y diferencias de atributos
6. Disenar el formato del reporte de reconciliacion (JSON, HTML o tabla en consola)
7. Simular cada escenario: eliminar un recurso manualmente, crear uno fuera de Terraform, modificar un atributo
8. Ejecutar la herramienta y verificar que detecta las tres categorias de discrepancias
9. Agregar soporte para multiples tipos de recurso de forma extensible
10. Implementar sugerencias de remediacion para cada tipo de discrepancia

## Validacion

- [ ] La herramienta parsea correctamente el estado de Terraform via `terraform show -json`
- [ ] Detecta recursos que existen en el estado pero fueron eliminados en AWS
- [ ] Detecta recursos en AWS que no estan gestionados por Terraform
- [ ] Identifica diferencias de atributos entre el estado y la realidad
- [ ] Genera un reporte de reconciliacion legible con las tres categorias
- [ ] Soporta al menos tres tipos de recurso diferentes (EC2, S3, security groups)
