# 21.05 Multi-State Drift Dashboard

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform + Python + QuickSight/Grafana |
| **Features TF** | Plan JSON, multi-workspace scanning |

## Descripcion

Construye un dashboard que muestre el estado de drift en todos los archivos de estado de Terraform dentro de la organizacion. Agrega resultados de multiples workspaces y cuentas. Muestra tendencias, los recursos con mas drift, y las principales causas de desviacion.

## Objetivos de aprendizaje

- Visibilidad de drift a nivel organizacional
- Agregacion de metricas de infraestructura
- Construccion de dashboards operacionales

## Requisitos previos

- Conocimientos: Terraform multi-workspace, Python, conceptos de visualizacion de datos
- Herramientas: Terraform CLI, Python 3, Grafana o QuickSight, AWS CLI
- Servicios/Features: S3 (remote state), Lambda, DynamoDB o RDS (almacenamiento de metricas), Grafana/QuickSight

## Arquitectura

Un proceso programado (Lambda o Step Functions) itera sobre todos los workspaces y cuentas de Terraform configurados. Para cada uno, ejecuta `terraform plan -json` y extrae la informacion de drift. Los resultados se almacenan en una base de datos (DynamoDB o RDS) con timestamp, workspace, recurso afectado y tipo de cambio. Grafana o QuickSight se conecta a esta fuente de datos para generar visualizaciones: estado actual de drift por workspace, tendencias historicas, top de recursos con mas drift y causas principales de desviacion.

## Pasos sugeridos

1. Configurar multiples workspaces de Terraform con recursos simples en cada uno
2. Desarrollar un script Python que enumere todos los workspaces y ejecute `terraform plan -json` en cada uno
3. Implementar el parser que extraiga informacion de drift del plan JSON (recurso, tipo de cambio, atributos)
4. Disenar el esquema de almacenamiento de metricas (DynamoDB o RDS)
5. Almacenar los resultados de cada escaneo con timestamp y metadata del workspace
6. Configurar Grafana o QuickSight como herramienta de visualizacion
7. Crear el panel de estado actual: semaforo de drift por workspace/cuenta
8. Crear el panel de tendencias: evolucion del drift a lo largo del tiempo
9. Crear el panel de top ofensores: recursos y causas mas frecuentes de drift
10. Programar la ejecucion periodica del escaneo con EventBridge

## Validacion

- [ ] El sistema escanea multiples workspaces de Terraform automaticamente
- [ ] Los resultados de drift se almacenan con timestamp en la base de datos
- [ ] El dashboard muestra el estado actual de drift por workspace
- [ ] Se visualizan tendencias historicas de drift
- [ ] Se identifican los recursos con mas drift y las causas principales
- [ ] El escaneo se ejecuta de forma programada sin intervencion manual
