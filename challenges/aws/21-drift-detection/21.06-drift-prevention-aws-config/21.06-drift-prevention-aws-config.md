# 21.06 Drift Prevention con AWS Config

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform |
| **Servicios AWS** | AWS Config, EventBridge, SNS, Lambda |

## Descripcion

Utiliza reglas de AWS Config para detectar cambios que causarian drift en Terraform. Alerta inmediatamente cuando alguien modifica un recurso gestionado por Terraform a traves de la consola o CLI. Usa tags para identificar los recursos gestionados por Terraform.

## Objetivos de aprendizaje

- Prevencion proactiva de drift
- Deteccion en tiempo real de cambios no autorizados
- Identificacion de recursos basada en tags

## Requisitos previos

- Conocimientos: Terraform intermedio, AWS Config, EventBridge, patrones de eventos
- Herramientas: Terraform CLI, AWS CLI
- Servicios/Features: AWS Config, EventBridge, SNS, Lambda, IAM

## Arquitectura

Los recursos gestionados por Terraform se etiquetan con un tag identificador (por ejemplo, `ManagedBy=terraform`). AWS Config monitorea continuamente los cambios en la configuracion de estos recursos. Cuando detecta un cambio fuera de Terraform, genera un evento de configuracion. EventBridge captura estos eventos y los filtra por recursos que tienen el tag de Terraform. Una funcion Lambda evalua el cambio y envia una alerta SNS con detalles del recurso modificado, quien lo modifico y que cambio.

## Pasos sugeridos

1. Desplegar recursos base con Terraform asegurando que todos tengan el tag `ManagedBy=terraform`
2. Configurar AWS Config recorder y delivery channel via Terraform
3. Crear reglas de AWS Config personalizadas que evaluen cambios en recursos con el tag de Terraform
4. Configurar EventBridge rules para capturar eventos de AWS Config cuando un recurso etiquetado cambia
5. Desarrollar la funcion Lambda que procese el evento, identifique el recurso y el cambio realizado
6. Configurar el topico SNS y las suscripciones para las alertas
7. Implementar la logica para extraer quien realizo el cambio (usando CloudTrail)
8. Modificar un recurso manualmente via consola y verificar que se genera la alerta
9. Validar que cambios realizados via Terraform no generen alertas falsas
10. Agregar metricas de CloudWatch para tracking de cambios fuera de Terraform

## Validacion

- [ ] Los recursos desplegados por Terraform tienen el tag `ManagedBy=terraform`
- [ ] AWS Config detecta cambios en recursos etiquetados en tiempo real
- [ ] La regla de EventBridge filtra correctamente los eventos de cambio
- [ ] La funcion Lambda identifica el recurso, el cambio y el actor
- [ ] Se recibe una alerta SNS cuando se modifica un recurso via consola/CLI
- [ ] Los cambios realizados via Terraform no generan alertas falsas
