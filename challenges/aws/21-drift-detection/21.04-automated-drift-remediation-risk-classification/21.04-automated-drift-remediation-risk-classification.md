# 21.04 Automated Drift Remediation Risk Classification

## Metadata
| Campo | Valor |
|-------|-------|
| **Complejidad** | Avanzado |
| **Modalidad** | Terraform + Lambda/Step Functions |
| **Features TF** | Plan JSON parsing, targeted apply |

## Descripcion

Construye un sistema inteligente de remediacion de drift que clasifica cada desviacion segun su nivel de riesgo. El sistema distingue tres niveles: bajo (cambios en tags), medio (cambios de configuracion) y alto (cambios en security groups). El drift de bajo riesgo se remedia automaticamente, el de riesgo medio genera una alerta, y el de alto riesgo dispara una notificacion de guardia (page-on-call).

## Objetivos de aprendizaje

- Remediacion basada en niveles de riesgo
- Gobernanza automatizada de infraestructura
- Clasificacion inteligente de drift

## Requisitos previos

- Conocimientos: Terraform intermedio, AWS Lambda, Step Functions, JSON parsing
- Herramientas: Terraform CLI, AWS CLI, Python o Node.js para Lambda
- Servicios/Features: AWS Lambda, Step Functions, SNS, PagerDuty/OpsGenie (opcional), terraform plan -json

## Arquitectura

El flujo comienza con una ejecucion programada de `terraform plan -json` que genera un plan detallado de cambios. Una funcion Lambda analiza el JSON del plan e identifica cada recurso con drift, clasificandolo en bajo, medio o alto riesgo segun el tipo de recurso y atributos modificados. Un workflow de Step Functions orquesta la remediacion: para riesgo bajo ejecuta `terraform apply -target` automaticamente, para riesgo medio envia una alerta a SNS, y para riesgo alto dispara una notificacion urgente al equipo de guardia.

## Pasos sugeridos

1. Crear la infraestructura base que sera monitoreada (EC2, security groups, recursos con tags)
2. Implementar un script que ejecute `terraform plan -json` y almacene el resultado en S3
3. Desarrollar la funcion Lambda de clasificacion que parsee el plan JSON y categorice cada cambio por riesgo
4. Definir las reglas de clasificacion: mapear tipos de recurso y atributos a niveles de riesgo
5. Crear el workflow de Step Functions que orqueste la logica de remediacion segun el nivel
6. Implementar la rama de remediacion automatica (bajo riesgo) usando `terraform apply -target`
7. Configurar las notificaciones SNS para riesgo medio y la integracion de paging para riesgo alto
8. Programar la ejecucion periodica con EventBridge
9. Introducir drift manual en cada categoria y verificar que el sistema responde correctamente
10. Agregar logging y metricas para auditar las acciones de remediacion

## Validacion

- [ ] El sistema detecta drift de bajo riesgo (cambios en tags) y lo remedia automaticamente
- [ ] Los cambios de configuracion (riesgo medio) generan una alerta SNS sin remediacion automatica
- [ ] Los cambios en security groups (riesgo alto) disparan una notificacion urgente
- [ ] El plan JSON se parsea correctamente y cada recurso se clasifica en el nivel adecuado
- [ ] El workflow de Step Functions ejecuta la rama correcta segun el nivel de riesgo
- [ ] Despues de la remediacion automatica, `terraform plan` no muestra cambios para los recursos de bajo riesgo
